# OpenList 与夸克扫码网盘媒体库设计

> 2026-07-19 更新：本文的 OpenList 设计仍然有效；其中夸克扫码登录部分已被新的 Cookie 原生接入设计替代，后续以 `2026-07-19-quark-cookie-cloud-library-design.md` 为准。

## 目标

在看影音中增加两类网盘媒体库数据源：稳定的 OpenList 接入，以及实验性的夸克扫码直连接入。两类来源使用统一的索引和播放接口，但认证与远程协议完全隔离。应用扫描远程视频并合并到媒体库，播放时获取实时直链交给现有播放器。

## 范围

首期支持：

- 配置一个或多个 OpenList 服务器。
- 通过二维码登录一个或多个夸克网盘账号。
- 选择每个服务器中的一个或多个媒体目录。
- 扫描远程视频、外部字幕和基本文件元数据。
- 将网盘内容加入现有媒体库，并与本地来源明确区分。
- 复用现有系列识别、季度与类型拆分、TMDB 刮削、播放历史和播放器。
- 播放时获取最新直链，支持拖动进度和自动刷新一次失效直链。
- 下载同目录匹配的字幕小文件到临时缓存后加载。

首期不支持视频离线缓存、上传、重命名、移动或删除网盘文件。OpenList 中的网盘账号仍由用户在自己的 OpenList 中配置；夸克直连账号由看影音扫码登录。

## 接入方式

OpenList 使用 API 完成认证、目录浏览、文件信息和播放直链获取。浏览与索引不依赖 WebDAV；WebDAV 可作为后续兼容方案。夸克直连通过独立适配器提供相同的文件浏览和播放能力。

- 登录后保存 OpenList 访问令牌和服务器配置。
- 应用不保存 OpenList 明文密码；Windows 使用系统安全存储保存凭据，其他平台使用对应安全存储能力。
- API 地址统一由服务器基础地址生成，并规范化结尾斜杠。
- 服务器支持自签名证书时必须由用户显式开启，不默认跳过证书校验。

## 夸克扫码登录

- “添加网盘”提供“夸克扫码登录”和“OpenList”两个入口。
- 登录弹窗显示二维码、有效期和当前状态。
- 状态明确区分：正在获取二维码、等待扫码、已扫码待确认、登录成功、二维码过期、当前版本不兼容。
- 二维码由夸克适配器请求，客户端按服务端建议间隔轮询；弹窗关闭、登录成功或二维码过期时立即停止轮询。
- 二维码过期后由用户手动刷新，不在后台无限生成或轮询。
- 登录成功后只保存后续浏览和播放所需的最小会话信息，不保存账号密码。
- 会话使用平台安全存储加密保存在本机，不上传到看影音、OpenList 或其他服务。
- 会话失效后提示重新扫码，不提供密码登录或自动续期。
- 二维码内容、Cookie、令牌、设备标识和签名参数不得写入日志。
- 夸克协议不兼容时显示“当前版本暂不兼容夸克扫码登录”，不得伪装成普通网络错误。

## 数据模型

新增网盘数据源模型：

- 唯一来源 ID。
- 来源类型、显示名称与服务器地址（夸克直连不需要用户填写地址）。
- 用户名或账号显示信息、凭据引用和已选媒体目录。
- 启用状态、最近扫描时间与连接状态。

远程媒体索引项保存：

- 来源 ID、远程路径、文件名、大小和修改时间。
- 文件稳定标识；OpenList 未提供稳定 ID 时使用来源 ID 与规范化路径组合。
- 视频类型、解析出的系列名、季度、集数和作品类型。
- 匹配到的远程字幕路径。
- TMDB 元数据与本地缓存海报路径。
- 不持久化可能过期的播放直链。

本地与远程索引通过统一媒体条目接口提供给媒体库，但各自的扫描器、路径解析和播放请求解析器保持独立。

## 扫描流程

1. 用户添加 OpenList 服务器或通过二维码登录夸克网盘。
2. 用户浏览对应网盘目录并选择媒体根目录。
3. 扫描器分页或分批递归读取目录，按现有视频扩展名和大小规则筛选。
4. 同目录读取受支持字幕，复用本地字幕匹配规则建立关联。
5. 使用路径、大小和修改时间生成目录指纹，仅更新变化目录。
6. 将扫描结果写入独立远程索引，再由媒体库聚合本地与远程条目。
7. 扫描支持取消、进度显示、失败目录列表和单独重试。

远程扫描不得阻塞本地媒体库浏览。服务器离线时保留上次成功索引，并标记来源当前不可用。

## 媒体库呈现

- 网盘视频直接进入现有媒体库海报墙，不增加独立网盘主页面。
- 卡片显示来源图标或来源名称，避免与本地文件混淆。
- 搜索、排序、季度拆分、OVA、特别篇和剧场版独立显示规则与本地媒体一致。
- 详情页展示远程路径和服务器来源，不展示访问令牌或真实播放直链。
- 用户可以按来源筛选，并可仅刷新指定网盘来源。

## TMDB 与海报

- 远程条目复用现有 TMDB 匹配和手动候选流程。
- TMDB 海报下载到应用缓存目录，按来源 ID 和媒体稳定标识隔离。
- 不把 `tmdb-poster.jpg` 写回任何网盘，避免修改远程内容。
- 海报缓存失败时继续使用 TMDB 网络地址，离线时显示已有缓存或占位图。

## 播放流程

1. 用户点击网盘媒体卡片。
2. 播放请求解析器根据来源类型、来源 ID 和远程路径，通过对应适配器请求最新文件信息。
3. 取得 `raw_url`、必要请求头和字幕信息，不把直链写入长期存储。
4. 外部字幕存在时下载到应用临时字幕目录。
5. 将播放 URL、请求头、字幕路径、播放列表和历史进度传给现有 `LocalVideoController`/播放器接口的远程媒体扩展。
6. 播放器收到鉴权失败或直链过期错误时，重新取直链并重试一次；再次失败则停止并显示错误。

播放器继续支持播放速度、硬件解码、超分辨率、字幕样式和按视频记忆的字幕时间偏移。远程视频的字幕偏移键由来源 ID 与远程路径组成。

## 认证与安全

- 日志中对令牌、密码、签名参数和播放直链进行脱敏。
- OpenList 设置只允许重新输入密码，不回显已有密码；夸克来源只允许重新扫码。
- 删除数据源时同步删除令牌、凭据和该来源的临时直链缓存；媒体索引与海报缓存由用户确认是否清理。
- 连接测试区分地址错误、证书错误、认证失败、权限不足和服务器不可达。
- 所有 API 请求配置连接与读取超时，并允许用户取消长时间扫描。

## 错误处理

- OpenList 登录过期：自动重新认证一次，失败后要求用户更新凭据。
- 夸克会话失效：停止使用旧会话并要求重新扫码。
- 服务器离线：保留旧索引并标记不可播放。
- 文件删除或移动：下一次扫描移除旧索引；播放时发现不存在则立即提示。
- 直链过期：刷新直链后重试一次，禁止无限重试。
- 字幕下载失败：继续播放视频并提示字幕未加载。
- 单目录扫描失败：记录失败目录并继续其他目录。

## 测试

- OpenList 登录、令牌刷新、目录列表和文件信息解析。
- 夸克二维码状态轮询、取消、过期、成功、会话失效和协议不兼容。
- 测试使用本地模拟响应，不请求真实二维码，也不使用真实夸克账号。
- 多来源配置、凭据脱敏和删除清理。
- 递归扫描、增量指纹、取消、失败继续和来源隔离。
- 视频与字幕匹配、季度和类型拆分保持现有行为。
- 本地与远程条目聚合、来源筛选和不可用状态。
- 播放前获取实时直链、请求头透传和过期直链单次重试。
- 字幕临时下载与按远程媒体键恢复字幕偏移。
- TMDB 海报缓存不写回网盘。
- 完整回归测试确保本地媒体库和本地播放不受影响。

## 分阶段交付

第一阶段实现统一网盘接口、OpenList 客户端和数据源配置。第二阶段实现夸克扫码适配器和只读目录浏览。第三阶段实现远程索引并合并媒体库。第四阶段实现直链播放、字幕和错误重试。第五阶段补齐 TMDB 缓存、来源筛选、完整测试和 Windows 安装包。
