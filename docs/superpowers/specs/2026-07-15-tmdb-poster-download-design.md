# TMDB 刮削海报下载设计

## 目标

TMDB 信息刮削成功后，将匹配到的海报保存到视频所在文件夹，使封面能够离线使用，并在本地页面和媒体库中立即显示。

## 文件规则

- 下载文件统一命名为 `tmdb-poster.jpg`。
- 不覆盖 `cover.jpg`、`poster.jpg`、`folder.jpg`、视频同名图片或其他用户封面。
- 同一作品的视频分布在多个文件夹时，每个含视频的文件夹都保存一份。
- 同一文件夹内只下载一次，避免按剧集重复请求。
- 重新刮削或手动重新匹配时，允许覆盖旧的 `tmdb-poster.jpg`。

## 封面识别顺序

本地封面查找顺序调整为：

1. `tmdb-poster`，支持现有图片扩展名集合。
2. 当前目录的系列封面。
3. 视频同名封面。
4. `cover`、`poster`、`folder` 等通用封面。
5. 已生成的视频缩略图。
6. 兼容旧目录结构的上级目录封面。

该顺序确保 TMDB 下载海报生效，同时完整保留用户已有封面文件。

## 集成位置

- 下载逻辑集成到 TMDB 刮削服务的成功结果处理，不分散到页面回调。
- 自动刮削、批量刮削、单部刮削、重新刮削和手动选择候选统一执行下载。
- 刮削服务从媒体索引取得同系列视频路径，按目录去重后下载海报。
- 下载成功后，将对应索引项的本地封面路径更新为该目录的 `tmdb-poster.jpg`。
- 控制器在服务完成后重新加载媒体索引，触发本地页面和媒体库即时刷新。

## 结果与错误处理

- TMDB 元数据保存和海报下载是两个独立结果。
- 海报下载失败不会撤销已经成功保存的 TMDB 信息。
- 全部下载成功时沿用“TMDB 信息已更新”提示。
- 部分或全部目录下载失败时提示“TMDB 信息已更新，部分封面下载失败”。
- 下载失败记录目录和错误日志，后续重新刮削时可再次尝试。
- 海报 URL 缺失时视为无可下载封面，不创建空文件。

## 组件边界

- `LocalTmdbScrapeService`：协调匹配、元数据保存、目录去重、海报下载和索引更新。
- `PosterService`：负责把指定 URL 下载到明确的目标路径。
- `LocalCoverFinder`：识别并优先返回 `tmdb-poster`。
- `LocalController`：刷新索引并向界面暴露下载结果。
- 页面：只根据结果显示用户提示，不自行处理文件下载。

## 测试

- 匹配成功后在每个视频目录生成 `tmdb-poster.jpg`。
- 同一目录的多集视频只下载一次。
- 不覆盖已有用户封面文件。
- 重新刮削可以替换旧 `tmdb-poster.jpg`。
- `LocalCoverFinder` 优先识别 TMDB 本地海报。
- 部分目录下载失败时保留 TMDB 元数据，并返回可供界面提示的失败结果。
- 手动选择候选与自动匹配执行相同下载流程。
- 索引刷新后本地页面立即显示下载后的封面。
