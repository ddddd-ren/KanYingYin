# TrueHD 播放与全量日志体系设计

## 背景

当前 Windows 安装包使用的精简版 `libmpv-2.dll` 未包含 TrueHD 音频和 PGS 字幕解码器。使用真实文件验证时，播放器报告 `Failed to initialize a decoder for codec 'truehd'`，PGS 字幕同时报告找不到解码器。现有 TrueHD 兼容重试无法弥补二进制缺失的解码能力。

应用日志目前只在用户开启播放器调试模式后写入完整 libmpv 输出，导致用户首次遇到问题时缺少底层证据。本设计同时解决播放器能力和诊断信息缺失问题。

## 目标

- Windows 安装包默认支持 TrueHD 音频和 PGS 字幕。
- 默认记录完整应用日志与完整 libmpv 日志，不要求用户先开启调试模式。
- 日志自动轮转，单文件最大 10 MB，最多保留 10 个文件。
- 本地媒体路径完整保留；所有远程凭据和敏感 URL 内容自动脱敏。
- 用户可以直接打开日志目录或导出一个可分享的诊断包。
- 日志系统故障不得阻止应用启动、扫描或播放。

## 非目标

- 不在播放时临时转码 TrueHD 音轨。
- 不修改用户视频、音轨或字幕文件。
- 不把日志自动上传到任何服务器。
- 不为非 Windows 平台改变媒体二进制依赖。

## 完整 libmpv 集成

Windows 构建使用 Shinchiro `mpv-winbuild-cmake` 发布的标准 x64 开发构建：

- 发布版本：`20260610`
- 文件：`mpv-dev-x86_64-20260610-git-304426c.7z`
- SHA-256：`8cbb25ea784f01afbb3f904217cab1317430a8bcfd5680fd827a866367f71cc9`

CMake 在构建目录下载和缓存压缩包，始终先校验 SHA-256，再解压并将完整 `libmpv-2.dll` 安装到 Release 目录。哈希不匹配或下载失败时构建必须失败，不能退回缺少解码器的精简 DLL。

标准 x64 构建用于兼容普通 Windows x64 设备，不采用要求更高指令集的 v3 构建。该 DLL 已用问题文件验证：TrueHD 使用 `truehd` 解码器，PGS 使用 `pgssub` 解码器，音频和视频均进入播放状态。

## 播放器行为

播放器继续优先使用用户选择的硬件视频解码设置。TrueHD 是音频能力，不应触发软件视频解码重建。

当当前 TrueHD 音轨失败且存在非 TrueHD 音轨时，可以切换到兼容音轨。若所有音轨都是 TrueHD，播放器记录底层错误并显示明确提示，不再执行无效的“软件视频解码 + 立体声”重复初始化。

完整 libmpv 支持后，问题文件应直接完成首次初始化，不进入兼容分支。

## 日志架构

新增统一日志服务作为应用日志、Flutter 未捕获异常、平台错误和播放器日志的唯一落盘入口。现有 `AppLogger` 保持调用接口，底层写入统一服务，避免大范围修改业务代码。

日志按时间顺序使用 UTF-8 文本记录，至少包含：

- 时间戳、级别、分类和消息。
- 应用版本、构建号和会话标识。
- 启动、退出、页面级关键操作和未捕获异常。
- 本地与网盘扫描阶段、进度摘要、失败类型和耗时。
- 播放器创建、媒体打开、轨道列表、解码器选择、音频输出、字幕、硬件解码、缓冲、错误、重试和资源释放。
- libmpv 的完整日志流，不受“播放器调试模式”开关限制。

控制台调试输出可以继续受开发设置控制，但文件日志始终完整记录。

## 轮转与并发

- 活动文件名固定为 `kanyingyin.log`。
- 写入后文件达到 10 MB 时执行轮转。
- 历史文件使用时间戳命名，最多保留 9 个历史文件，加活动文件共 10 个。
- 删除最旧文件失败时记录到当前日志并继续运行。
- 所有写入通过单一异步队列串行化，避免应用日志和 libmpv 回调并发破坏文件。
- 应用退出时尽力刷新队列，但不得无限等待。

## 脱敏规则

本地媒体完整路径保留，用于定位具体文件、移动硬盘和字幕匹配问题。

远程内容在进入日志队列前完成脱敏：

- URL 仅保留协议、主机和端口。
- 删除 URL 用户信息、查询参数、片段和远程文件路径。
- `Authorization`、Token、Cookie、密码、签名、API Key 和临时访问参数替换为 `[REDACTED]`。
- 异常文本和 libmpv 文本中的远程 URL 使用同一脱敏器处理。
- 导出诊断包时再次执行全量脱敏扫描，形成第二道保护。

## 用户入口

设置页的日志区域提供两个明确操作：

- “打开日志目录”：打开当前应用日志文件夹。
- “导出诊断日志”：生成 ZIP，包含诊断摘要和最近 10 个日志文件。

诊断摘要包含应用版本、Windows 版本、Flutter/播放器版本、CPU 架构、关键播放器设置和生成时间，不包含凭据。导出失败时显示可理解的错误，并保留原日志。

## 更新描述弹窗

新版本首次启动时的更新弹窗使用面向普通用户的描述，不出现 libmpv、FFmpeg、SHA-256 或解码器初始化等实现术语。弹窗标题使用实际发布版本，正文包含：

- 现在可以正常播放带 TrueHD 音轨的视频。
- 支持显示和切换蓝光视频内置的 PGS 字幕。
- 应用会自动保存完整运行日志，遇到问题时可以从设置页导出诊断日志。
- 日志会自动清理旧文件，网盘账号、密码和访问凭据不会写入导出内容。

`RELEASE_NOTES.md` 与 `lib/utils/version_history.dart` 使用相同含义的用户文案，版本更新弹窗不得继续显示上一版本说明。

## 故障处理

- 日志目录创建失败时退化为控制台输出，应用继续启动。
- 单次写入失败不能终止日志队列，后续写入仍会尝试。
- 轮转失败时继续向可用文件写入，避免因日志维护影响播放。
- libmpv 下载、校验或解压失败属于构建失败，禁止生成不完整安装包。
- 诊断导出只读取和复制日志，不移动或删除原文件。

## 测试与验收

自动化测试覆盖：

- 敏感 URL、请求头和异常文本脱敏。
- 本地 Windows 路径保持完整。
- 10 MB 阈值触发轮转，并只保留 10 个文件。
- 并发写入后的日志完整、可按行解析。
- 日志写入和轮转失败不会抛出到业务层。
- libmpv 日志在调试模式关闭时仍写入文件。
- TrueHD 失败不再触发无效的软件视频重建。
- Windows 构建脚本固定资源 URL、SHA-256 和覆盖顺序。

交付验收包括：

- `flutter test` 通过。
- `flutter analyze` 无错误。
- Windows Release 与 MSIX 构建通过。
- 包内 `libmpv-2.dll` 哈希与完整构建解压产物一致。
- 使用 `Rigor.Mortis.2013.1080p.BluRay.Remux.AVC.TrueHD.5.1.2Audio-SONYHD.mkv` 实测 TrueHD 音频可播放。
- 同一文件的 PGS 简体字幕可显示并可切换。
- 日志中可看到 TrueHD、PGS、音频输出和播放状态，同时不包含远程凭据。
- 新版本首次启动时显示本版本更新说明，内容与发布说明和版本历史一致。
- 最终安装包签名有效并复制到桌面。

## 发布影响

完整 DLL 约 117 MB，安装包体积预计增加约 80 MB。版本号、MSIX 版本、发布说明和版本历史必须同步更新，用户文案说明新增 TrueHD/PGS 支持和完整诊断日志。
