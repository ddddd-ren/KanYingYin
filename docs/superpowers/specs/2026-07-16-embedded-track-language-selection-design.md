# 内嵌音轨与字幕语种选择设计

## 目标

为 Windows 播放器增加内嵌多语言音轨和字幕检测、默认选择与手动切换能力。控制栏采用紧凑的“字幕”“语言”文字入口和弹出菜单，不改变现有播放器主体层级、动画和播放流程。

## 范围

- 检测视频文件中的全部内嵌音轨和字幕轨道。
- 识别并展示语种、配音类型、字幕类型、标题、编码与声道。
- 首次播放时按用户确认的中文优先级自动选择。
- 提供桌面和小窗口播放器一致的紧凑切换菜单。
- 保留外部字幕导入、附近字幕、字幕时间和字幕样式设置。
- 不修改用户视频文件，不向媒体文件写入轨道信息。

## 语言识别

新增独立的轨道语言识别与显示模型，综合 media-kit 轨道的 `language`、`title`、`codec`、声道字段判断。

中文标识兼容：

- `zh`、`zho`、`chi`、`zh-CN`、`zh-Hans`、`chs`
- 中文、国语、普通话、Mandarin
- 粤语、广东话、Cantonese、yue
- 台配、台湾国语
- 简体、简中、繁体、繁中、简繁、双语

无法识别时显示“未知语种”，并保留轨道编号及原始标题，确保所有轨道仍可选择。

## 默认选择

音轨优先级：

1. 国语
2. 粤语
3. 台配
4. 文件标记的默认音轨
5. 第一条有效音轨

中文字幕优先级：

1. 简体中文
2. 简繁双语
3. 繁体中文
4. 文件标记的默认字幕
5. 没有合适字幕时保持关闭

自动选择只在当前媒体首次获得稳定轨道列表时执行一次。用户手动选择后，后续轨道事件不得覆盖用户选择。切集或打开新媒体时重置选择状态并重新检测。

## 播放器交互

桌面播放器和小窗口播放器的控制栏均显示两个文字入口：

- `字幕`
- `语言`

### 语言菜单

点击“语言”显示紧凑弹出菜单。每项包含：

- 用户可理解的语种或配音类型，如“国语 / Mandarin”“粤语 / Cantonese”
- 原始轨道标题（存在时）
- 编码和声道，如 `TrueHD 5.1`、`FLAC 2.0`
- 当前轨道高亮与选中标记

点击立即调用 media-kit 切换音轨。失败时保持原音轨、显示简短提示并记录详细日志。

### 字幕菜单

点击“字幕”显示紧凑弹出菜单，依次包含：

- 关闭字幕
- 全部内嵌字幕轨道
- 外部字幕与字幕设置入口

内嵌字幕项显示语种、简繁类型、标题和编码，例如“简体中文 · PGS”“繁体中文 · ASS”。当前字幕高亮并显示选中标记。

## 字幕渲染策略

- 内嵌 PGS 必须使用 libmpv 原生渲染，避免位图字幕经过 Flutter 文本字幕通道而丢失。
- 内嵌 ASS/SSA 优先使用 libmpv 原生渲染，保留文件内样式。
- 外部 ASS/SRT/VTT 继续使用现有加载、导入、时间偏移和样式设置流程。
- 选择内嵌字幕时清除外部字幕状态并开启原生字幕可见性。
- 选择外部字幕时关闭当前内嵌字幕，避免两套字幕重叠。
- 选择“关闭字幕”时同时关闭内嵌与外部字幕状态。

## 状态与 TrueHD 兼容

播放器控制器维护强类型的可用音轨、可用内嵌字幕、当前选择和当前媒体自动选择状态。UI 只消费这些状态，不直接解析 media-kit 原始字符串。

现有 TrueHD 失败后切换兼容音轨的逻辑继续保留。若用户已手动选择音轨，则不自动覆盖；失败信息提示用户手动选择其他音轨或导出诊断日志。

## 日志与异常

- 记录检测到的轨道数量、自动选择结果和用户切换结果。
- 本地文件路径可保留；远程地址和凭据继续经过统一脱敏。
- 轨道切换失败不得导致播放器退出或重建视频解码器。
- 无轨道、字段缺失和未知编码均以可选择的回退标签显示。

## 测试与验收

自动测试覆盖：

- 中文语言代码、英文名称和常见缩写识别。
- 国语、粤语、台配音轨优先级。
- 简体、简繁双语、繁体字幕优先级。
- 未知语种与缺失标题的回退显示。
- 内嵌 PGS、ASS、SRT 轨道检测和切换。
- 自动选择只执行一次，用户手动选择不被覆盖。
- 切换媒体后清理旧轨道并重新选择。
- 桌面与小窗口控制栏均包含“字幕”和“语言”入口。
- 外部字幕功能保持可用，内外字幕不会重叠。

实机验收使用包含多语言 TrueHD 音轨和多条 PGS 字幕的蓝光文件，确认国语默认优先、简体字幕默认优先、手动切换立即生效、画面与声音持续播放且无无响应弹窗。
