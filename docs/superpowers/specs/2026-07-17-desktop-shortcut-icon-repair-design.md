# 桌面快捷方式图标修复设计

## 目标

修复 Windows 桌面“看影音”快捷方式在应用升级或 MSIX 包标识切换后显示空白图标的问题。已有快捷方式应自动更新为当前安装包的启动目标和图标；缺少快捷方式时仍由用户决定是否创建，不能静默新增。

## 根因

当前桌面快捷方式可能保存过期的 AUMID，并把图标固定到已经失效的 WindowsApps 版本目录。应用升级后快捷方式仍存在，但图标路径可能失效。

原生创建函数在发现同名 `.lnk` 已存在时直接返回成功，不检查目标、参数、AUMID 或图标。因此应用无法修复旧快捷方式。初始化页面又只在首次询问时调用创建函数，用户确认过以后不会再次刷新快捷方式。

## 修复行为

应用每次启动进入快捷方式检查流程时：

1. 先通过 Windows 原生通道判断桌面“看影音.lnk”是否存在。
2. 若存在，静默覆盖该快捷方式，使其指向当前包 AUMID，并把图标设置为当前运行中的可执行文件；不再次弹窗。
3. 若不存在，继续沿用现有首次询问逻辑。用户未同意时不创建快捷方式。
4. 快捷方式创建函数不再因文件已存在而提前返回，保存时允许覆盖旧文件。

MSIX 模式下快捷方式使用 `explorer.exe` 启动 `shell:AppsFolder\<当前包族名>!kanyingyin`，并设置 `PKEY_AppUserModel_ID`。便携模式继续直接指向当前可执行文件。两种模式都显式使用当前可执行文件作为图标来源。

## 身份与文案

- 快捷方式名称使用“看影音”。
- 描述使用“启动看影音”。
- 原生调用参数统一使用“看影音”和“本地视频媒体库与播放器”描述。
- 平台通道继续使用 `com.kanyingyin.player/shortcut`。

## 错误处理

- 快捷方式存在性检查失败时按“不存在”处理，保持现有用户询问流程。
- 自动修复已有快捷方式失败时不阻断应用启动，仅记录现有调试输出。
- 创建或覆盖失败时，用户主动创建流程继续显示“桌面快捷方式创建失败”。
- 不删除用户桌面上的其他文件或快捷方式。

## 测试

- 源码契约测试确认存在性检查方法和平台通道调用。
- 原生契约测试确认不再对已有 `.lnk` 提前返回。
- 原生契约测试确认 MSIX 使用当前 AUMID、`explorer.exe` 参数和当前 EXE 图标。
- 初始化流程测试确认已有快捷方式会自动修复，缺失快捷方式仍进入询问逻辑。
- 身份残留测试确认不再包含旧名称的十六进制调用参数或在线资源描述。
- 通过完整 Flutter 测试、静态分析和 Windows Release 构建。

## 版本与交付

本修复进入新安装包，版本更新为 `2.0.9+20009`，MSIX 清单版本为 `2.0.9.0`。同步更新 API 版本、README、更新弹窗、发布说明和版本历史。生成并签名 MSIX 后验证清单身份 `com.kanyingyin.player`，复制到桌面并命名为 `看影音-2.0.9.msix`。

交付完成后，使用当前已安装的正式包信息立即覆盖用户桌面现有“看影音.lnk”，并验证目标、AUMID、图标路径及图标文件均有效。
