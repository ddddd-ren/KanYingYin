# 删除历史记录与隐身模式设计

## 目标

从看影音中彻底删除观看历史、跨会话播放进度、断点续播和隐身模式。删除必须覆盖界面、路由、依赖注入、持久化模型、播放器写入与恢复链路、遗留数据和当前功能文案，同时不得影响当前播放会话中的进度控制，也不得删除用户原始视频文件。

## 范围

### 删除

- “我的”页面中的“历史记录”入口。
- 历史记录页面、卡片、控制器、路由和 Flutter Modular 注册。
- `History`、`Progress`、`HistorySourceType` 数据模型及 Hive 适配器。
- 历史仓库接口、实现和依赖注入。
- 播放器每秒写入观看记录、集数、来源和播放位置的逻辑。
- 本地与网盘视频从历史记录恢复播放位置的流程。
- “播放设置”中的“隐身模式”开关、设置键和历史写入判断。
- 与上述能力直接相关的测试、当前 README 功能宣称和当前用户文案。

### 保留

- 当前播放会话中的进度条、拖动跳转、快进快退、选集和自动连播。
- 网盘链接刷新或播放器内部重试期间保持当前会话播放位置的能力。
- 字幕时间偏移等与观看历史无关的按视频设置。
- 本地媒体索引、网盘媒体索引、海报、字幕缓存和用户原始视频文件。
- 历史版本说明中对旧版本行为的真实记录。

## 数据清理

应用升级或重新安装后不再打开 `histories` Hive box。初始化存储时检测并删除遗留的 `histories.hive` 与 `histories.lock`，该清理仅针对看影音专属应用数据目录。清理失败时记录脱敏日志但不阻止应用启动、本地扫描或播放。

代码中保留一个范围受限的旧数据清理函数，不再注册历史模型适配器，也不再暴露历史数据访问 API。清理操作幂等，文件不存在时直接完成。

## 播放链路

播放器定时器继续同步播放状态、当前位置、缓冲、音量和自动连播，但不再调用历史控制器。视频页面控制器不再暴露历史来源标识或跨会话恢复秒数。

本地播放会话不再携带 `resumePositionSeconds`。新建播放参数时默认从视频开头开始。网盘播放解析仍可在同一次播放会话内因链接过期而恢复当前位置，这属于故障恢复，不写入持久化存储。

## 界面与归属说明

“我的”页面删除历史记录卡片。“播放设置”删除隐身模式开关，其余控件顺序与交互保持不变。

README 的“开源来源与致谢”以及应用“关于”页面简单注明：“界面与操作参考 [Kazumi](https://github.com/Predidit/Kazumi)。”继续保留 GPL-3.0 许可证和其他第三方组件归属。

## 版本与发布

根据本次用户明确要求，应用版本保持 `1.0.0+10000`，MSIX 清单版本保持 `1.0.0.0`，GitHub 标签保持 `v1.0`，不进行版本递增。

实现完成并通过完整测试、静态分析、Windows Release 构建和 MSIX 验证后：

1. 删除私密 GitHub 仓库中现有的 `v1.0` Release 和远端标签。
2. 从最终工作树创建只有一个初始提交的新 `main` 历史。
3. 强制替换远端 `main`，远端不保留旧测试版本提交记录。
4. 创建指向新初始提交的 `v1.0` 标签。
5. 重新创建非草稿、非预发布的 `v1.0` Release，并上传已签名 MSIX。

远端最终只保留 `main` 分支、`v1.0` 标签和一个正式 Release。仓库必须保持私密。
