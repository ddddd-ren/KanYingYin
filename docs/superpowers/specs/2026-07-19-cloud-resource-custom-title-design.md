# 网盘资源自定义剧名设计

## 目标

在“网盘资源”页面允许用户修改资源在看影音中的显示剧名，并将该名称用于后续 TMDB 搜索。该功能只修改应用内元数据，不重命名、移动或删除网盘中的文件和文件夹。

## 用户行为

- 文件夹和媒体根目录中的独立视频菜单新增“修改剧名”。
- 对话框默认显示当前有效剧名；用户输入非空名称后保存。
- 自定义剧名始终优先于 TMDB 中文标题和原始文件名显示。
- 已设置自定义剧名时，对话框提供“恢复 TMDB 标题”。清除后：已有 TMDB 匹配则显示 TMDB 标题，否则显示原文件或文件夹名。
- 自动刮削、手动刮削和重新匹配都使用自定义剧名作为查询词。
- 修改显示剧名不触发远程重命名，不改变远程 ID、路径、字幕关联或播放链接。

## 数据模型

在 `CloudResourceTmdbRecord` 增加可空的 `customTitle` 字段，并提供：

- `effectiveTitle`：依次返回非空的 `customTitle`、TMDB `title`、`displayName`。
- `withCustomTitle(String)`：保存去除首尾空白后的自定义剧名。
- `clearCustomTitle()`：只清除自定义剧名，保留 TMDB 匹配信息。

`customTitle` 参与 JSON 往返、值相等和哈希计算。未进行 TMDB 匹配的资源也能以 `unchecked` 记录保存自定义剧名。记录继续使用现有稳定键 `sourceId + remoteId + normalizedRemotePath`，不存储凭据、播放地址或其他秘密。

## 服务与数据流

1. 页面通过控制器读取资源记录和 `effectiveTitle`。
2. 用户保存名称时，控制器调用协调器；协调器将现有记录更新后写入 `CloudResourceTmdbRepository`。没有记录时创建 `unchecked` 记录。
3. 协调器加载目录并生成 `CloudResourceTmdbTarget` 时，将缓存中的自定义剧名作为可选查询名传入。
4. `CloudResourceTmdbService` 优先使用目标的自定义查询名；没有时继续使用原文件或文件夹名清洗后的查询名。
5. TMDB 匹配、未匹配或失败状态覆盖时必须保留原有 `customTitle`。
6. 删除来源继续通过现有资源 TMDB 仓库事务清理，自定义剧名随记录一起删除和回滚。

## 界面

- 资源卡片仍显示原文件名作为辅助信息，便于识别真实网盘内容。
- 菜单顺序为“修改剧名”“TMDB 刮削”“重新匹配”。
- 修改对话框包含单行输入框、“取消”“保存”；存在自定义剧名时额外显示“恢复 TMDB 标题”。
- 保存空白输入时不写入，并提示“剧名不能为空”。
- 保存或恢复后立即刷新当前卡片，不重新加载远程目录。

## 故障与安全

- 存储失败时保留原显示状态并提示修改失败。
- TMDB 无 API Key、断网或请求失败不影响自定义剧名的保存和显示。
- 任何代码路径都不得调用远程重命名、移动或删除接口。
- 删除来源失败时沿用现有记录快照回滚，自定义剧名随记录恢复。

## 测试与验收

- 模型测试：`customTitle` JSON 往返、有效标题优先级、清除后恢复 TMDB 标题。
- 仓库/协调器测试：无 TMDB 记录也可保存；匹配、未匹配和失败更新不丢失自定义剧名。
- 服务测试：自定义剧名经过清洗后成为 TMDB 查询词，远程路径保持不变。
- 页面测试：菜单、保存、即时显示、原文件名辅助显示和“恢复 TMDB 标题”。
- 删除测试：来源删除和失败回滚覆盖含自定义剧名的记录。
