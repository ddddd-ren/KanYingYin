# 网盘资源主导航与目录浏览设计

## 目标

在应用左侧主导航的“本地”上方新增“网盘资源”入口。用户可在独立页面中浏览已启用的 OpenList 和夸克网盘来源，逐级进入文件夹并直接播放视频。

本地页继续专注本地目录与本地媒体库，不将网盘目录状态堆入 `LocalPage` 或 `LocalController`。

## 导航与路由

主导航顺序为：

1. 网盘资源
2. 本地
3. 我的

新增 `/tab/cloud/` 路由及独立 `CloudResourcesModule`。默认启动页仍为 `/tab/local/`，现有用户的启动页设置继续有效。横向窗口使用左侧 `NavigationRail`，竖向窗口同步增加底部导航项。

导航图标使用系统 Material 网盘/云端图标，保持现有选中态、动画时长、曲线和间距。

## 页面结构

`CloudResourcesPage` 使用现有 Material 风格，包含：

- 来源选择：显示所有已启用的 OpenList 和夸克来源，标明提供商。
- 来源操作：添加/管理网盘来源、刷新当前目录、移除当前来源。
- 目录导航：返回上级、面包屑路径和当前目录搜索。
- 内容区：文件夹优先、视频其后；字幕和其他不支持文件不显示为视频卡片。

没有可用来源时，显示空状态和“添加 OpenList”、“添加夸克网盘”入口。

## 多根目录与目录状态

页面控制器使用 `CloudRemoteRef` 保存当前目录的 ID 和展示路径。

- 来源只有一个媒体根目录时，选中来源后直接进入该目录。
- 来源有多个媒体根目录时，先显示仅由已配置根目录组成的虚拟根页。
- 进入子目录时把上一个 `CloudRemoteRef` 压入导航栈；切换来源时清空该栈。
- 目录状态仅保留在当前页面会话内，重启后回到配置根目录，不需要新增持久化字段。

OpenList 继续主要使用 `path`，夸克继续主要使用 `id`。页面不依赖路径到文件 ID 的隐式内存映射。

## 数据流与架构边界

新增专用的 `CloudResourcesController`，负责：

- 从 `CloudSourceRepository` 读取已启用来源。
- 通过 `CloudProviderRegistry` 创建对应提供商客户端。
- 载入根目录、子目录和视频文件，并管理加载、错误、搜索与导航栈状态。
- 在切换来源或页面销毁时关闭正在使用的客户端。

页面不复制 OpenList 或夸克专属请求逻辑，也不读取明文凭据。凭据由已有 `SecureCloudCredentialStore` 和提供商客户端即时使用。

## 播放

点击视频时，使用条目当前的 `id` 与 `remotePath` 构造 `CloudPlaybackTarget`，通过现有 `CloudPlaybackResolver` 解析临时地址，再交给 `LocalVideoController` 和现有 media-kit 播放器。

播放请求继续使用现有安全请求头、远程字幕、链接过期单次刷新与播放会话竞态保护。本次不新增视频下载、缓存、上传或文件管理能力。

## 移除与安全

“移除来源”必须显示确认对话框，并明确说明只会删除应用内来源配置、安全凭据、对应索引和缓存。实现复用 `CloudLibraryController.delete()`，不向远程客户端增加删除文件请求。

Cookie、用户名、密码、临时播放地址和请求头不得进入页面状态的可序列化输出、日志或普通存储。

## 错误处理

- 单个来源无凭据、离线或协议不兼容时，页面显示提供商映射后的明确错误与重试操作。
- 错误不会清空其他来源，也不影响本地媒体库和本地播放。
- 切换来源或连续进入目录时使用请求代次丢弃旧请求结果，避免慢响应覆盖新页面。
- 播放解析失败使用现有错误提示，不将完整 URL 或请求头显示给用户。

## 测试与验收

按测试驱动开发，至少覆盖：

- 主导航顺序为“网盘资源、本地、我的”，默认启动页仍为本地。
- 页面只列出已启用的 OpenList 和夸克来源。
- 单根目录直接进入，多根目录显示虚拟根页。
- 目录加载、搜索、返回上级、刷新和乱序请求保护。
- OpenList 按路径、夸克按文件 ID 浏览，且两者都使用注册器创建客户端。
- 点击视频构造完整 `CloudPlaybackTarget` 并调用现有播放流程。
- 移除来源只清理应用内数据，不调用远程删除。
- 网盘错误不影响本地媒体库。
