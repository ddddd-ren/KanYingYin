# 网盘资源 TMDB 刮削设计

## 目标

为“网盘资源”目录浏览页增加 TMDB 自动刮削和手动重新匹配。文件夹卡片显示海报、中文标题、评分和匹配状态；媒体根目录中的独立视频也可刮削。进入已匹配文件夹后，页面顶部展示系列海报、标题和简介，单集视频沿用文件夹系列信息，不重复请求 TMDB。

无 TMDB Key、断网、TMDB 限流或匹配失败时，网盘目录浏览和播放必须继续可用。

## 方案

采用混合元数据层：

1. 网盘浏览页使用独立的强类型 `CloudResourceTmdbRepository`，支持尚未扫描入媒体库的文件夹和独立视频。
2. 现有 `CloudMediaIndexRepository` 仍是媒体库 TMDB 信息的权威来源。
3. 浏览页先尝试从现有云媒体索引复用已匹配元数据；没有可复用数据时才查询独立缓存或请求 TMDB。
4. 手动确认匹配后，若能根据来源、规范化路径和系列名唯一定位云媒体索引条目，同步中文标题、简介、评分、海报和背景图；无法唯一确认时不猜测更新。

## 领域模型与持久化

新增 `CloudResourceTmdbRecord`，包含：

- `sourceId`
- `remoteId`
- `remotePath`
- `displayName`
- `resourceKind`（文件夹或独立视频）
- `status`（未检查、已匹配、未匹配、失败）
- TMDB ID、媒体类型、中文标题、原始标题、简介、评分、海报 URL、背景图 URL
- `posterCachePath`
- `checkedAt`

稳定键为 `sourceId + remoteId + normalizedRemotePath`。当提供商的文件 ID 为空时，规范化路径仍可唯一读取 OpenList 记录。

`CloudResourceTmdbRepository` 使用 Hive 普通配置箱中的独立键存储 JSON 数组，并为测试提供内存存储。记录不得保存 Cookie、用户名、密码、播放 URL、分享令牌或请求头。

来源删除流程在现有凭据、媒体索引、海报和字幕缓存清理基础上，增加该来源的浏览页 TMDB 记录清理，不执行任何远程删除。

## 匹配与查询

新增 `CloudResourceTmdbService`，复用现有 `ITmdbClient`、`TmdbMatcher`、`TmdbScrapeOptions`、`CloudPosterCache` 和 TMDB 图片 URL 规则。

查询名处理：

- 文件夹使用文件夹展示名，移除首尾空白、常见分辨率、编码、完结集数和无意义括号标记，保留年份用于匹配。
- 媒体根目录中的独立视频使用去扩展名后的文件名，复用现有集数解析避免把剧集单集当成独立影片。
- 自动类型先根据名称、年份和子条目特征判断电影或剧集；手动流程允许用户通过现有刮削选项强制类型。

自动匹配使用现有最低分数和候选领先幅度，不降低阈值以强行填充海报。结构不确定时保持“未匹配”，交给手动候选选择。

## 自动刮削调度

`CloudResourcesController` 在目录条目加载成功后，将当前可刮削资源交给 `CloudResourceTmdbCoordinator`：

- 文件夹始终是可刮削资源。
- 只有当前目录是已配置媒体根目录时，才自动刮削其中的独立视频。
- 进入普通子文件夹后，页面读取该文件夹的元数据用于顶部信息，不对单集视频重复发起搜索。
- 单来源自动请求最多 2 个并发，切换来源或目录时丢弃旧代次的界面结果。
- 已匹配且名称未变的记录不重复查询。
- “未匹配”作为负缓存保留 7 天；期满后可自动重试。
- “失败”不作为长期负缓存，用户刷新目录或手动刮削时可立即重试。

自动调度前先检查 TMDB Key。Key 缺失时不发起网络请求，不显示阻断式错误，只在用户主动点击刮削时提示前往 TMDB 设置。

## 手动刮削

页面工具栏增加“刮削当前目录”，对当前可刮削资源执行带进度的批量匹配。

文件夹与独立视频卡片操作菜单包含：

- `TMDB 刮削`：使用当前刮削选项尝试自动匹配，不确定时展示候选列表。
- `重新匹配`：忽略已有正缓存和负缓存，直接显示搜索候选。

手动流程复用 `TmdbScrapeOptionsSheet` 和 `TmdbMatchSheet`。用户选择候选后请求详情、缓存海报、保存记录，并按可确认映射规则同步云媒体索引。

## 界面

文件夹和独立视频卡片的元数据展示优先级：

1. 已缓存海报文件。
2. TMDB 网络海报。
3. 现有文件夹或视频图标占位。

卡片显示 TMDB 中文标题；原文件名作为次要文本保留。评分大于 0 时显示一位小数和星标。匹配进行中显示小型进度指示，未匹配不用错误色阻断目录操作。

进入已匹配文件夹后，搜索框上方显示紧凑系列头部：海报、中文标题、评分和最多三行简介。保持现有 Material 样式、导航结构、间距、动画时长和动画曲线。

## 错误与安全边界

- TMDB 错误不写入 `CloudResourcesController.errorMessage`，不替换网盘目录错误，避免误导用户认为目录无法访问。
- 自动刮削失败仅更新对应资源状态；手动刮削失败使用 `SnackBar` 显示可理解错误。
- TMDB 请求不携带网盘 Cookie、OpenList 令牌或播放请求头。
- 日志不记录 TMDB API Key、完整请求头或网盘播放 URL。
- 删除 TMDB 浏览缓存、海报缓存或网盘来源都不得删除远程文件。

## 测试与验收

按测试驱动开发，至少覆盖：

- `CloudResourceTmdbRecord` JSON 往返、稳定键和敏感信息排除。
- 存储的按来源查询、更新、删除和并发不丢数据。
- 查询名清理、电影/剧集类型选择、自动匹配、候选和手动选择。
- 已匹配正缓存不重试，未匹配负缓存 7 天内不重试，失败可重试。
- 自动刮削并发不超过 2，切换目录后不提交旧界面结果。
- 文件夹自动刮削，根目录独立视频自动刮削，子文件夹单集不重复刮削。
- TMDB Key 缺失不发请求，TMDB 错误不影响目录浏览和播放。
- 卡片海报、中文标题、评分、原名和匹配进度展示。
- 手动刮削、重新匹配、刮削选项和候选选择。
- 可确认映射时同步云媒体索引，映射不唯一时不错更。
- 来源删除清理浏览页 TMDB 记录，不调用远程删除。
