# 网盘刮削沉浸式海报卡片设计

## 背景

“网盘资源”当前将海报与信息分成上下两块：上方显示海报，下方固定显示标题、评分、大小和日期。用户希望 TMDB 刮削后的电影与剧集像本地媒体库参考图一样，以海报铺满卡片，并在底部渐变层中显示标题、媒体信息和状态标签。

本设计只改变卡片表现与必要的展示数据，不改变 TMDB 匹配流程、网盘目录结构、播放解析或远程文件。

## 目标

- 已刮削电影和剧集目录使用沉浸式大海报卡片。
- 独立视频即使尚未刮削，也使用相同尺寸的媒体占位卡，保证网格整齐。
- 没有 TMDB 匹配的普通目录继续使用文件夹卡片，保持目录导航辨识度。
- 海报、标题、真实文件名、评分、年份、类型、大小和状态在一张卡片内清晰呈现。
- 网盘媒体卡与本地媒体卡复用同一套渐变、文字和标签视觉原件。
- 海报、TMDB 或字幕状态不可用时，目录浏览和播放继续正常工作。

## 非目标

- 不修改网盘中的文件名、目录名或路径。
- 不移动、删除或重新整理网盘文件。
- 不改变夸克专用播放接口、字幕下载或播放列表逻辑。
- 不重新设计 TMDB 匹配弹窗或批量刮削流程。
- 不把没有 TMDB 匹配的普通目录猜测成电影或剧集。
- 不为卡片增加悬停才能访问的关键操作。

## 已选方案

采用“方案 A：底部渐变叠层”。

- 海报铺满卡片。
- 底部渐变从卡片中段开始，逐步过渡到接近不透明的黑色。
- 文字和状态标签始终可见，不依赖鼠标悬停。
- 鼠标悬停只增加轻微高亮，不改变信息内容。
- 右上角三点菜单始终可用。

相较固定毛玻璃信息栏，该方案遮挡海报更少；相较悬停详情方案，信息发现性和键盘可访问性更好。

## 卡片分类

### 沉浸式媒体卡

以下资源使用沉浸式媒体卡：

- 所有独立视频文件。
- `CloudResourceTmdbRecord.status == matched` 的电影或剧集目录。

独立视频没有 TMDB 海报时使用媒体渐变占位图。目录只有在已有明确 TMDB 匹配时才成为媒体卡，不根据目录名、大小或层级猜测。

### 普通目录卡

没有匹配 TMDB 的目录继续显示当前文件夹图标、目录名称和菜单。它与媒体卡使用相同网格单元尺寸，但保持现有目录语义和点击进入行为。

`unmatched`、`failed` 或 `unchecked` 的目录仍属于普通目录卡。

## 信息层级

### 主标题

优先级保持现有有效标题规则：

1. 用户自定义剧名。
2. TMDB 简体中文标题。
3. 真实网盘文件名或目录名。

主标题最多两行，超出部分省略。文字使用白色、中等偏粗字重和轻微阴影。

### 副标题

副标题显示真实网盘文件名或目录名，最多一行。它与主标题完全相同时不重复显示。副标题用于提醒用户看影音中的展示名称不会改变远程文件。

### 媒体信息

- 已匹配资源显示 TMDB 评分、电影/电视剧类型和上映或首播年份。
- 独立视频额外显示文件大小。
- 剧集目录不显示虚假的目录大小。
- 缺少评分、年份或媒体类型时直接省略对应片段。
- 信息使用 ` · ` 分隔，最多两行，超出部分省略。

### 状态标签

- 当前正在刮削：`刮削中`，带小型进度指示。
- 已匹配：`已刮削`。
- 独立视频被确认无匹配：`未匹配`。
- 独立视频尚未处理：`未刮削`。
- 最近一次请求失败：`刮削失败`，但菜单仍允许重试。
- 同目录存在同名字幕的独立视频：`有字幕`。

目录卡不显示无法确认的字幕状态。已匹配剧集目录本版本也不递归扫描子目录来推断字幕；只有现有索引能够明确提供状态时才允许显示“有字幕”。不显示“无字幕”，避免把未知状态误报为确定结果。

## 共享表现组件

新增职责单一的 `ImmersiveMediaCard` 表现组件，放在媒体库共享表现层。组件只接收已整理好的展示数据和回调，不读取仓库、网盘客户端或 TMDB 服务。

组件输入包括：

- 封面 Widget 或封面构建器。
- 主标题、副标题、媒体信息。
- 状态标签列表。
- 是否加载、是否悬停、叠层显示模式。
- 点击、右键、长按和操作菜单回调。

叠层模式至少包含：

- `hover`：供本地媒体库继续使用，保持现有交互。
- `always`：供网盘媒体卡使用，信息始终可见。

本地媒体库只把现有私有卡片的渐变、文字和标签部分迁移到共享组件；封面回退顺序、Hero、点击、右键、动画时长和悬停行为保持不变。网盘页面使用 `always` 模式，并保留现有三点菜单。

## 网盘卡片展示数据

新增只读的 `CloudResourceCardViewData` 或等价私有强类型对象，将 `CloudFileEntry`、`CloudResourceTmdbRecord`、刮削状态和字幕状态转换为展示字段。

它负责：

- 判断资源使用媒体卡还是目录卡。
- 计算有效标题和真实名称副标题。
- 组合评分、类型、年份和文件大小。
- 生成刮削与字幕标签。
- 提供本地海报路径、TMDB 网络海报地址和占位类型。

Widget 不直接拼装领域规则，便于对分类和文案做单元测试。

## TMDB 日期字段

`CloudResourceTmdbRecord` 增加可空 `releaseDate` 字段，用于电影上映年份或电视剧首播年份。

- `matched` 工厂从 `TmdbMetadata.releaseDate` 写入。
- `fromJson`、`toJson`、相等和哈希包含该字段。
- 旧记录缺少字段时读取为 `null`。
- 保存或清除自定义标题时必须保留已有匹配记录中的日期。现有 `asFailed` 会清除整组匹配元数据，本次继续沿用该语义，因此不单独保留日期。
- 媒体索引与 TMDB 搜索请求不因该字段改变。

年份只在字符串至少包含四位有效开头年份时显示；非法或缺失值直接省略。

## 海报与降级

海报顺序保持：

1. 存在且可读取的本地海报缓存。
2. TMDB 网络海报。
3. 媒体渐变占位图。

海报使用 `BoxFit.cover` 填满网盘媒体卡。缓存文件加载失败后尝试网络海报；网络图片失败后显示占位图。图片失败不修改 TMDB 匹配状态，不显示阻断式错误，也不影响点击播放或进入目录。

普通目录卡继续使用文件夹渐变与文件夹图标，不使用媒体海报占位。

## 字幕状态数据流

页面继续复用现有同名字幕匹配规则：同一远程目录、去扩展名后的主文件名一致、字幕扩展名受支持。

1. `CloudResourcesPage` 已持有当前完整目录条目。
2. 页面或 Controller 从完整条目生成有同名字幕的视频稳定键集合。
3. `CloudResourcesGrid` 接收该只读集合。
4. 独立视频卡根据集合显示“有字幕”。

不得为了显示标签额外下载字幕、访问播放地址或递归请求网盘目录。

## 网格与响应式布局

桌面网格以本地媒体库的大卡片密度为基准：

- 内容宽度小于 650：2 列。
- 650 至 999：3 列。
- 1000 及以上：4 列。
- 卡片宽高比约 `0.68`。
- 横纵间距为 12 像素，外边距为 12 像素。
- 可根据实际卡片宽度计算高度，并限制在 320 至 680 像素之间。

媒体卡和目录卡必须在同一行对齐。调整窗口时只改变列数和卡片尺寸，不丢失当前目录、搜索词或刮削状态。

## 交互

- 独立视频左键：继续调用现有播放流程。
- 目录左键：继续进入目录。
- 三点菜单：继续提供修改剧名、TMDB 刮削和重新匹配。
- 右键或长按：若现有网盘卡没有独立行为，本版本不新增不同于三点菜单的命令；共享组件必须保留未来接入能力。
- 键盘焦点：卡片和三点菜单均可聚焦，Enter/Space 触发主操作，菜单键打开操作菜单。
- 刮削中：仅遮罩当前卡片，不阻止其他资源播放、进入或刮削。

共享组件的悬停动画沿用本地媒体库现有 160ms `easeOut`；网盘卡叠层始终显示，悬停只调整表面高亮或阴影。

## 刷新与状态更新

`CloudResourcesGrid` 继续由页面传入 `records` 和 `scrapingKeys`。Coordinator 完成刮削、重新匹配或自定义标题保存后通知 Controller，页面重建当前卡片：

- 普通目录卡可原位切换为已刮削媒体卡。
- 海报、标题、评分、年份和标签一起更新。
- 不重新加载远程目录。
- 不重新扫描来源。
- 不改变滚动位置、当前路径或搜索条件。

卡片稳定键继续使用来源 ID、远程 ID 和规范化远程路径，避免重建后状态错位。

## 错误与安全边界

- 海报异常只降级图片，不影响 TMDB 记录、播放或目录导航。
- 缺失展示字段只省略对应文本，不显示 `null`、空分隔符或虚假值。
- 旧记录缺少 `releaseDate` 时继续显示其他元数据。
- 字幕无法确认时不显示结论。
- 刮削失败只更新当前卡片状态，其他卡片继续可操作。
- 卡片层不得持有或记录 TMDB API Key、夸克 Cookie、播放直链或请求头。
- 所有写操作仍只面向看影音元数据与缓存，不调用网盘重命名、移动或删除接口。

- 自定义标题相关转换不丢失日期。

- 五种刮削状态标签。
- 同名字幕显示“有字幕”，未知状态不显示“无字幕”。

### Widget 测试

- 海报填满、底部渐变、主副标题、媒体信息和标签。
- 本地缓存、网络海报和占位图降级顺序。

- 刮削中遮罩只覆盖目标卡片。
- 普通目录保持文件夹卡表现。
- 点击、进入目录、三点菜单与键盘操作。
- 2、3、4 列响应式布局和 `0.68` 卡片比例。

### 回归测试

## 验收标准

- 已刮削电影和剧集目录显示与参考图一致的全海报渐变卡片。
- TMDB 中文标题为主，真实网盘名称为副，自定义标题继续拥有最高优先级。
- 评分、年份、类型、大小和可确认状态正确显示。
- 普通目录仍明显是目录，不被误画成影视海报卡。
- 海报或 TMDB 不可用时仍能浏览目录和播放视频。
- 刮削成功后卡片原位更新，不重新扫描网盘。
