# 网盘资源 TMDB 渐进式匹配设计

## 背景

“网盘资源”已经支持 TMDB 自动刮削、重新匹配和候选选择，但单项操作仍依赖从资源名称直接生成查询词。名称包含季集编号、分辨率、片源、编码或字幕组时，用户看不到系统识别了什么，也不能在请求前修正搜索条件。

本次参考 [Pan-Fetcher](https://github.com/mguyenanastacio-glitch/pan-fetcher) 的媒体名称结构化解析与候选信息展示思路，渐进式增强现有网盘 TMDB 刮削。继续保留看影音现有的严格评分、类型校验和候选领先幅度，不采用低相似度时直接选择第一条结果的策略。

## 目标

- 单项“TMDB 刮削”和“重新匹配”在请求前展示可编辑搜索词与结构化识别结果。
- 用户可修改搜索词、媒体类型、年份和本次刮削选项。
- 搜索结果展示足够的辨别信息，并按看影音的严格匹配规则排序。
- 单项操作必须由用户确认候选后才保存；批量“刮削当前目录”继续自动执行。
- 无唯一结果时保持待确认，不猜测写入错误元数据。
- 只更新看影音中的元数据，不重命名、移动、删除或修改网盘文件。
- TMDB 不可用时，夸克网盘浏览和播放继续可用。

## 范围

本次覆盖“网盘资源”页面中文件夹和可刮削视频的单项操作，以及该页面“刮削当前目录”的匹配内核。实现位于共享的云资源抽象层，因此 OpenList 可复用相同行为，但夸克网盘是主要验收对象。

本次不包含：

- 修改夸克网盘中的文件名或目录名。
- 移动、删除或重新组织网盘文件。
- 改造本地媒体库的单项刮削界面。
- 自动采纳低置信度候选。
- 为名称解析器建立可配置规则语言或插件系统。

## 用户流程

### 单项刮削与重新匹配

1. 用户从资源卡片菜单点击“TMDB 刮削”或“重新匹配”。
2. 系统解析资源名称，生成原始名称、默认搜索词、媒体类型、年份、季号和集号。
3. 居中弹窗展示解析结果，允许用户修改搜索词、类型、年份及本次刮削选项。
4. 用户点击“搜索 TMDB”。
5. 系统查询、评分并展示候选列表。
6. 用户选择候选后，“应用匹配”按钮可用。
7. 用户确认后获取详情、缓存图片、保存元数据并同步云媒体索引。
8. 弹窗关闭，页面显示新的应用内标题、海报和简介；网盘原文件保持不变。

“TMDB 刮削”和“重新匹配”共用同一弹窗。前者从当前资源名称或自定义剧名生成草稿；后者优先以当前有效标题生成草稿，并显示现有匹配作为参考。两者都不会在弹窗打开或搜索阶段修改数据。

### 批量刮削

“刮削当前目录”不逐项弹窗。它使用相同名称解析器和严格匹配器：只有唯一候选满足标题、年份、媒体类型、最低分数和领先幅度时才自动保存；其他项目归入待确认或无结果，继续处理后续资源。

## 界面设计

采用居中双栏弹窗，并保持当前 Material 主题、间距和交互风格。

### 顶部

- 标题根据入口显示“TMDB 刮削”或“重新匹配 TMDB”。
- 次要文本显示当前网盘原始文件名或目录名。
- 明确提示“仅更新看影音中的资料，不会修改网盘文件”。

### 左栏：搜索准备

- 可编辑的搜索词，默认选中便于直接替换。
- 媒体类型：自动、电影、电视剧。
- 可空年份；只接受四位有效年份。
- 只读识别标签：季号、集号。未识别时不显示对应标签。
- 可折叠的本次刮削选项，复用 `TmdbScrapeOptions` 中已有配置。
- “搜索 TMDB”按钮；输入框按 `Enter` 也可搜索。

### 右栏：候选结果

- 未搜索时展示简短说明。
- 搜索中展示骨架或进度状态，并禁用重复提交。
- 候选卡片展示海报、简体中文标题、原始标题、年份、媒体类型和评分。
- 展示标题、年份和类型的匹配提示；最高分且满足阈值的候选标注“推荐”，但不自动应用。
- 选中候选后才启用“应用匹配”。
- 无结果或请求失败时在右栏原位显示说明、重试入口和修改搜索条件的建议。

弹窗使用受窗口尺寸约束的最大宽度。可用宽度不足时，左右栏改为上下排列，不压缩候选海报和文字。`Esc` 关闭弹窗；关闭和取消均不写入数据。

## 名称解析

新增职责单一的云媒体名称解析器。输入为原始资源名、资源类型和可选自定义标题，输出不可变的 `TmdbMatchDraft`：

- `originalName`
- `searchTitle`
- `mediaTypeMode`
- `year`
- `seasonNumber`
- `episodeNumber`

解析器负责：

- 文件先去除扩展名，目录保留完整名称。
- 识别 `S01E01`、`S1E1` 及已有项目支持的等价季集形式。
- 识别独立的四位年份或括号年份。
- 清理分辨率、片源、视频编码、音频编码、HDR、完结集数和常见发布标签。
- 清理中英文方括号标签及多余分隔符，但不得删除可能属于正式标题的普通文字。
- 自定义剧名存在时优先作为搜索标题，同时仍从原名称识别季集结构。
- 无法可靠识别的字段返回空值，不根据文件大小或目录层级猜测内容身份。

分辨率、片源、编码和字幕组只用于清理搜索词，本版本不持久化为媒体元数据。这样吸收 Pan-Fetcher 的解析思路，同时避免扩张为完整媒体技术信息系统。

## 领域模型

### `TmdbMatchDraft`

表示弹窗打开时的结构化识别结果。它只存在于当前交互中，不写入仓库。用户编辑后生成搜索请求，原始名称和识别出的季集信息保持只读。

### `CloudResourceTmdbSearchRequest`

表示一次明确的云资源搜索：

- 非空搜索标题。
- 媒体类型模式。
- 可空年份。
- 语言。
- 本次 `TmdbScrapeOptions`。

搜索请求与 `CloudResourceTmdbTarget` 分离，避免为临时输入修改资源身份、稳定键、远程路径或自定义剧名。

### `TmdbRankedCandidate`

包装现有 `TmdbMetadata`，并携带：

- 综合匹配分数。
- 标题是否匹配。
- 年份是否匹配或接近。
- 媒体类型是否匹配。
- 是否满足自动匹配阈值。

`TmdbMatcher` 扩展为能够返回稳定排序的候选列表；现有 `choose` 行为继续由同一评分实现驱动，避免手动列表与批量自动匹配使用两套标准。

### `CloudResourceTmdbSelectionOutcome`

表示用户确认候选后的保存结果，包含已保存的资源记录、海报是否已缓存、媒体索引是否已同步。资源记录写入成功但图片或索引步骤失败时，调用方能够显示准确的部分成功提示，不把已经保存的选择误报为完全失败。

## 搜索、评分与缓存

- 用户指定电影或电视剧时只查询对应 TMDB 搜索端点。
- 类型为“自动”时同时查询电影和电视剧，合并后统一评分；不得因第一个类型返回非空结果就跳过另一类型。
- 标题、年份和媒体类型继续使用现有权重、最低分数和候选领先幅度。
- 排序首先按综合分数降序，再保持 TMDB 原始顺序，保证同分结果稳定。
- 单项搜索始终展示候选并等待确认，即使存在唯一高置信度结果。
- 批量搜索只有 `shouldAutoMatch` 为真时保存；其余结果不写入错误匹配。

搜索候选使用服务实例内的 LRU 内存缓存：

- 缓存键为规范化搜索词、媒体类型模式和语言。
- 最多保存 50 个查询。
- 有效期 10 分钟。
- API Key 变化时由现有服务重建逻辑自然清空。
- 缓存只保存候选搜索结果，不保存详情响应、凭据或网盘信息。
- 缓存命中后仍使用当前年份和匹配选项重新评分。

## 调用链与组件边界

1. `CloudResourcesPage` 请求 `CloudResourcesController` 为资源创建匹配草稿。
2. Controller 将资源和已有 `CloudResourceTmdbRecord` 交给名称解析器。
3. 用户提交搜索后，Controller 将 `CloudResourceTmdbSearchRequest` 传给 `CloudResourceTmdbCoordinator`。
4. Coordinator 继续负责 API Key 校验、进行中状态、服务生命周期和通知。
5. `CloudResourceTmdbService` 查询一个或两个 TMDB 类型、使用缓存、调用扩展后的 `TmdbMatcher` 评分，并返回排序候选。
6. 页面只负责显示候选和收集用户选择，不直接写仓库。
7. 用户确认后，现有 `select` 路径获取 TMDB 详情、解析图片、保存 `CloudResourceTmdbRecord` 并同步 `CloudMediaIndexRepository`，返回 `CloudResourceTmdbSelectionOutcome`。
8. `customTitle` 继续拥有最高显示优先级；重新匹配不得丢失自定义标题。

批量入口直接从草稿生成搜索请求并调用自动匹配路径，不创建 Widget，也不等待用户选择。

## 并发与状态

- 弹窗为每次搜索递增请求代次，只接受最后一次请求的结果。
- 用户修改搜索条件并重新提交时，旧请求即使稍后完成也不得覆盖新候选。
- 搜索期间禁止重复提交，但允许关闭弹窗。
- 弹窗关闭后异步结果不得调用已经销毁的界面状态。
- 选择保存期间锁定候选与确认按钮，防止重复写入。
- 现有批量最大并发数保持为 2。

## 错误处理与恢复

- 名称无法完整识别：保留去除扩展名后的原名称作为搜索词，未知字段为空，允许用户修改。
- 搜索词为空或年份格式无效：在字段旁显示验证错误，不发送请求。
- 未配置 TMDB API Key：在弹窗内提示前往设置，不影响网盘浏览和播放。
- 断网、超时、鉴权失败或限流：保留输入和旧元数据，在结果区显示可重试错误。
- 搜索无结果：显示空状态；单项手动搜索不写入“未匹配”记录。
- 详情请求失败：不替换现有匹配。
- 海报缓存失败：保存文字元数据和 TMDB 图片地址，向用户报告图片缓存未完成，不因本地图片失败放弃整个匹配。
- 资源记录保存成功但媒体索引同步失败：资源记录作为事实来源保留并返回部分成功。Coordinator 在下次加载包含该资源的目录时，使用已保存记录幂等地重试一次索引同步；同步失败不阻断目录加载，也不重复请求 TMDB。
- 批量单项失败：记录失败并继续；最终分别汇报成功、待确认、无结果和失败数量。

日志可以记录错误类型、资源稳定标识和处理阶段，但不得记录 TMDB API Key、夸克 Cookie、完整请求头或播放直链。

## 数据与安全边界

- 搜索草稿、请求代次和候选缓存仅驻留内存。
- 持久化继续使用现有 `CloudResourceTmdbRecord` 和云媒体索引，不为临时搜索条件新增长期存储。
- 所有写操作仅面向看影音的元数据仓库、媒体索引和海报缓存。
- 不调用任何网盘提供商的重命名、移动或删除接口。
- 远程 ID、路径、字幕匹配关系和播放解析流程保持不变。
- 清理来源或缓存仍不得删除用户网盘中的原始文件。

## 测试策略

按测试驱动开发，至少覆盖以下内容。

### 名称解析单元测试

- 中英文电影名、电视剧名和括号年份。
- `S01E01`、季目录、只有单集视频的季目录。
- 分辨率、片源、编码、HDR、字幕组和多种分隔符。
- 自定义剧名优先，但季集编号继续从原名称识别。
- 无法识别或包含疑似正式标题词时不进行激进清理。

### 匹配器与服务测试

- 标题完全匹配、包含匹配和不匹配。
- 年份相同、相差一年、缺失和明显冲突。
- 媒体类型正确、冲突和自动双类型候选。
- 同分候选稳定排序、领先分不足和阈值边界。
- 指定搜索词不改变资源稳定键和远程路径。
- 缓存键、10 分钟有效期、50 项淘汰和缓存命中后重新评分。
- 无结果、详情失败、海报缓存失败和索引同步失败。
- 选择结果能区分完全成功、图片缓存未完成和索引待同步；重新加载目录会重试待同步索引。

### Widget 测试

- 单项两个入口都打开准备弹窗。
- 默认字段和结构化识别标签正确。
- 搜索词、类型、年份和刮削选项可修改并传入服务。
- 加载、空结果、失败重试和最后一次请求获胜。
- 候选信息、推荐标记、选择和应用按钮状态。
- 取消、`Esc`、`Enter` 和保存期间防重复提交。
- 窄窗口下双栏变为上下排列。
- 确认后刷新应用内元数据，原网盘名称仍可见。

### 批量与回归测试

- “刮削当前目录”不创建单项弹窗。
- 唯一高置信度结果自动保存，歧义结果保持待确认。
- 单项失败后继续，并正确统计四类结果：成功、存在候选但未唯一匹配、无候选、请求失败。
- 自定义显示剧名、已有 TMDB 元数据、季目录刮削、网盘字幕关联不回退。
- 夸克目录浏览和专用播放接口不受 TMDB 失败影响。
