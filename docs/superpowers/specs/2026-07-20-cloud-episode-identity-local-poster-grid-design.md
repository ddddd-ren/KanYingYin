# 网盘剧集强类型身份与本地海报墙统一设计

## 背景

网盘索引已经能为同一集的不同资源保存正确的季度、集号和版本信息。例如两个真实文件都属于 `S03E01`，但真实路径可能分别位于不同版本目录中，文件名甚至都只是 `01.mkv`。资源集合层把索引项转换成用于展示和播放的 `CloudFileEntry` 时，只保留了虚拟名称与真实路径，丢失了已识别的强类型季集字段。选集界面随后重新解析真实路径；解析失败时又使用列表位置生成“第 2 集”“第 4 集”，导致同集的第二个版本被错误显示为新的集数。

本地海报墙当前在桌面宽度下固定为三列或四列。窗口最大化后列数不会继续增加，卡片会随可用宽度被放大。网盘海报墙已经采用最大卡片宽度为 300 像素的自适应网格，因此两处媒体库在宽屏下的展示密度不一致。

## 目标

- 网盘选集直接使用索引已经确认的季度和集号，不再以列表位置代替已识别集号。
- 同一集的多个真实资源全部保留，并统一显示相同的 `SxxExx` 标识和各自版本标签。
- 季度卡片的集数继续按唯一集号统计，不累计同集的不同资源版本。
- 本地海报墙与网盘海报墙使用相同的 300 像素卡片宽度上限和自适应列数规则。
- 窗口放大时增加海报列数，不放大海报；窗口缩小时自动减少列数且不产生横向滚动。
- 交付版本为 `2.1.22+20122`，MSIX 清单版本为 `2.1.22.0`。

## 非目标

- 不修改、删除或重命名本地视频、网盘文件、目录、远程 ID 和真实播放路径。
- 不把纯数字文件名普遍解释为剧集，以免电影年份或数字片名被误识别。
- 不合并或隐藏同一集的不同清晰度、编码、动态范围、音轨或字幕版本。
- 不改变本地海报的 `BoxFit.contain`、悬浮信息层、点击、右键、长按或 Hero 动画行为。
- 不改变网盘海报的裁切、遮罩、菜单、刮削或播放交互。
- 不在本轮重做 TMDB 搜索候选窗口；季度海报仍由作品详情中的季度元数据提供。

## 网盘剧集展示身份

`CloudMediaIndexItem` 继续作为识别结果的事实来源。资源集合层生成虚拟 `CloudFileEntry` 时，除现有的远程 ID、真实路径、虚拟名称、大小和修改时间外，还附带以下可选展示字段：

- `seasonNumber`：索引确认的季度号。
- `episodeNumber`：索引确认的集号。
- `variantLabel`：同集存在多个资源时，由分辨率、码率、来源、编码、动态范围、音轨和字幕标签组合而成；无法区分时使用稳定的“版本 N”回退标签。

这些字段只丰富应用内虚拟条目，不修改云服务返回的原始条目，也不参与播放地址解析。未经过新集合链路的旧条目可以没有这些字段。

选集界面按以下优先级确定标签：

1. 同时存在强类型季度号和集号时，直接显示 `SxxExx`。
2. 只有强类型集号时，显示 `Exx`。
3. 强类型字段缺失时，为兼容旧数据，再解析真实路径。
4. 仍无法识别时才显示顺序性回退文本；该回退不得覆盖已有强类型身份。

版本标签同样优先使用 `variantLabel`，旧条目才从虚拟名称末尾的方括号中兼容提取。点击任意版本仍返回其原始 `CloudFileEntry` 并使用真实远程 ID 与路径播放。

## 排序与计数

资源集合继续按季度号、集号、真实路径排序。同集的多个版本因此相邻展示，每一行都使用同一个季集标识，例如：

- `S03E01 · 2160p WEB-DL H265 DDP 5.1 Atmos`
- `S03E01 · 4K DV HDR`
- `S03E02 · 2160p WEB-DL H265 DDP 5.1 Atmos`
- `S03E02 · 4K DV HDR`

季度卡片和选集标题的集数取非空 `episodeNumber` 的去重数量。若一季包含 12 个文件、每集两个版本且覆盖 6 个唯一集号，界面显示“6 集”，选集仍展示 12 个可播放行。完全没有可用集号时才回退为视频文件数量。

## 本地海报墙布局

本地媒体网格采用与网盘海报墙一致的最大横轴尺寸委托：

- 卡片最大宽度：300 像素。
- 横向间距：12 像素。
- 纵向间距：12 像素。
- 页面内边距：12 像素。
- 卡片宽高比：0.68。

不再按照 900 像素断点固定三列或四列，也不再根据固定列数计算 `mainAxisExtent`。网格根据可用宽度自行选择列数；620、1320、1920 像素等宽度下列数应逐级增加，任何卡片的实际宽度都不超过 300 像素。

本轮只统一布局密度。本地封面继续完整显示，信息层继续只在悬浮时出现，避免因布局修复引入裁切或遮挡变化。

## 数据流

1. 网盘扫描器读取真实目录和文件，不修改远端内容。
2. 媒体树解析作品、季度、集号及版本标签。
3. 索引器把强类型季集信息保存到 `CloudMediaIndexItem`。
4. 资源集合按索引身份排序、计算唯一集数，并把季号、集号和版本标签传给虚拟展示条目。
5. 选集界面优先读取虚拟条目的强类型字段，旧数据才回退到路径解析。
6. 用户选择某一版本后，播放链路继续使用该条目的真实远程 ID 与路径。
7. 本地媒体库把现有卡片数据交给最大宽度网格，窗口宽度只影响列数和剩余空间分配。

## 失败与兼容回退

- 新索引字段缺失时，选集保持现有路径解析能力，避免旧缓存立即失效。
- 路径也无法解析时，仍提供可点击的顺序回退文本，不阻断播放。
- 版本标签缺失时保留虚拟文件名，不丢弃或合并真实资源。
- TMDB 无 API Key、断网、超时或无结果时，本地扫描、网盘索引、选集和播放继续可用。
- 网格可用宽度异常或很窄时由 Flutter 网格约束缩小卡片，不引入横向滚动。

## 测试与验收

自动化测试必须覆盖：

- 两个不同版本目录中的真实文件名均为纯数字，但索引均标记为 `S03E01` 时，两行都显示 `S03E01`，不会出现“第 2 集”。
- 同集不同资源分别显示自身版本标签，并保持相邻排序。
- 12 个文件对应 6 个唯一集号时，季度卡片和选集标题均显示“6 集”，选集保留 12 行。
- 缺少强类型身份的旧条目仍能从路径解析 `SxxExx`。
- 620、1320、1920 像素宽度下，本地海报墙列数递增，卡片宽度不超过 300 像素，间距和宽高比与网盘一致。
- 本地卡片现有点击、右键、悬浮、封面适配和滚动控制器行为不回归。
