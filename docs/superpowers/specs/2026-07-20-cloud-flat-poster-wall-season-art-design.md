# 网盘全量海报墙与季度海报设计

本设计将网盘资源页从目录浏览器改为媒体库入口。用户选择夸克或 OpenList 来源后，无须逐层打开文件夹，即可看到该来源全部合格视频组成的作品海报墙。页面优先显示持久化索引，并在后台递归刷新；电视剧按作品和季度组织，每一季使用 TMDB 对应的季度海报。

## 目标与范围

本版本完成以下目标：

- 递归扫描网盘来源配置的全部根目录
- 将合格视频扁平汇总为与本地媒体库一致的海报墙
- 不在网盘资源页显示文件夹入口或目录导航
- 同一电视剧只占一个作品入口，进入后按季度和集数选择播放
- 多季度电视剧使用各季度对应的 TMDB 海报
- 复用现有手动匹配、自动继承、自动批量整理和播放能力
- 过滤非视频文件及未超过“网盘视频识别大小”的视频

以下行为不在本次范围内：

- 重命名、移动或删除网盘文件
- 修改远程目录、资源 ID、播放路径或请求头
- 为网盘文件建立本地副本
- 改变本地媒体库的数据结构或页面行为
- 在没有 TMDB 数据时凭空生成季度海报

本设计取代上一版“当前目录海报墙保留文件夹导航”的页面方案。已有同系列匹配规则继续保持“来源、父目录、标准剧名”的精确传播边界；页面展示可以跨目录聚合作品，但不会扩大自动匹配规则的安全边界。

## 采用的方案

采用“持久化索引立即展示、后台递归刷新”的共享索引方案。

页面打开时先读取 `CloudMediaIndexRepository` 中该来源的最后一次完整快照，并立即构建海报墙。随后使用共享的 `CloudMediaIndexer` 递归扫描所有配置根目录。扫描完成后比较索引变化，只更新新增、删除或元数据发生变化的作品。

该方案相比页面内临时递归扫描，首次以后的打开速度更快，扫描结果也能被 TMDB 整理和其他网盘媒体功能复用。相比定时预扫描，它能在用户进入页面后立即检查新增资源，不需要增加后台定时任务。

## 页面与交互

网盘资源页保留来源选择、搜索、刷新、扫描状态和现有作品操作，不再呈现以下内容：

- 配置根目录芯片
- 文件夹卡片
- 面包屑路径
- 返回上级目录操作
- 依赖逐层点击的目录加载流程

选中来源后，页面直接显示全部作品。存在缓存时先显示缓存海报墙，并以非阻塞状态提示后台刷新；没有缓存时显示扫描中占位状态。刷新操作重新扫描全部配置根目录，但不清空现有海报墙，直到新快照可用。

搜索范围包括作品中文标题、原始标题、标准剧名和组内视频文件名。搜索不改变作品身份、季度归属或匹配规则。

电影海报点击后直接播放对应视频。电视剧海报点击后进入季度与选集视图，选集使用每个网盘文件自己的远程 ID、路径、直链解析参数和字幕关联。

## 扫描与数据流

`CloudMediaIndexer` 注册为共享实例，同时供网盘媒体库和网盘资源页控制器使用。网盘资源页控制器负责协调展示，不重复实现递归遍历。

数据流固定为：

1. 用户选择网盘来源
2. 控制器读取该来源的持久化索引快照
3. 快照立即经过视频过滤、TMDB 记录合并和作品聚合
4. 页面显示海报墙
5. 后台索引器递归扫描全部配置根目录
6. 扫描过程中保存视频、字幕和目录快照
7. 已有精确规则先离线继承；配置了 TMDB API Key 时再执行现有自动匹配
8. 完整快照提交后重新聚合，并原位更新海报墙

来源切换、页面销毁或控制器释放后，旧扫描结果不得写回当前页面。刷新期间保留可播放的旧快照，避免海报墙闪空。

## 视频与字幕过滤

网盘资源页复用 `MediaRecognitionSettings.cloudMinSizeBytes`，不增加新的大小设置。

只有同时满足以下条件的条目可以进入海报墙和 TMDB 处理：

- 条目不是目录
- 扩展名属于受支持的视频格式
- 文件大小严格大于网盘视频识别阈值

小于或等于阈值的视频、大小无效的视频以及图片、文档、压缩包等非视频文件均不显示。字幕文件也不显示为资源卡片，但保留在索引中，并继续按路径、文件名或季集信息与视频关联。

修改网盘视频识别阈值后，下一次刷新或后台扫描使用新值。旧索引中的不合格条目在页面聚合层立即排除。

## 标题清理与作品聚合

文件名进入 TMDB 识别前，继续清理分辨率、来源、编码、动态范围和音轨等发布规格。`2160p WEB-DL HEVC DDP 5.1.mkv` 等后缀不得参与搜索词、标准剧名或海报标题。

作品聚合限定在当前网盘来源内，优先级如下：

1. 已匹配资源按媒体类型和 TMDB ID 聚合
2. 未匹配电视剧按清理后的标准剧名聚合，并保留解析到的季号和集号
3. 无法可靠识别为电视剧的视频作为独立电影候选，不做模糊合并

页面允许同一来源中不同目录的同一 TMDB 作品合并展示。未匹配资源只有标准剧名精确一致时才能合并。已有同系列规则的自动传播仍要求来源、父目录和标准剧名同时一致，避免一个目录中的手动确认误匹配另一个目录。

电视剧组内先按季号、集号排序，再按文件名排序。重复季集号保留为独立可播放条目，并显示足以区分的文件名信息。

## 多季度与季度海报

同一电视剧在主海报墙只显示一个作品入口，主卡片使用 TMDB 电视剧级海报。进入作品后按解析到的季度分组，例如“第 1 季”“第 2 季”“第 3 季”。每个季度只包含索引中实际存在的网盘视频，不为缺失分集创建虚拟条目。

季度和集数使用现有解析能力识别 `S01E01`、`Season 1`、`第1季` 等常见格式。识别到集号但没有明确季号时按现有解析规则处理；无法可靠识别集号的文件不得被强行放入某一季度。

完成电视剧 TMDB 匹配后，读取该作品的季度元数据，并为实际存在的季度保存：

- TMDB 季度 ID 与季号
- 季度名称
- 首播日期或年份
- 季度简介
- 季度海报地址及本地缓存路径

季度海报显示优先级固定为：

1. 已缓存的对应季度海报
2. TMDB 对应季度海报地址
3. 电视剧级主海报
4. 应用默认占位海报

手动匹配整部电视剧后立即获取并应用季度信息。后续扫描到的新分集继承同一电视剧匹配时，直接复用已缓存的季度数据；发现尚未缓存的新季度且 TMDB 可用时再补充读取。断网或没有 API Key 时继续显示已有季度缓存或回退海报，不影响选集和播放。

## TMDB 协调

快照进入页面前先合并已有自定义标题和 TMDB 记录。后台扫描后的处理顺序保持为：

1. 保留自定义剧名
2. 保留已有 TMDB 匹配
3. 应用精确同系列规则
4. 跳过有效期内的无结果记录
5. 在 API Key 可用时执行现有自动匹配

TMDB 不可用、没有 API Key 或断网时，扫描、海报墙展示和播放必须继续可用。自动匹配失败不得移除旧记录、旧海报或已缓存季度数据。

## 错误与并发处理

后台扫描失败时保留上次成功快照，并在页面显示可重试提示。单个根目录失败时记录该目录错误，其他根目录继续扫描；只有完整、可辨识的扫描结果才能替换对应部分的持久化快照，避免把暂时不可访问的目录误判为已删除。

单个文件的 TMDB 匹配、海报下载或字幕关联失败，不得阻断其他作品入库。日志和错误提示不得包含 Cookie、TMDB API Key、播放直链或请求头。

同一来源只允许一个主动刷新任务。重复刷新复用或取消旧任务，具体遵循现有索引器并发约束。来源切换后，旧来源可以完成持久化索引，但不能更新新来源的页面状态。

## 测试设计

### 控制器与扫描

- 选择来源后先展示持久化快照，再启动后台递归扫描
- 没有快照时显示扫描状态，完成后显示作品
- 多个配置根目录中的视频汇总到同一海报墙
- 页面不再输出文件夹、根目录芯片或目录导航
- 刷新期间旧海报墙保持可见和可播放
- 来源切换后旧任务不污染当前页面
- 单个根目录失败不清空其他目录的旧索引

### 过滤与聚合

- 非视频、字幕和图片不显示为作品卡片
- 小于、等于和大于阈值的视频分别得到正确结果
- 字幕虽然不显示，仍能关联到对应视频
- 发布规格后缀不进入 TMDB 搜索词和显示标题
- 同一 TMDB 电视剧跨目录聚合为一个作品
- 不同来源、不同 TMDB ID 和不同标准剧名不合并
- 电影直接播放，电视剧进入真实选集

### 多季度与海报

- 多季度分集归入同一电视剧作品，并按季号、集号排序
- 主海报墙只出现一个电视剧入口
- 第 1、2、3 季分别使用各自的 TMDB 季度海报
- 每季只显示索引中实际存在的分集
- 季度海报缺失时按季度缓存、远程地址、主海报和占位图顺序回退
- 手动匹配后获取季度数据，后续新增分集复用缓存
- 断网、无 API Key 和季度详情失败不影响选集播放

### 回归与交付

- 手动匹配、同系列规则继承和自动批量整理继续可用
- 夸克和 OpenList 的视频直连、请求头、字幕和选集继续可用
- 删除来源或索引不会删除远程文件
- 本地媒体库行为保持不变
- 完整 `flutter test` 通过
- `flutter analyze` 无错误
- Windows Release 构建通过
- 签名 MSIX 清单、版本和桌面安装包验证通过
