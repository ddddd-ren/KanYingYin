# 网盘文件夹剧名与季度识别设计

## 背景

网盘视频不一定在文件名中携带完整剧名和季度。常见目录可能是 `权力的游戏 第2季/01.mkv`，也可能是 `权力的游戏/第2季/01.mkv`。现有扫描主要解析视频文件名，因此会把只有数字集号的文件当成电影，或无法把不同季度归到同一部剧。

本次改造让网盘扫描同时识别视频文件夹和文件名，将可靠的剧名、季号、集号写入索引，供 TMDB 匹配、海报墙聚合和季度选集统一使用。

## 目标

- 从“剧名加季度”的单层文件夹识别剧名和季号。
- 从“剧名目录/季度目录”的两层结构识别剧名和季号。
- 从纯集数视频文件名识别集号，并与文件夹结果组合。
- 文件名和文件夹季号冲突时，以文件名为准并记录日志。
- 重新扫描现有来源时更新旧索引，不要求用户删除或重建来源。
- 无法可靠识别时维持现有行为，不阻断扫描和播放。

## 非目标

- 不修改夸克网盘中的文件夹名或视频文件名。
- 不改变现有 TMDB 手动匹配、系列规则安全边界或播放器行为。
- 不把任意上级分类目录直接当成剧名。

## 识别器边界

新增独立的网盘路径识别器。识别器接收完整远程视频路径，输出剧名、季号、集号、各字段的识别依据以及季号冲突标记。索引器只消费该结构化结果，不自行重复解析目录。

识别器复用现有视频文件名和集数解析能力，并补充文件夹语义。这样可以保持现有 `S01E01` 行为，同时让目录识别在单一边界内独立测试和演进。

## 识别优先级

1. 文件名包含明确剧名、季号和集号，例如 `权力的游戏.S01E03.mkv` 或 `权力的游戏 第1季第3集.mkv` 时，优先使用文件名结果。
2. 文件名仅提供集号时，检查当前文件夹：
   - `权力的游戏 第2季/03.mkv` 从当前文件夹提取剧名 `权力的游戏` 和季号 `2`。
   - `权力的游戏/第2季/03.mkv` 从当前文件夹提取季号 `2`，从紧邻的上一级文件夹提取剧名 `权力的游戏`。
3. 文件夹不能可靠提供剧名或季号时，保留现有文件名解析结果。
4. 文件名与文件夹均提供季号但不一致时，采用文件名季号，并通过现有日志能力记录远程路径和两个候选季号。

不会跨越多个不相关目录搜索剧名。纯季度目录只读取紧邻上一级目录，避免把 `电视剧`、`夸克网盘`、`已整理` 等分类目录误识别为作品名。

## 支持格式

季度文件夹支持：

- `第2季`、`第二季`
- `Season 2`
- `S02`
- `权力的游戏 第2季`
- `权力的游戏 Season 2`
- `权力的游戏 S02`

纯集数文件名支持：

- `01.mkv`
- `E01.mkv`
- `EP01.mkv`
- `第1集.mkv`

中文季度数字支持 `一` 至 `九十九` 的常用写法；阿拉伯数字季号限制在现有合理范围内。无法转换或超出范围的季度文本不参与目录识别。

## 数据流

1. 网盘递归扫描获得视频的完整远程路径。
2. 路径识别器同时分析文件名、当前文件夹和必要时的紧邻上一级文件夹。
3. 索引器将结果写入 `CloudMediaIndexItem`：
   - `seriesName` 保存清理后的剧名。
   - `seasonNumber` 保存最终季号。
   - `episodeNumber` 保存集号。
   - 成功获得剧名和有效集号时，`mediaType` 保存为 `episode`。
4. 索引仓库按现有方式持久化记录。
5. 海报墙按标准剧名或已确认 TMDB 身份聚合作品，选集页按 `seasonNumber` 分区并使用对应季度海报。
6. TMDB 自动或手动匹配使用清理后的剧名作为搜索词，不携带季度标记。

## 与系列规则的关系

手动匹配后的系列规则仍使用“来源、父目录、标准剧名”安全边界。路径识别器只改进规则生成和查找时使用的剧名与季号，不扩大规则传播范围，也不跨来源合并。

对于 `剧名/第2季` 结构，规则身份应以解析后的作品剧名为准；现有父目录约束继续避免同名作品互相污染。跨季度聚合仍以唯一 TMDB 身份或标准剧名为依据。

## 错误处理与兼容

- 目录格式不支持时不抛出扫描错误，直接使用现有文件名解析结果。
- 只有季号但没有可靠剧名时，不将分类目录猜测为剧名。
- 只有剧名但无法识别有效集号时，不强制把视频标记为剧集。
- 季号冲突只记录诊断信息，不中断索引、TMDB 匹配或播放。
- 重新扫描会覆盖旧的错误识别字段；TMDB 已确认记录和用户自定义剧名继续按现有持久化规则保留。

## 测试方案

单元测试覆盖路径识别器：

- “剧名加季度”单层目录。
- “剧名/季度”两层目录。
- 阿拉伯数字、英文和中文数字季度。
- `01`、`E01`、`EP01`、`第1集` 四种纯集数文件名。
- 文件名季号覆盖文件夹季号并产生冲突标记。
- 分类目录不被当成作品名。
- 原有 `S01E01` 文件名识别保持不变。

索引与集成测试覆盖：

- 纯集数视频扫描后写入正确的剧名、季号和集号。
- 同一剧不同季度聚合为一个海报墙作品。
- 选集页按季度分区并读取对应 TMDB 季度海报。
- 索引持久化后重新读取保持识别结果。
- 重新扫描旧索引后得到新的目录识别结果。

## 验收

- 两类约定目录结构均能得到正确剧名、季号和集号。
- 文件名与文件夹冲突时最终结果采用文件名。
- TMDB 搜索词不包含季度后缀。
- 无法识别的资源仍可扫描和播放。
- `flutter test`、`flutter analyze` 和 Windows Release 构建通过。
- 版本、发布说明、应用内版本历史和 MSIX 版本同步更新，签名 MSIX 复制到桌面并通过清单与哈希验证。
