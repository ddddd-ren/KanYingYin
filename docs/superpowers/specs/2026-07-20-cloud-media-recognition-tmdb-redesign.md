# 网盘媒体识别与 TMDB 季度海报重做设计

## 背景

现有网盘媒体链路分别在路径解析、名称清理、系列身份、TMDB 记录传播和海报墙聚合中推断剧名。多套规则各自维护正则，导致真实分享目录中的作品目录、季度目录、发布规格和广告内容相互混淆。

典型问题包括：

- `第 3 季 - 2160p WEB-DL H265 DDP 5.1 Atmos` 因为“季”不在目录名末尾而无法识别为季度目录，剩余规格文字被当作剧名。
- `第三季（2025）4K DV&HDR` 清理后残留 `DV&`，被当作 TMDB 搜索标题。
- `01.mkv` 等纯集号文件名无法独立提供剧名和季号，必须继承目录语义。
- 视频文件中的英文名称优先于中文作品总目录，可能将同一作品拆成多个身份。
- TMDB 已返回全部季度海报，但主海报墙只显示电视剧级主海报。
- 手动匹配规则使用视频直接父目录作为边界，不同季度目录之间无法一并纠正。

本次重做以完整目录树为识别输入，将作品、季度、分集和发布规格拆为独立字段。TMDB 匹配归属于作品，主海报墙按季度展示，播放仍归属于真实网盘文件。

## 目标

- 对用户配置的全部夸克与 OpenList 来源、全部媒体根和全部作品应用同一套识别与刮削流程，不限定具体作品、语言或目录模板。
- 从混合了分享编号、中文标题、年份、画质、全集说明和广告内容的目录树中识别可靠的作品边界。
- 将作品、季度、分集、有效版本、发布规格和广告内容分类，禁止把规格残留或纯季度名称当作剧名。
- 支持视频省略剧名和季号、仅以集号命名的目录结构。
- 同一作品只执行一次 TMDB 匹配和详情请求，按季号应用对应季度海报。
- 在网盘主海报墙直接显示每个实际存在季度的独立海报卡。
- 手动确认任意季度的 TMDB 候选后，同一作品总目录内的所有季度和分集一并应用该匹配。
- 用户修改的名称作为作品级 TMDB 刮削查询覆盖，并参与生成应用内虚拟名称；网盘真实名称保持不变。
- 相同来源中其他目录的同名作品保持独立，不扩大手动匹配传播范围。
- 不修改、移动或删除网盘中的任何文件和目录。
- TMDB 不可用、无 API Key 或断网时，扫描、索引、选集和播放继续可用。

## 非目标

- 不为任何具体作品、TMDB ID、中文标题、英文标题或截图样本编写硬编码分支。
- 不自动删除广告、样片、重复分集或低质量版本。
- 不依据文件大小单独判断广告、垃圾或重复内容。
- 不自动合并不同作品总目录，即使它们解析出相同标题或 TMDB ID。
- 不修改本地视频文件、网盘文件名或目录结构。
- 不为 TMDB 中不存在的季度凭空生成海报。
- 不在本次重做中改变播放器控件、动画、字幕交互或播放直链解析行为。

## 参考原则

### TMDB media library curator skill

本设计采用以下安全和归并原则：

- 只扫描用户已经配置的媒体根目录。
- 广告、推广图片和下载站说明只从媒体识别结果中排除，不执行删除。
- TMDB 无结果不代表内容无效，未知内容继续保留和播放。
- 导演剪辑版、加长版、重剪版、特别篇和其他有效版本保持内容身份，不作为重复文件清除。
- 字幕、NFO、压缩包和图片不成为作品卡，但不能仅凭名称或大小自动删除。

### 现有本地视频解析规则

共享识别层复用或兼容本地 `LocalEpisodeParser` 已验证的能力：

- `S01E01`、`第2季第3集`、`E03`、`EP03`、`Episode 03`、`[03]` 等季集格式。
- `480p`、`720p`、`1080p`、`1440p`、`2160p`、`4K`、`8K` 等分辨率。
- `WEB-DL`、`WEBRip`、`BDRip`、`BluRay`、`HDTV` 等片源。
- `x264`、`x265`、`H264`、`H265`、`HEVC`、`AV1` 等编码。
- 发布组、括号标签和分隔符规范化。
- 年份、`4K/8K` 和电影发布尾号不能误判为集号。

网盘常见目录还需补充 `DV`、`Dolby Vision`、`HDR`、`HDR10`、`HDR10+`、`DDP`、`EAC3`、`Atmos`、声道、全集说明和分享推广特征。

## 采用方案

采用“共享名称分析器 + 网盘目录树解析器 + 作品级 TMDB 记录 + 季度级海报卡”方案。

不继续扩充现有多套独立正则。单条路径解析无法可靠判断作品总目录，文件级 TMDB 记录也无法表达跨季度共享身份。每季度独立请求 TMDB 会产生重复请求和跨季误匹配，因此也不采用。

## 组件边界

### `MediaNameAnalyzer`

无状态共享分析器，供本地媒体和网盘媒体使用。输入单个文件名或目录名及其节点类型，输出强类型 `MediaNameAnalysis`：

- `role`：作品、季度、分集、有效版本、广告、未知。
- `titleCandidates`：保留来源和置信度的中文、英文及备用标题候选。
- `seasonNumber`。
- `episodeNumber` 或连续集数范围。
- `year`。
- `releaseTags`：分辨率、片源、编码、动态范围、音频和发布组。
- `confidence` 与 `evidence`：说明字段来自作品目录、季度目录、文件名、用户纠正还是 TMDB。

分析器只解析单节点语义，不决定作品边界，也不执行 TMDB 请求。

### `CloudMediaTreeResolver`

接收用户配置根目录下的完整远程目录快照以及每个节点的名称分析结果，输出作品树：

- `CloudWorkIdentity`：来源、作品总目录远程 ID、规范路径、原始名称、应用内标题、标题候选和已确认身份。
- `CloudSeasonIdentity`：所属作品、季号、季度目录集合、原始名称集合、应用内名称和季度年份。
- `CloudEpisodeIdentity`：所属季度、集号、原始文件名、应用内文件名、真实远程文件引用、字幕和发布规格。

解析器负责定位作品总目录、继承上下文、合并同季目录、隔离不同作品目录和报告冲突。它不自行重复清理标题。

### 作品级 TMDB 记录

TMDB 记录从单文件归属改为作品归属。作品记录至少保存：

- 来源 ID、作品总目录远程 ID 和规范路径。
- 用户设置的 `scrapeTitleOverride`、应用内有效标题及全部标题别名。
- TMDB ID、媒体类型、中文标题、原始标题、简介、评分、首播日期、主海报和背景图。
- 完整季度元数据及季度海报缓存路径。
- 匹配状态、确认方式和更新时间。

季度和分集通过作品键引用该记录，不复制多份权威 TMDB 身份。

### 原始名称与虚拟名称

所有作品、季度和分集同时保存两类名称：

- `remoteName`：网盘返回的真实名称，仅用于远程定位、媒体详情、文件信息和故障诊断。
- `displayName`：由目录树识别、用户刮削名称和 TMDB 结果生成的虚拟名称，用于主海报墙、季度卡、选集列表和搜索结果展示。

播放、字幕关联和重新扫描始终使用远程 ID、规范路径及 `remoteName`，不能使用虚拟名称反向定位网盘文件。主海报墙和选集列表不直接显示原始名称；用户需要时可在“媒体详情/文件信息”中查看。

虚拟名称生成规则：

- 作品：优先使用已确认的 TMDB 中文标题，其次使用 `scrapeTitleOverride`，最后使用目录树识别标题。
- 季度：`作品有效标题 第 n 季`。
- 分集：`作品有效标题 SnnEnn`，季号与集号至少补齐两位，在文件列表场景保留原扩展名。
- 重复季集号：在虚拟名称后附加能够区分的画质、编码、动态范围、音轨或版本序号，不覆盖任何版本。

### 季度级展示模型

主海报墙使用季度卡作为电视剧展示单位。季度卡的稳定键由作品键和季号组成，不使用单个视频文件键。电影仍以作品卡展示。

季度卡包含对应季度海报、作品标题、季度名称、年份、集数和实际分集列表。点击季度卡只打开该季度的分集，不再先打开整部剧的多季度弹层。

## 目录角色识别

### 识别顺序

1. 规范化 Unicode、空白和路径分隔符，但保留原始名称用于媒体详情和诊断。
2. 提取括号、书名号、季集标记、年份和发布规格，不直接丢弃原文。
3. 根据结构字段判断节点角色。
4. 节点确认为季度或分集后，剩余规格文字不得回退为剧名。
5. 目录树解析器结合父子关系确定作品总目录和继承字段。

### 季度目录

季度标记允许位于目录名开头、中间或结尾，后面可以继续出现年份和发布规格。以下目录均识别为纯季度目录，不产生标题候选：

- `第一季`
- `第 3 季 - 2160p WEB-DL H265 DDP 5.1 Atmos`
- `第三季（2025）4K DV&HDR`
- `Season 3 2160p`
- `S03 Dolby Vision`

如果目录形式为 `作品名 第2季 2160p`，分析器可以同时输出作品标题候选和季号；目录树已有明确作品总目录时，仍以总目录身份为准。

### 纯集号视频

以下文件名可以提供集号：

- `1.mkv`、`01.mkv`、`001.mkv`
- `E01.mkv`、`EP01.mkv`、`Episode 01.mkv`
- `第1集.mkv`

纯集号文件只有位于已确认季度目录中时才能成为分集。剧名继承作品总目录，季号继承季度目录，纯数字文件名不参与 TMDB 标题搜索。普通电影目录中的 `01.mkv` 不会被无条件认作电视剧。

分集排序使用数值集号。重复季集号保留为独立播放版本，并显示足以区分的路径或发布规格。

### 广告与非媒体内容

名称含有明确推广特征的节点不参与作品身份，例如：

- `0001更多资源请访问...`
- `0002全网搜索资源...`
- `0000防走失地址...`
- 带有网址、下载站、推广入口或请访问官网等明确内容。

推广图片、说明文档和普通图片不进入海报墙。广告分类只影响媒体识别，不授权删除远程内容。用途无法确认时标记为未知并保留，不武断归类为广告。

## 作品总目录定位

目录树解析器仅在配置根目录范围内工作，并使用以下结构证据定位作品总目录：

- 同一目录包含一个或多个明确季度子目录，季度子目录中包含有效分集。
- 当前视频路径经过明确季度目录，季度目录的直接父目录提供作品标题候选。
- 多个分集文件共享相同有效标题和作品边界。

`电视剧`、`剧集`、`动漫`、`Anime`、`TV`、`Shows`、`媒体`、`视频`、`网盘`、`夸克网盘`、`已整理` 等分类名称不能成为作品标题。解析不会跨越多个无关上级目录寻找剧名。

同一作品总目录下出现两个第三季目录时，两个目录映射到同一个 `CloudSeasonIdentity(seasonNumber: 3)`。不同作品总目录即使标题相同，也生成不同作品键。

## 标题候选与优先级

TMDB 查询标题来源优先级固定为：

1. 用户手动设置的 `scrapeTitleOverride`。
2. 已确认的作品级 TMDB 身份。
3. 作品总目录中的有效中文或原始标题。
4. 多个视频文件共同出现的稳定标题。
5. 单个完整视频文件名提供的备用标题。

分享编号、发布规格、年份、全集说明和推广文字不会进入最终查询词。标题清理生成候选而不是破坏性覆盖原文。

`scrapeTitleOverride` 由“修改刮削名称”产生：它不会写回网盘文件或目录，不替换远程名称，也不改变播放路径。设置后立即参与应用内虚拟名称生成并用于下一次 TMDB 查询；匹配成功后虚拟名称统一使用 TMDB 中文标题，尚未匹配时使用该覆盖名称。

对于：

`154332_《弥留之国的爱丽丝3》(2025) 4K 全6集 内附第一二季`

结合季度结构与英文分集文件名生成候选：

1. `弥留之国的爱丽丝`
2. `弥留之国的爱丽丝3`
3. `Alice in Borderland`

末尾数字不会在所有标题中无条件删除；只有作品结构、季号和文件别名共同支持时，才生成去除季度数字的更高优先级候选。目录中的 `2025` 在多季度合集上下文中不作为整部剧首播年份的硬过滤条件。

示例作品只用于回归测试。生产解析器只能依据节点角色、路径层级、季集证据、发布规格、标题别名和 TMDB 候选评分工作，禁止检查“弥留之国的爱丽丝”“Alice in Borderland”或任何其他固定作品文本。`The 100`、`1923` 等合法数字标题必须保留，不能套用示例中的末尾季号清理。

## 季集冲突

季度目录提供季号，完整文件名提供季号和集号。两者一致时提高置信度；冲突时保留两项证据并将该分集标记为待确认，不静默覆盖。

冲突不会阻断其他资源扫描、作品卡生成或播放。日志只记录脱敏后的来源、路径和候选值，不包含 Cookie、API Key、播放直链或请求头。

## TMDB 刮削流程

1. 扫描完成后先建立作品树和季度卡，即使没有 TMDB 也可展示和播放。
2. 已确认是季度结构的作品只搜索电视剧类型。
3. 依次使用高置信度中文候选和英文别名搜索，不将发布规格或纯集号作为查询词。
4. 只有标题、类型和目录结构共同达到自动匹配阈值时自动确认；候选接近或冲突时进入待手动匹配。
5. 每个作品只请求一次电视剧详情。
6. 优先读取 `zh-CN` 详情；简介、主图或季度图缺失时使用 `en-US` 对应字段补齐。
7. 按 `seasonNumber` 将季度元数据应用到实际存在的季度卡，不为网盘中不存在的季度创建卡片。
8. 季度海报逐季缓存，缓存键使用作品键和季号，不能使用单个视频键。

季度卡海报回退顺序：

1. 对应季度本地缓存海报。
2. 对应季度 TMDB 网络海报。
3. 电视剧级主海报缓存。
4. 电视剧级主海报网络地址。
5. 应用默认占位图。

## 自动匹配、刮削名称与手动确认

自动扫描完成后，应用按作品执行一次 TMDB 匹配并为所有实际季度分配海报。自动匹配失败不写入错误身份，不移除旧海报和旧匹配。

用户在任意季度卡执行“修改刮削名称”时，更新作品级 `scrapeTitleOverride`，立即重建同作品的虚拟作品名、季度名和分集名，随后使用该名称重新搜索 TMDB。用户确认候选后更新作品级匹配记录，并使用 TMDB 中文标题再次统一虚拟名称，同时刷新全部季度卡、分集索引和季度海报。

修改刮削名称不会重命名网盘作品目录、季度目录或视频文件，也不会修改应用中保存的远程原名。用户取消搜索或没有匹配结果时，应用继续显示基于该刮削名称生成的虚拟名称，原有海报和播放状态不变。

传播边界固定为：

- 相同来源。
- 相同作品总目录远程 ID；远程 ID 不可用时使用规范路径。
- 相同作品键。

其他目录中的同名作品保持独立，不自动跟随。`scrapeTitleOverride` 和已确认身份的优先级高于后续自动扫描。

## 主海报墙行为

电视剧在主海报墙按实际季度直接展开为独立卡片。例如：

- `弥留之国的爱丽丝 第 1 季`
- `弥留之国的爱丽丝 第 2 季`
- `弥留之国的爱丽丝 第 3 季`

每张卡使用对应季度海报并显示实际集数。点击第 3 季卡片只显示第 3 季分集。两个第三季目录中的重复集号均保留，可通过文件名、画质、编码、音轨或路径区分。

搜索同时匹配作品中文标题、原始标题、标题别名、季度名称和组内文件名。搜索不改变作品身份或传播边界。

## 数据迁移与重新扫描

新增识别规则版本并写入网盘索引快照或指纹。版本变化时，即使远程目录指纹未变化，也必须重新计算作品、季度和分集身份。

迁移规则：

- 旧 `customTitle` 迁移为作品级 `scrapeTitleOverride`；未匹配时用它生成虚拟名称，已匹配时显示已确认的 TMDB 中文标题。
- 保留已确认 TMDB ID。
- 同一作品总目录中的旧文件级记录 TMDB ID 一致时，迁移为作品级记录。
- 同一作品总目录中存在多个冲突 TMDB ID 时，不自动选择，标记为待纠正。
- 旧季度海报缓存可以按 TMDB ID 和季号安全复用；无法确认归属的缓存不绑定到新作品。
- 重新扫描只替换派生索引和缓存引用，不修改网盘文件。
- 扫描失败时保留上次完整可用快照，禁止用部分空结果清除旧媒体库。

## 错误、离线与并发

- 无 API Key 时跳过 TMDB 请求，继续生成结构化季度卡。
- 断网或 TMDB 请求失败时保留已有身份、季度元数据和海报缓存。
- 单季海报下载失败不影响其他季度海报、选集或播放。
- 单个目录读取失败时复用该子树上次快照，其他目录继续扫描。
- 同一来源只允许一个主动扫描任务，刷新期间继续显示旧快照。
- 来源切换后旧任务可以完成持久化，但不能覆盖新来源页面状态。
- 删除来源、索引或缓存不得删除用户原始视频文件。
