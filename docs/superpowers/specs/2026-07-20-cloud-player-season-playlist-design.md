# 网盘播放器按季度显示完整选集设计

## 背景

网盘海报墙的选集弹窗已经持有当前作品或季度的完整视频列表，但用户点击某一集后，`CloudResourcesPage` 只把被点击的视频构造成单个 `CloudPlaybackTarget` 交给 `LocalVideoController`。播放器只能根据收到的目标生成选集，因此列表中只剩当前视频。

本地媒体库入口会传入完整的系列分集，所以同一播放器在该入口能够正常显示全部选集。这说明问题位于网盘海报墙到播放器的启动参数边界，不是刮削、索引或播放器列表渲染错误。

## 目标

- 从网盘海报墙的选集弹窗点击任意一集后，播放器显示该集所属季度的完整真实视频列表。
- 播放器初始选中并播放用户点击的文件。
- 不把同一作品的其他季度混入当前播放列表。
- 未识别季度的多视频作品仍传递该作品的全部视频。
- 单个电影或单文件资源继续使用单项播放列表。
- 每个视频继续保留远程 ID、真实路径、显示名称和同目录字幕引用。

## 方案比较

### 方案一：传递整个作品的所有季度

实现最直接，但会把第一季、第二季等混入同一播放器列表，不符合每季度独立选集的现有展示语义。

### 方案二：播放器打开后重新查询媒体库

可以避免页面传递列表，但会让播放器依赖网盘索引和分组规则，增加重复查询、来源状态和异步竞态，职责边界不清晰。

### 方案三：页面启动播放时传递当前季度列表

由已经拥有 `CloudResourceMediaGroup` 的页面确定播放范围，再把完整目标列表交给现有 `openCloudPlayback`。播放器保持通用，只消费明确的播放列表。该方案改动最集中，也与本地媒体库入口一致，因此采用此方案。

## 数据流

1. 用户在 `CloudResourceEpisodeSheet` 中点击一个真实视频文件。
2. `CloudResourcesPage` 保留弹窗对应的 `CloudResourceMediaGroup` 和选中视频。
3. 若选中视频有季度号，从 `group.seasons` 选择相同季度的全部视频；若没有季度号，则使用当前未识别季度分组或 `group.videos`；单文件资源使用自身。
4. 页面为播放范围内的每个视频构造 `CloudPlaybackTarget`，分别关联其字幕。
5. 页面以当前作品展示名、完整目标列表和选中视频稳定 ID 调用 `LocalVideoController.openCloudPlayback`。
6. 控制器沿用现有逻辑生成 `roadList`，首次只解析当前集，切集时再实时解析对应网盘地址。

## 顺序与多版本

- 播放列表沿用 `CloudResourceSeasonGroup.videos` 的现有排序，不在播放入口再次排序。
- 同一集存在 4K、1080P 等多个真实文件时全部保留，显示各自现有文件名或版本标签，不静默删除资源。
- 季度范围以索引中的 `seasonNumber` 为主；缺失时沿用现有文件名解析结果，避免纯数字文件丢失。

## 接口调整

页面测试注入目前只接收一个 `CloudPlaybackTarget`，无法验证完整播放会话。将其改为接收一次播放启动请求，包含：

- `seriesTitle`
- `targets`
- `selectedStableId`

生产路径和测试路径都使用同一请求数据，防止测试只验证当前文件而遗漏播放列表。

## 异常与兼容

- 选中视频不在生成的季度列表时立即报参数错误，不进入播放器。
- 来源缺失、视频不可用或播放地址解析失败继续沿用现有提示和重试行为。
- 本修复不修改网盘文件、目录、远程 ID、索引分组或 TMDB 元数据。
- 本地媒体库、OpenList 与夸克的播放地址解析逻辑保持不变。

## 测试

- 先增加失败测试：从包含两个季度的网盘作品选择第一季第二集，启动请求必须包含第一季全部分集、排除第二季，并选中第一季第二集。
- 验证未识别季度多视频作品传递全部视频。
- 验证单文件资源仍只传递一个目标。
- 保留字幕匹配断言，确保每个分集分别关联正确字幕。
- 完成后执行定向测试、全量 `flutter test`、`flutter analyze`、Windows Release 和 MSIX 清单、签名、哈希验证。
