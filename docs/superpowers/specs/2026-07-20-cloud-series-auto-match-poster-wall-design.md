# 手动匹配后自动关联同系列与网盘海报墙设计

本设计让网盘资源页按作品展示当前目录的视频。手动确认一集的 The Movie Database（TMDB）电视剧后，看影音立即匹配同目录的其他分集，并让以后新增的同系列分集自动继承。识别过程遵守现有“网盘媒体识别大小”设置，不修改网盘文件、目录或播放路径。

## 目标与范围

本版本完成两个相互配合的目标：

- 手动匹配一个分集后，立即匹配当前目录中标准剧名相同的其他分集
- 将当前目录的视频按电影或剧集分组，显示与本地媒体库一致的作品海报墙

传播范围固定为同一网盘来源、同一父目录和同一标准剧名。规则不跨目录、不跨来源，也不使用模糊相似度。

以下行为不在本次范围内：

- 重命名、移动或删除夸克与 OpenList 文件
- 将不同目录中的同名剧集合并
- 覆盖已经匹配的资源或自定义剧名
- 对无法识别集号的文件推断电视剧身份
- 取消现有文件夹导航

## 当前问题

手动匹配目前只按资源 `stableKey` 保存单个文件的记录。`The.Resurrected.S01E01.2160p.WEB-DL.mkv` 匹配为《回魂计》后，`S01E02`、`S01E03` 和 `S01E04` 仍是独立的未匹配资源。后续扫描也不知道这几集属于用户已经确认的 TMDB 电视剧。

网盘资源页还把文件夹和每个视频放进同一个网格。即使多个视频属于同一部剧，它们仍各占一张卡片。页面只按扩展名过滤普通文件，没有应用现有的网盘视频大小限制。

## 采用的方案

看影音持久化“同系列匹配规则”。规则保存用户明确确认的 TMDB 电视剧，并以来源、父目录和标准剧名作为精确键。当前目录和后续扫描先使用该规则，再决定是否请求 TMDB。

网盘资源页增加独立分组层。文件夹保留为导航入口，符合识别条件的视频进入作品分组。电视剧分集共享一张作品海报，电影保持独立卡片。

相比只保存搜索词，该方案不会为每一集重复搜索，也不会因搜索排序变化选择其他条目。相比只处理当前文件，该方案能覆盖应用重启后新增的分集。

## 系列身份与安全边界

新增职责单一的系列身份解析组件。组件接收 `CloudResourceTmdbTarget`，使用现有 `LocalEpisodeParser` 从远程路径提取标准剧名、季号和集号。

只有满足以下条件的资源才能建立或命中规则：

- 资源是独立视频，不是目录
- 文件扩展名属于受支持的视频格式
- 文件大小大于当前“网盘媒体识别大小”
- 集数解析器返回有效的标准剧名和集号
- 手动候选的媒体类型是 TMDB 电视剧

系列键包含以下字段：

- 网盘来源 ID
- 规范化父目录路径
- 规范化标准剧名

标准剧名去除首尾空白，将点、下划线和连续空白统一为空格，并使用不区分大小写的比较。系列键不包含季号，因为 TMDB 电视剧 ID 覆盖同一剧的全部季。

例如以下文件生成同一个系列键：

- `The.Resurrected.S01E01.2160p.NF.WEB-DL.H.265.DDP5.1.Atmos.mkv`
- `The.Resurrected.S01E02.2160p.NF.WEB-DL.H.265.DDP5.1.Atmos.mkv`
- `The.Resurrected.S02E01.1080p.WEB-DL.mkv`

相同文件放入另一个目录后生成不同的键。不同来源中的相同路径也生成不同的键。

## 持久化模型

新增 `CloudSeriesMatchRule` 强类型模型。规则包含：

- `sourceId`
- `parentPath`
- `normalizedSeriesName`
- `tmdbId`
- `mediaType`
- 中文标题、原始标题、简介、评分和发行日期
- 海报与背景图地址
- 可用的海报缓存路径
- 语言、确认时间和规则更新时间

规则只存储公开元数据，不存储 Cookie、TMDB API Key、播放直链或请求头。

新增 `CloudSeriesMatchRuleRepository`。仓库提供按键查询、按来源读取、覆盖保存和按来源删除。Hive 存储使用看影音现有设置盒中的独立键。相同系列再次手动匹配时，最后一次确认覆盖原规则。

删除网盘来源时，删除该来源的规则。清理规则不删除远程文件，也不删除其他来源的数据。

## 手动匹配后的传播流程

手动匹配沿用现有候选搜索和详情读取。用户选择候选后执行以下步骤：

1. 保存当前分集的 `CloudResourceTmdbRecord`
2. 精确同步当前分集对应的媒体索引
3. 验证当前目标是否符合系列规则条件
4. 保存或更新 `CloudSeriesMatchRule`
5. 从控制器的完整目录条目中找出同系列分集
6. 排除已有匹配或自定义剧名的分集
7. 批量保存剩余分集的匹配记录
8. 逐项同步这些分集的媒体索引
9. 通知页面原位刷新海报墙

传播使用已读取的 TMDB 详情，不为每个分集重复请求搜索或详情接口。其他分集共享同一剧集级元数据，但继续保留各自的远程 ID、路径、季号、集号、文件大小和字幕引用。

规则可以复用手动匹配得到的海报缓存路径。没有本地缓存时，卡片继续使用 TMDB 海报地址。传播不为每个分集下载重复海报。

匹配弹窗返回的结果增加传播摘要：

- 成功传播数量
- 索引同步失败数量
- 规则是否保存成功

全部成功时，页面提示 `已保存“回魂计”，并自动匹配同目录 7 个分集`。部分索引失败时，页面说明失败数量和重试方式。

## 后续资源自动继承

目录加载、媒体库扫描和自动批量整理都先检查同系列规则。命中规则后直接创建匹配记录并同步索引，不调用 TMDB 搜索或详情接口。

规则优先级高于七天无结果缓存。一个分集曾经搜索无结果，但随后同系列另一集得到手动确认时，该分集仍应继承规则。

处理顺序固定为：

1. 保留自定义剧名
2. 保留已有 TMDB 匹配
3. 应用精确同系列规则
4. 跳过七天内无结果记录
5. 执行现有 TMDB 自动匹配

自动批量整理的汇总将规则继承计入成功数量。规则继承不计入 TMDB 搜索请求数量。

## 当前目录海报墙

新增纯分组组件，将原始目录条目转换为文件夹导航和媒体作品组。分组组件不访问网络，也不修改存储。

组件输出两类数据：

- `CloudResourceFolderEntry`：保留当前目录的文件夹导航
- `CloudResourceMediaGroup`：电影或电视剧作品，包含显示记录和排序后的可播放视频

媒体分组遵守以下规则：

- 能识别有效集号的视频按系列键合并
- 同一剧的不同季仍属于一个作品组
- 无法识别集号的视频按远程资源键作为独立电影
- 不同标准剧名不合并
- 不同父目录不合并
- 组内分集先按季号排序，再按集号排序，最后按文件名排序

电视剧组的显示记录按以下优先级选择：

1. 包含自定义剧名的记录
2. 已匹配的记录
3. 解析得到的标准剧名

页面上方显示紧凑的文件夹导航区域。下方只显示作品海报墙，不再把文件夹卡片混入作品网格。

电视剧海报显示中文标题、评分、年份、集数和匹配状态。点击电视剧海报后打开选集列表。点击电影海报直接播放。每个选集继续使用自己的远程 ID、路径、播放解析参数和字幕引用。

作品菜单提供修改剧名、TMDB 刮削和重新匹配。对电视剧执行手动匹配时，选择组内第一条符合识别条件的视频作为锚点，并传播到整个组。

搜索框匹配文件夹名称、作品标题、标准剧名和组内文件名。搜索只改变当前显示，不改变分组身份或传播范围。

## 视频识别大小与文件过滤

网盘资源页复用现有 `MediaRecognitionSettings.cloudMinSizeBytes`，不增加第二套设置。控制器通过强类型提供者读取最新值。

只有 `LocalVideoFileTypes.isRecognizedVideo` 返回 `true` 的文件才能进入海报墙、当前目录刮削、系列传播和自动批量整理。该判断要求文件格式受支持，且文件大小严格大于设置值。

以下条目不显示，也不参与 TMDB 处理：

- 小于或等于网盘识别大小的视频
- 字幕、图片、文档、压缩包和其他非视频文件
- 大小无效且未达到识别阈值的视频

字幕虽然不显示，但控制器保留原始目录条目，用于播放时匹配同名字幕或相同季集号字幕。文件夹不属于媒体文件，继续作为导航入口显示。

相同过滤规则进入以下链路：

- 网盘资源页作品分组
- 当前目录 TMDB 刮削候选
- 来源级自动批量整理候选
- 网盘媒体索引
- 手动匹配后的同系列传播候选

设置变化后，刷新目录或重新扫描即可读取新阈值。页面不缓存旧阈值。

## 并发与错误处理

手动锚点的匹配记录先保存。规则或传播失败不得撤销用户已经确认的锚点匹配。

同系列记录使用批量仓库写入，避免只保存部分记录。媒体索引按资源逐项同步。单项索引同步失败不会阻断其他分集，协调器记录待重试目标，并在下次目录加载时重试。

规则保存失败时，当前目录仍可使用已获取元数据完成一次传播，但结果明确标记规则未持久化。页面提示后续新增资源暂时不能自动继承。

页面销毁、来源切换或控制器释放后不更新旧页面状态。匹配和传播不会写入目录加载错误，也不会阻止视频播放。

规则解析、进度、日志和错误文案不得包含 Cookie、TMDB API Key、播放直链或请求头。

## 测试设计

### 系列身份与规则仓库

- 截图中的 `S01E01` 到 `S01E04` 生成同一个系列键
- 不同码率和发布规格不改变标准剧名
- 不同父目录、不同来源和不同剧名生成不同键
- 无集号视频、电影候选和小文件不能建立规则
- 规则可持久化、覆盖更新和按来源删除
- 应用重启后仍能读取规则

### 手动传播与后续继承

- 手动匹配一集后，当前目录其他未匹配分集立即继承
- 传播只读取一次 TMDB 详情
- 已匹配记录和自定义剧名保持不变
- 最新手动匹配更新规则，但不覆盖已匹配分集
- 规则优先于近期无结果缓存
- 新增分集在目录刷新、媒体库扫描和自动批量整理时继承
- 命中规则时不调用 TMDB 搜索或详情接口
- 批量记录保存失败不产生部分持久化结果
- 单项索引失败不影响其他分集，并进入重试集合

### 海报墙与识别过滤

- 同目录同剧分集合并为一张作品海报
- 多季分集归入同一个作品组并正确排序
- 不同剧和电影保持独立作品卡片
- 文件夹只显示在导航区域
- 字幕、图片和文档不显示
- 字幕仍能随选中视频播放
- 小于、等于和大于阈值的视频分别得到正确结果
- 修改阈值后刷新立即更新海报墙
- 搜索能匹配作品标题和组内文件名
- 手动匹配后作品卡片原位更新中文标题、海报和状态

### 回归与交付

- 夸克和 OpenList 目录导航继续可用
- 夸克直连播放、请求头、字幕、选集和搜索继续可用
- 现有单项刮削、当前目录刮削和自动批量整理继续可用
- 删除来源不会删除远程文件
- 本地媒体库海报墙行为保持不变
- 完整 `flutter test`、`flutter analyze` 和 Windows Release 通过
- 签名 MSIX 清单和桌面安装包验证通过
