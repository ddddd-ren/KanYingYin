# 网盘来源目录变更实时刷新设计

## 背景

OpenList 和夸克来源允许用户重新选择参与媒体库扫描的根目录。当前来源编辑页保存后只持久化配置并退出，没有触发媒体索引重扫，也没有通知本地海报墙和网盘海报墙重新加载。两个媒体库从缓存快照恢复数据时又没有按来源的最新根目录过滤，因此已经移除的旧目录资源仍会显示，播放列表也可能继续使用旧资源的远程 ID。

这会造成配置页显示的是新目录，海报墙和选集却仍来自旧目录。用户点击旧资源后，还可能遇到“夸克播放地址不可用”等与过期引用有关的播放失败。

现有 `CloudMediaIndexer` 已经能够在扫描时正确处理根目录缩减和扩展。本轮不重写扫描器，而是补齐保存后的刷新协调，并在所有缓存读取入口增加最新目录范围过滤，使旧资源在重扫完成前就不可见。

## 目标

- OpenList 或夸克来源的媒体根目录发生变化后，保存成功即进入媒体库更新流程。
- 新配置保存后立即按最新根目录过滤缓存；已移除目录中的旧资源不得继续出现在本地海报墙、网盘海报墙、季度页或后续播放列表中。
- 每次目录变化只扫描发生变化的来源一次，避免本地与网盘两个页面各自重复触发扫描。
- 扫描完成后同步刷新本地海报墙和网盘海报墙，使新增目录资源及时出现。
- 保留仍在有效目录范围内的 TMDB 标题、海报、匹配记录和图片缓存，不因目录变更执行全量清空。
- 更新期间显示“正在更新媒体库”，禁用保存按钮并阻止重复提交。
- 扫描失败时保留已经保存的新目录配置，继续隐藏旧目录资源，并明确提示用户可稍后手动重试。
- 交付版本为 `2.1.23+20123`，MSIX 清单版本为 `2.1.23.0`。

## 非目标

- 不修改、删除或重命名用户的网盘文件、目录、远程 ID 或本地视频文件。
- 不强制中断保存目录前已经打开的播放器；已经开始的播放会话维持现状，后续媒体选择必须使用新索引。
- 不在本轮处理当前有效夸克资源仍无法由 CDN 播放的独立直链问题。若刷新后的当前文件仍失败，再单独排查播放链路。
- 不因仅修改来源名称、地址、凭据或转存目录而自动完整重扫；自动扫描只由媒体根目录集合变化触发。
- 不改变现有 TMDB 搜索、匹配、海报下载和手动纠正规则。
- 不要求 TMDB 可用才能刷新目录；无 API Key、断网或 TMDB 失败时，目录扫描和本地播放仍应工作。

## 目录范围规则

新增 `lib/services/cloud/cloud_source_path_scope.dart`，由 `CloudSourcePathScope` 统一负责路径规范化、根目录选择比较和包含判断。控制器不得各自复制字符串前缀逻辑。

路径规范化遵循以下规则：

- 使用 `/` 作为逻辑分隔符。
- 补齐开头的 `/`，移除根路径以外的末尾 `/`，折叠重复分隔符。
- 根路径 `/` 包含该来源中的所有绝对路径。
- 普通根目录只包含“路径等于根目录”或“路径以 `根目录/` 开头”的条目。
- `/A` 包含 `/A/file.mkv` 和 `/A/Season 01/E01.mkv`，但不包含 `/AB/file.mkv`。
- 新旧根目录集合比较应忽略顺序、重复项和仅由分隔符格式造成的差异。
- OpenList 根目录身份由规范化路径确定；夸克根目录身份由规范化路径与非空远程 ID 共同确定。夸克目录路径相同但远程 ID 改变时必须视为根目录变化并重新扫描。
- 空根目录集合表示用户没有选择任何媒体目录，不得回退为显示该来源的全部缓存。

过滤以索引条目保存的真实来源路径为准，不使用应用生成的虚拟剧名、季度名或文件名判断范围。

## 组件职责

### 路径范围组件

`CloudSourcePathScope` 提供纯函数式能力：

- 规范单个路径和根目录集合。
- 判断条目路径是否位于任一有效根目录下。
- 根据来源类型判断保存前后的根目录选择是否发生语义变化。

该组件不访问磁盘、网络、控制器或 Flutter 状态，便于覆盖边界测试。

### 本地媒体库控制器

`LocalController.reloadCloudLibraryIndex()` 读取每个网盘来源的缓存索引时，先取得该来源最新配置的媒体根目录，再通过统一路径范围组件过滤条目。只有过滤后的条目可以转换成本地海报墙作品、季度、选集和播放目标。

过滤发生在作品聚合之前，因此已经移除根目录中的资源不会影响作品数量、季度数量、集数、版本列表或播放目标。即使自动扫描失败，重新加载旧快照也只能看到仍属于新根目录的条目。

### 网盘资源控制器

`CloudResourcesController._loadSnapshot()` 在把快照写入 `_indexedItems` 和 `entries` 之前执行相同的最新根目录过滤。后续 `_works` 聚合、季度海报和选集都只消费过滤后的集合。

当新目录尚未完成扫描、过滤后 `_works` 为空时，页面可以显示空状态或更新状态，但不得回退展示未过滤的 `snapshot.items`。这条约束消除旧资源重新出现的旁路。

控制器增加公开的 `reloadSourcesAndSnapshot()` 入口，只重新读取来源配置，并为仍然存在的当前选中来源加载缓存快照。该入口保持当前来源选择，不自动扫描，用于刷新协调器在扫描前后更新视图，避免与 `CloudLibraryController.scanSource()` 形成重复扫描。

### 来源编辑页

`OpenListSourceEditorPage` 和 `QuarkSourceEditorPage` 接收可选的 `Future<void> Function(String sourceId)? onRootSelectionChanged` 回调。页面保存前保留原始 `CloudSource`；新建来源按“原始根目录为空”处理。持久化成功后使用 `CloudSourcePathScope` 比较新旧根目录选择：

- 根目录未变化：维持现有保存并退出行为，不扫描媒体库。
- 根目录发生变化：页面保持打开和忙碌状态，显示“正在更新媒体库”，调用一次更新回调。
- 更新成功：结束忙碌状态并退出页面。
- 更新失败：保留新配置，结束忙碌状态，页面不崩溃；显示“目录已保存，但媒体库更新失败，请稍后手动重试”。用户可以返回设置页使用现有手动扫描入口。

回调为可选参数，使不经过 `SettingsModule` 构造的测试或复用场景保持兼容。两个编辑页只负责交互与错误提示，不直接拼装扫描步骤。

### 刷新协调器

新增 `lib/services/cloud/cloud_source_root_refresh_coordinator.dart`，由 `CloudSourceRootRefreshCoordinator` 实现统一的来源目录刷新流程，并在 `IndexModule` 注册为单例。`SettingsModule` 从依赖容器取得协调器，把 `refreshSource(String sourceId)` 作为两个编辑页的 `onRootSelectionChanged` 回调。协调器以来源 ID 为输入，并严格按以下顺序执行：

1. 重新加载本地媒体库的网盘索引，使旧目录资源立即被最新配置过滤。
2. 让网盘资源控制器只重新加载来源配置和缓存快照，使当前网盘页面同步隐藏旧资源。
3. 调用 `CloudLibraryController.scanSource(sourceId)` 一次，扫描新目录并更新该来源快照。
4. 扫描成功后再次重新加载本地媒体库网盘索引。
5. 扫描成功后再次重新加载网盘资源来源与快照。

扫描前的两个刷新操作属于安全收缩：它们只让已移除资源消失，不依赖网络扫描成功。扫描后的两个刷新操作负责让新增资源出现。

若任一步骤失败，协调器记录首个异常并继续执行其余不依赖失败结果的安全过滤刷新，最后抛出保留原始原因的 `CloudSourceRootRefreshException`，由编辑页转换为用户提示。扫描步骤只有在两次扫描前过滤刷新都已尝试后执行；扫描失败时仍尝试最后两次快照重载，以确保视图保持最新范围过滤。只要来源配置已经保存，失败流程不得回滚到旧根目录，也不得重新注入未过滤缓存。

## 完整数据流

1. 用户在 OpenList 或夸克编辑页修改媒体根目录并点击保存。
2. 来源控制器保存新配置；从这一刻起，最新根目录是所有缓存可见性的唯一依据。
3. 编辑页检测到根目录集合发生语义变化，进入“正在更新媒体库”状态并禁止重复保存。
4. 协调器要求两个媒体库重新加载。两者在消费缓存前按新根目录过滤，因此旧目录资源立即消失。
5. 协调器只对该来源执行一次扫描。索引器保留有效目录中的已有识别与 TMDB 记录，移除失效条目并加入新目录条目。
6. 扫描成功后两个媒体库再次加载新快照，新增资源出现在海报墙和选集。
7. 后续点击媒体时，播放目标只能来自过滤后的新索引，不再使用已移除目录中的旧远程 ID。

## 状态与并发

- 来源编辑页复用现有保存忙碌状态，覆盖“持久化配置 + 更新媒体库”的完整生命周期。
- 忙碌期间保存按钮不可点击，目录选择控件不得再次发起保存。
- 同一次保存只调用一次刷新回调；协调器只调用一次 `scanSource(sourceId)`。
- 现有 `CloudLibraryController` 的扫描互斥继续生效，避免用户手动扫描与自动扫描并发破坏索引。
- 页面销毁后不得调用 `setState` 或显示依赖失效上下文的提示；异步完成后沿用现有 `mounted` 检查。
- 不要求强制刷新已经打开的播放器状态；媒体库下一次选择和播放队列生成必须以新索引为准。

## 错误处理与离线行为

- 来源配置保存失败：保持现有行为，不启动刷新，向用户展示保存失败原因。
- 扫描或刷新失败：配置不回滚，旧目录条目继续被范围过滤；提示“目录已保存，但媒体库更新失败，请稍后手动重试”。
- 新根目录为空或暂时无法访问：两个海报墙对该来源显示空状态，不展示旧缓存作为回退。
- TMDB 无 API Key、断网、超时或无结果：不影响路径过滤与目录扫描；已有 TMDB 数据和图片缓存保留，新资源可先以本地识别结果展示。
- 单个缓存条目的真实路径缺失或无效：该条目不属于任何明确配置根目录，不进入媒体库和播放列表。
- 扫描失败后用户触发手动扫描：成功后沿用同一快照加载与过滤链路恢复新增资源。

## 数据保留

目录变更不是“清空媒体库”。索引器继续按稳定资源身份复用仍有效条目的识别结果；过滤层只控制可见性，不删除 TMDB 图片文件、用户手动标题、匹配记录或其他来源的缓存。

只有不再属于该来源任何配置根目录的索引条目会由现有扫描收缩逻辑从该来源快照中移除。即使扫描尚未执行或执行失败，这些条目也会在读取时被隐藏。其他来源以及本地媒体索引完全不受影响。

## 测试与验收

自动化测试必须覆盖：

- 路径规范化后 `/A` 包含 `/A/file.mkv`，但不包含 `/AB/file.mkv`。
- 根目录集合比较忽略顺序、重复项、反斜杠和末尾分隔符差异；夸克同路径远程 ID 变化会被识别为变化。
- 空根目录集合过滤掉该来源全部缓存，而不是显示全部条目。
- 本地媒体库加载旧快照时，来源根目录已从 `/A` 改为 `/B`，旧 `/A` 条目立即不可见且不能生成播放目标。
- 网盘资源页在新根尚未扫描时同样过滤 `/A` 条目，且 `_works` 为空时不会回退展示旧 `entries`。
- OpenList 和夸克编辑页保存根目录变化后各触发一次更新回调，并在更新期间显示忙碌文案、禁用重复保存。
- 只修改来源名称、地址、凭据或转存目录而媒体根目录不变时，不触发自动扫描。
- 刷新协调流程只扫描变化来源一次，并在扫描前后分别重新加载本地与网盘媒体库。
- 扫描失败时新目录配置保留、旧目录资源继续隐藏、错误提示明确，并可通过手动扫描重试。
- 根目录缩减、扩展以及缩减与扩展同时发生时，现有 TMDB 标题、海报、手动匹配记录和图片缓存不会被全量删除。
- 被移除目录中的旧资源不再进入季度、集数、版本列表或后续播放目标。
- TMDB 无 API Key 或断网时，路径过滤、网盘扫描、本地扫描和本地播放仍可用。
