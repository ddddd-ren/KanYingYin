# 百度网盘 dlink 播放修复设计

## 问题

百度网盘中的视频可在百度网页端播放，但看影音 2.1.28 从网盘海报墙打开同一资源时提示“网盘视频解析或加载失败”。现有日志中没有该次播放对应的本机 `127.0.0.1` 播放地址，说明失败发生在播放器打开媒体之前。

代码回溯确认，百度播放解析已经取得 `dlink`，随后由 `BaiduRangeRemoteReader` 探测远程文件并建立本机中转。当前远程请求只发送 Access Token、`Accept-Encoding: identity` 和可选 Range，没有发送百度 dlink 下载要求的 User-Agent。百度官方接口答复要求 dlink 请求携带必传的 Access Token 和 UA 字段；普通浏览器 UA 仍可能被拒绝。

## 目标

使百度官方 dlink 能通过现有本机分段中转稳定播放，同时保持凭据隔离、一次刷新、缓存、预读、拖动和服务端限速规则不变。

本修复与本地剧集分季海报一起进入同一版本，但两项功能在代码和测试中保持独立。

## 请求策略

`BaiduRequestPolicy` 定义百度下载固定 User-Agent：`pan.baidu.com`。`BaiduRangeRemoteReader` 发起以下请求时统一发送该值：

- 初始探测请求。
- 分段 Range 请求。
- 不支持 Range 时的完整顺序读取。
- 经过安全校验后的重定向请求。
- 刷新 Access Token 与 dlink 后的重试请求。

User-Agent 不是用户凭据，可以传递到下载 CDN。Access Token 仍只以查询参数附加到百度官方初始 dlink；当服务器返回绝对重定向地址时，不把初始查询参数、Access Token、API Key、Secret 或授权头复制到新主机。

初始地址继续只接受百度官方 HTTPS 下载主机，重定向继续拒绝 HTTP、用户信息、空主机、本机、链路本地和私有 IP 地址。本轮不放宽现有主机或重定向安全边界。

## 播放流程

1. 根据稳定 `fs_id` 获取最新文件详情和 dlink。
2. 百度读取器向官方 dlink 发送 Access Token、固定 User-Agent 和单字节 Range 探测。
3. 探测成功后建立带随机令牌的本机 HTTP 中转。
4. MPV 继续只访问 `127.0.0.1`，不直接接收百度 dlink 或凭据。
5. 远程返回 401、403 或 412 时，只刷新一次 Access Token 与 dlink，并使用同一请求策略重试。
6. 重试仍失败时终止播放，不循环刷新。

## 错误与日志

保留页面现有用户提示和提供方错误映射。补充播放解析边界的脱敏日志，使后续可以区分：

- 文件详情或 dlink 获取失败。
- 百度远程探测失败。
- 本机中转启动失败。
- 播放器加载本机地址失败。

日志只记录来源 ID、提供方、处理阶段、错误类型和脱敏主机，不记录完整 dlink、查询参数、Access Token、Refresh Token、API Key、Secret 或完整请求头。

## 测试

- 探测请求携带 `User-Agent: pan.baidu.com`。
- Range 请求和完整顺序读取携带相同 User-Agent。
- 安全重定向继续携带 User-Agent，但不携带 Access Token 或其他凭据。
- 401、403、412 只刷新一次，并在刷新后的请求中继续携带 User-Agent。
- 非官方初始地址和不安全重定向仍被拒绝。
- 本机中转仍返回可供播放器使用的 `127.0.0.1` 地址。
- 夸克、OpenList 和本地播放请求行为不变。
- 日志脱敏检查不出现真实凭据或完整签名链接。

## 实机验收

- 使用截图中的百度视频验证能够进入看影音播放器并开始播放。
- 验证暂停、续播、拖动、切集和退出清理。
- 验证应用配置代理时百度读取仍按现有规则直连。
- 验证百度网页端可播、看影音端也可播；账号会员和服务端速度限制保持原样。
