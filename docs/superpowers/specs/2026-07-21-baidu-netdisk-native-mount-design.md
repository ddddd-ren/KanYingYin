# 百度网盘原生挂载设计

日期：2026-07-21  
目标版本：看影音 `2.1.28+20128`，MSIX `2.1.28.0`

## 1. 背景与目标

看影音已经具备 OpenList 与夸克网盘来源，并完成了云盘递归索引、媒体名称识别、TMDB 刮削、季度海报、多版本选集和夸克原文件分段播放。本轮在这些公共能力上新增百度网盘原生来源，不依赖 OpenList，不复制第二套媒体库页面。

第一版只提供个人账号的只读挂载：用户在设置中填写自己的百度开放平台 API Key 和 Secret Key，通过官方 OAuth `oob` 模式授权一个或多个百度账号，选择多个媒体根目录，随后在现有网盘海报墙中扫描、刮削并播放原文件。

本轮还交付一份供另一台 Windows x64 电脑安装的压缩包，并在该私人构建中注入当前电脑已经配置的 TMDB API Key 作为默认值。

## 2. 范围

### 2.1 包含

- 百度官方 OAuth 授权码、Access Token 与 Refresh Token 生命周期。
- 多个百度账号，每个账号作为独立网盘来源。
- 多个媒体根目录的浏览、选择、保存和实时刷新。
- 百度目录、文件详情、外部字幕和原文件下载地址读取。
- 复用现有云媒体递归索引、大小过滤、字幕匹配和作品树。
- 复用现有剧名、季度、集数、发布规格、多版本和手动确认规则。
- 复用现有 TMDB 中文元数据、季度海报、海报墙和播放器选集。
- 把夸克专用 Range 中转抽象为提供方无关的公共云播放中转。
- 百度原文件边下边播、分段缓存、预读、重连和令牌刷新。
- 私人构建的内置默认 TMDB Key。
- 签名 MSIX 和异机安装 ZIP。

### 2.2 不包含

- 百度 Cookie、BDUSS、STOKEN 或网页内部接口。
- 百度分享链接转存。
- 上传、远程改名、移动、复制或删除。
- 百度网页转码、会员加速或任何绕过服务端限速的行为。
- Windows 文件系统盘符映射。
- 在仓库、日志、诊断包或安装说明中保存真实凭据。

## 3. 官方协议边界

授权仅使用百度官方 OAuth：

- 授权地址使用 `https://openapi.baidu.com/oauth/2.0/authorize`。
- `response_type=code`。
- 无 Web Server 的桌面端使用 `redirect_uri=oob`。
- 权限固定请求 `basic,netdisk`。
- 使用随机 `state` 和短时本地授权会话防止并发授权结果串用；若百度 `oob`
  页面返回 `state` 则必须核对，若页面只显示授权码，则以当前来源、授权会话和
  十分钟有效窗口绑定该授权码，不宣称已经验证浏览器回调状态。
- 授权码只能使用一次，并在十分钟后过期。
- 令牌交换和刷新必须使用用户自己的 API Key 与 Secret Key。

网盘访问仅使用百度官方开放平台接口。目录和文件使用稳定的 `fs_id` 标识，路径只用于展示和目录边界。播放地址通过文件详情接口请求 `dlink`，不使用网页抓包接口或非公开签名参数。

官方依据：

- [百度 OAuth 接入指南](https://openauth.baidu.com/doc/doc.html)
- [百度网盘开放平台接口调用说明](https://developer.baidu.com/question/detail.html?id=179)
- [百度网盘开放平台常见问题](https://developer.baidu.com/article/details/293412)

若官方接口字段、权限或调用方式无法由文档和脱敏实测交叉验证，必须返回“当前版本暂不兼容百度网盘接口”，不得猜测字段继续运行。

## 4. 架构

### 4.1 公共来源层

在 `CloudSourceType` 中新增 `baidu`，并由 `CloudProviderRegistry` 注册。百度来源复用现有：

- `CloudSource`
- `CloudRemoteRef`
- `CloudDriveClient`
- 媒体索引仓库
- 来源目录范围过滤和刷新协调器
- 作品、季度、分集和版本树
- TMDB 协调器和海报缓存
- 网盘海报墙与播放器入口

百度的 `CloudRemoteRef.id` 保存十进制字符串形式的 `fs_id`，`path` 保存官方返回的完整路径。不得用路径或列表位置替代 `fs_id` 作为文件身份。

### 4.2 百度专属组件

- `BaiduOAuthClient`：生成授权地址、交换授权码、刷新令牌和验证账号。
- `BaiduApiClient`：统一请求、分页、错误分类、重试和响应结构校验。
- `BaiduResponseParser`：将官方响应解析为强类型对象。
- `BaiduDriveClient`：实现账号验证、目录、文件详情和播放资源解析。
- `BaiduDirectoryPickerPage`：使用 `fs_id` 浏览和选择多个目录。
- `BaiduSourceEditorPage`：编辑凭据、授权状态、账号和目录。
- `BaiduRangeRemoteReader`：向公共中转提供远程文件长度和 Range 数据。

百度代码不得依赖夸克实现，夸克代码也不得出现百度分支。二者只通过公共接口接入。

### 4.3 公共云播放中转

把当前夸克专用中转拆为提供方无关的公共层：

- `CloudRangeRelayService`
- `CloudRangeRelaySession`
- `CloudRangeChunkCache`
- `CloudRangeRelayProtocol`
- `CloudRangeRemoteReader`
- 提供方无关的中转状态模型

夸克和百度分别实现 `CloudRangeRemoteReader`。公共层负责本机 HTTP、缓存、预取、并发、租约和播放器状态；提供方读取器负责凭据、下载地址刷新、允许主机策略和远程错误分类。

重构后夸克原文件播放参数、缓存上限、状态文案和安全规则必须保持兼容。

## 5. 凭据与安全存储

`CloudCredential` 增加具有明确语义且可空的字段：

- `clientId`
- `clientSecret`
- `accessToken`
- `refreshToken`
- `accessTokenExpiresAt`

旧 OpenList 和夸克 JSON 必须继续兼容。百度每个来源独立保存一份凭据，所有字段写入 Windows 安全存储。

安全规则：

- 普通 `CloudSource` JSON 不含百度凭据。
- 编辑页不回显 Secret、Access Token 或 Refresh Token。
- API Key 只显示掩码或“已配置”，不得显示完整值。
- 留空表示保留已保存的 API Key 或 Secret。
- 修改 API Key 或 Secret 后，必须丢弃旧令牌并重新授权。
- 授权码不持久化。
- `toString()`、异常、日志和诊断包不得包含任何凭据。
- 删除来源时删除该来源的安全凭据、索引、TMDB 记录、海报和播放缓存。
- 删除来源不得调用任何百度远程写接口。

## 6. 授权流程

1. 用户填写来源名称、API Key 和 Secret Key。
2. 看影音生成高熵随机 `state` 和授权会话 ID，只保存于当前编辑会话内存。
3. 使用系统浏览器打开官方授权页，参数包含 `redirect_uri=oob` 和 `scope=basic,netdisk`。
4. 用户授权后将页面显示的一次性授权码粘贴回看影音。
5. 看影音校验授权码格式、当前来源、授权会话和十分钟有效窗口；若 `oob` 页面
   同时返回 `state`，还必须核对 `state`，随后再交换令牌。
6. 调用官方账号信息接口，验证令牌和 `netdisk` 权限。
7. 成功后原子保存 API Key、Secret、Access Token、Refresh Token 和过期时间。
8. 只有授权成功的来源才能进入目录选择器和保存流程。

Access Token 在到期前五分钟刷新。同一来源并发请求只允许一个刷新任务，其他请求等待同一个结果。官方返回新 Refresh Token 时必须与 Access Token 一起原子替换；失败结果不得覆盖仍然有效的旧令牌。

刷新失败、授权撤销或缺少权限时，将来源标记为“需要重新授权”，但保留已有索引、TMDB 元数据和海报。重新授权成功后继续使用原有媒体根目录。

## 7. 目录、扫描与媒体库

授权成功后，百度目录选择器从根目录开始分页浏览。支持选择一个或多个目录，并保存 `fs_id + path`。

分页处理必须具备：

- 按 `fs_id` 去重。
- 空目录正常结束。
- 重复页和游标不前进时停止并报告不兼容。
- 单页失败不会伪装成目录为空。
- 限流按有上限退避重试。
- 取消扫描后停止后续分页和子目录调度。

扫描只进入用户选择的根目录，复用现有视频扩展名、文件大小阈值、外部字幕和路径范围规则。百度条目写入现有媒体索引后，由公共作品树完成：

- 规范剧名解析。
- 文件夹季度和纯集号继承。
- 发布规格后缀清理。
- 同一集多版本归并。
- 唯一集数统计。
- 作品级 TMDB 匹配。
- 每季度对应海报。
- 完整选集与真实播放版本。

修改媒体根目录后立即隐藏旧范围资源，只协调一次百度来源扫描。扫描失败时保留旧索引；百度不可用不得阻断本地媒体库、OpenList、夸克或应用启动。

## 8. 原文件边下边播

播放步骤：

1. 根据 `fs_id` 获取最新文件详情和 `dlink`。
2. 建立百度远程读取器与本机中转会话。
3. MPV 只接收带随机令牌的 `127.0.0.1` 地址。
4. 本机中转把播放器的 Range 请求映射为百度远程 Range 请求。
5. Token 或下载地址失效时，由百度读取器刷新后重试一次。

公共参数沿用夸克已验证配置：

- 16 MiB 分段。
- 256 MiB、16 段 LRU 缓存。
- 头尾预取和后续两个分段预取。
- 前台请求优先。
- 最多两个远程请求。
- 500ms、1s、2s 连接重试。
- HEAD、完整 GET、单 Range 和 416。
- 退出、切集、刷新和销毁时释放租约并清理缓存。

百度读取器必须：

- 只向官方接口附加 Access Token。
- 不向 CDN 重定向主机发送 API Key、Secret 或授权头。
- 仅接受 HTTPS。
- 校验 206、`Content-Range`、文件总长度和目标 `fs_id`。
- 对 401、403、鉴权错误或过期地址，刷新 Token 与 `dlink` 后只重试一次。
- 不在日志中记录完整 `dlink`、查询参数或重定向地址。

若上游不支持 Range，则以顺序流方式继续播放并提示拖动可能不可用，不完整预下载文件，也不伪造 Range 响应。百度账号、会员和服务端带宽限制保持原样，本机中转不提供限速绕过。

播放器显示“正在连接百度网盘、预缓冲、重新连接、速度不足、需要重新授权”等状态。第一版只使用原文件，不调用未验证的百度网页转码接口。

## 9. 界面

“网盘数据源”增加百度网盘选项。百度编辑页包含：

- 来源名称。
- API Key。
- Secret Key。
- 打开百度授权。
- 一次性授权码。
- 授权状态与脱敏账号信息。
- 重新授权。
- 多媒体根目录选择。
- 启用或停用来源。

百度来源直接进入现有网盘资源页和来源筛选，不新增独立海报墙。卡片、手动确认、修改显示剧名、季度海报、多版本、选集和字幕交互与夸克一致。

百度来源不显示分享导入、上传、改名、移动或删除远程文件操作。

## 10. 错误模型

在现有 `CloudDriveErrorType` 上补足或映射以下百度错误：

- API Key 或 Secret 无效。
- 授权码无效、已使用或过期。
- 用户拒绝授权。
- Access Token 无效。
- Refresh Token 刷新失败。
- 缺少 `netdisk` 权限。
- 接口限流。
- 目录或文件不存在。
- 下载地址过期。
- 下载服务器拒绝 Range。
- 网络速度低于视频码率。
- 官方响应结构不兼容。

错误只影响对应百度来源。目录失败不得改写为“空目录”，播放失败不得删除索引或触发全盘扫描。

日志只允许记录来源 ID、错误类型、HTTP 状态和脱敏主机。

## 11. 内置默认 TMDB Key

目标安装包为私人构建，用户已明确接受内置 TMDB Key 可被提取的风险。

构建流程从当前电脑看影音设置中读取已保存的 TMDB Key，并通过私密临时 `dart-define` 文件注入 Release：

- 不把 Key 写入 Git、规格、测试或发布说明。
- 不把 Key 直接放在构建命令参数中。
- 临时文件只允许当前用户读取。
- Release 构建和 MSIX 封装完成后删除临时文件。
- 构建失败时也必须在清理阶段删除。

运行时的 Key 优先级：

1. 用户在设置页保存的 Key。
2. 构建时注入的默认 Key。
3. 无 Key 时只读 TMDB 缓存。

设置页只显示当前使用“用户 Key”或“内置默认 Key”，不得显示默认 Key 内容。清除用户 Key 后恢复默认 Key。

该 ZIP 必须标记为仅供个人设备使用，不得公开发布。

## 12. 异机安装包

桌面交付：

- `看影音-2.1.28.msix`
- `看影音-2.1.28-异机安装包.zip`

ZIP 内容：

- 签名的 `看影音-2.1.28.msix`。
- `KanYingYin.cer` 公钥证书。
- `安装看影音.ps1`。
- `安装看影音.cmd`。
- UTF-8 编码的 `安装说明.txt`。
- `SHA256.txt`。

ZIP 不得包含 PFX、私钥、证书密码、百度凭据、临时构建密钥文件或可编辑的 TMDB Key。

安装脚本先执行不依赖证书信任的预检：

1. MSIX 文件 SHA-256。
2. 包标识为 `com.kanyingyin.player`。
3. 清单版本为 `2.1.28.0`。
4. 发布者为 `CN=KanYingYin`。
5. 架构为 x64。
6. 包内签名文件存在。
7. MSIX 的签名证书指纹与 ZIP 中 `KanYingYin.cer` 完全一致。

任一预检失败立即退出，不导入证书、不安装应用。全部通过后，只把公钥证书导入
当前用户的“受信任人员”，再重新验证 MSIX 的 Windows 签名状态为 `Valid`，最后
使用系统 MSIX 安装能力安装应用。导入证书后的签名验证失败时停止安装并报告错误。
脚本不得导入到本地计算机证书库，不得要求或接触签名私钥。

## 14. 实机验收

真实 API Key、Secret 和百度令牌只在应用设置中输入，不进入聊天或仓库。

主电脑验证：

- 至少一个真实百度账号完成 OAuth 授权。
- 选择至少两个媒体根目录。
- 验证电影、电视剧、多季度、纯集号、字幕和多清晰度版本。
- 实测普通视频、4K、拖动进度、切集、重启和令牌刷新。
- 回归夸克、本地视频、OpenList、字幕、全屏、硬件解码和 Anime4K。

另一台 Windows x64 电脑验证：

- 解压异机安装包。
- 执行一键安装并确认签名与证书信任正常。
- 首次启动时按既有规则检查桌面和开始菜单快捷方式。
- 确认无需手动填写 TMDB Key 即可刮削。
- 填写该电脑使用的百度 API Key、Secret 并完成 OAuth 授权。
- 扫描和播放百度视频。

若无法取得真实百度开放平台凭据，自动化、Release 和 MSIX 可以完成，但不得宣称真实百度账号扫描与播放已经实机通过。
