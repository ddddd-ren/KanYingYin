# 本地剧集分季海报刮削设计

## 目标

本地媒体刮削电视剧时，根据每个视频已识别的季度编号，使用 TMDB 对应季度的海报。不同季度目录不再统一使用整部剧的总海报，行为与现有夸克网盘刮削保持一致。

## 范围

- 仅调整本地 TMDB 元数据合并和海报下载逻辑。
- 不改变本地媒体分组、播放器、页面控件、缓存文件名或用户原始视频。
- 不改变电影刮削行为。
- TMDB 不可用、未配置 API Key 或断网时，本地扫描和播放继续可用。

## 数据流

1. TMDB 匹配成功后，将作品资料及 `seasons` 季度资料一并合并到每个本地索引项。
2. 下载海报前，按视频所在目录分组，确保同一目录只下载一次。
3. 从目录内索引项读取 `seasonNumber`，在 TMDB `seasons` 中查找相同编号的季度。
4. 对电视剧优先使用该季度的 `posterUrl`；季度不存在、季度海报为空或季度未识别时，使用作品级 `posterUrl`。
5. 电影始终使用作品级 `posterUrl`。
6. 海报继续保存为目录内的 `tmdb-poster.jpg`，并将成功保存的路径写回该目录所有相关索引项。

## 元数据合并

现有标题、简介和海报锁定规则保持不变。季度资料属于作品结构信息，应随最新 TMDB 详情保存，不因作品级海报锁定而丢失。这样既能选择正确季海报，也能让元数据序列化和后续重新加载保持一致。

“获取海报”选项、现有封面和作品级海报锁定行为保持不变；本轮只改变已允许下载海报时所选择的海报地址。

## 回退与错误处理

- 对应季有海报：使用对应季海报。
- 对应季缺少海报：回退作品总海报。
- 未识别季度：回退作品总海报。
- 作品总海报也不存在：跳过该目录，不把它计为下载失败。
- 单个目录下载失败：记录失败并继续处理其他目录，已刮削的 TMDB 元数据继续保留。
- 强制重新匹配：允许覆盖旧的统一海报，使已有媒体库更新为分季海报。

## 测试

- 同一部剧第一季和第二季分别下载对应季度海报。
- 同一季度多个视频位于同一目录时只下载一次。
- TMDB 缺少某季海报时使用作品总海报。
- 未识别季度和电影继续使用作品总海报。
- 合并后的本地索引保留季度元数据。
- 手动选择 TMDB 候选与自动匹配使用相同分季逻辑。
- 海报部分下载失败时不丢失已匹配元数据，并继续处理其他目录。
