# 让本地季度海报和 TMDB 交互与网盘保持一致

本文定义本地季度海报修复、媒体卡片悬停交互和本地完整 TMDB 刮削对话框。目标是让本地与网盘使用同一种卡片行为和候选确认流程，同时保留各自的数据仓库与文件边界。

## 目标和验收范围

本轮修改完成以下结果：

- 本地不同季度优先显示各自的 TMDB 季度海报
- 本地季度海报缺失时回退作品总海报
- 本地和网盘卡片的底部信息只在鼠标悬停时显示
- 本地和网盘卡片右上角的三点菜单保持常驻
- 本地三点菜单可以打开完整的 TMDB 刮削与重新匹配对话框
- 本地对话框支持搜索词、媒体类型、年份、季度提示、刮削选项、候选搜索和确认应用
- 搜索候选不会修改索引，只有确认候选才保存元数据和海报

本轮不改变播放器控件、动画时长、动画曲线、媒体识别规则或用户原始媒体文件。

## 已确认的交互

本地和网盘继续使用 `ImmersiveMediaCard`。两端都将信息覆盖层设置为 `hover`，沿用现有 160 ms `easeOut` 动画。覆盖层包含标题、副标题、集数、格式、评分、年份和刮削状态。

三点菜单不受覆盖层透明度影响。它始终显示在卡片右上角，用户无需悬停即可打开操作菜单。本地菜单保留现有播放、修改剧名、自定义封面、在线查找封面和复制路径，并增加明确的 **TMDB 刮削** 与 **重新匹配** 入口。

两个入口打开同一个完整对话框。**TMDB 刮削** 使用当前识别结果作为默认值，**重新匹配** 也使用当前值，但文案明确表示将重新搜索。两者都先搜索候选，再由用户确认。

## 本地季度海报根因和修复

本地刮削服务已经按季度下载 `tmdb-poster.jpg`。当前卡片仍显示相同海报，原因有两层：

1. `tmdbPosterUrlForPaths()` 始终读取作品总海报 `metadata.posterUrl`
2. `LibraryMediaCoverFallback` 始终让网络海报覆盖本地已下载海报

修复在数据选择源头完成：

- `tmdbPosterUrlForPaths()` 根据索引项的 `seasonNumber` 调用共享 `TmdbPosterPolicy`
- 策略优先返回对应季度的 `posterUrl`，缺失时返回作品总海报
- 卡片将自定义封面或已下载海报放在网络海报之前
- 本地封面加载失败时回退季度网络海报
- 生成的视频缩略图仍位于 TMDB 网络海报之后

最终优先级为：自定义或已下载本地海报、季度 TMDB 网络海报、作品总海报、视频缩略图、占位图。

## 共享 TMDB 对话框

在页面共享边界新增来源无关的 `TmdbMatchDialog<TResult>`。该组件只依赖共享 TMDB 类型，不导入本地或网盘记录模型；不放入 `features/library/presentation`，避免媒体库纯展示组件越层依赖 TMDB 服务类型。

对话框接收以下输入：

- 标题和说明文字
- 初始搜索词、年份和媒体类型
- 可选季度号
- 当前 `TmdbScrapeOptions`
- 搜索回调
- 应用候选回调

搜索回调返回 `TmdbRankedResult`。应用回调返回泛型 `TResult`，对话框成功后将结果交还调用页面。网盘页面通过适配闭包继续调用现有 `searchPrepared()` 和 `selectPrepared()`；本地页面调用新增的本地准备搜索接口。

原 `CloudTmdbMatchDialog` 的布局、键盘操作、输入校验、候选卡片和错误提示迁移到共享组件。网盘只做调用方式调整，视觉和行为保持不变。

## 本地搜索和保存流程

`LocalTmdbScrapeService` 新增只读的准备搜索方法。该方法执行以下步骤：

1. 从当前卡片路径取得索引项
2. 通过 `LocalTmdbSubjectBuilder` 生成季集证据
3. 用对话框中的搜索词、年份和媒体类型覆盖搜索计划
4. 调用共享 `TmdbScrapeEngine`
5. 返回排序候选，不写入索引和海报文件

用户确认候选后，本地页面调用现有 `selectCandidate()`。该方法继续负责字段合并、匹配来源、规则版本、季度过滤、海报下载和索引提交。

本地卡片只有一个正季度号时显示季度提示，例如 **第 2 季**。没有季度证据或卡片跨多个季度时不显示季度提示。

## 错误处理和数据安全

没有 TMDB API Key 时，对话框不发出网络请求，并提示用户先配置 Key。搜索失败保留当前卡片和索引。海报下载失败仍保存已确认的文字元数据，并显示现有部分成功提示。

本轮操作不重命名、移动或删除本地与网盘媒体。候选搜索不写入任何记录。不同 TMDB ID 只有在用户点击应用后才替换本地结果。

## 测试策略

新增或调整以下测试：

- 本地两个季度分别解析到不同季度海报
- 季度海报缺失时回退作品总海报
- 本地已下载海报优先于网络海报，加载失败时正确回退
- 网盘卡片覆盖层从 `always` 改为 `hover`
- 本地和网盘悬停前隐藏、悬停后显示标题与标签
- 两端三点菜单始终存在
- 本地菜单可以打开共享 TMDB 对话框
- 本地候选搜索不写索引，确认候选后才保存
- 共享对话框校验空搜索词和非法年份，并转发搜索与应用参数

回归测试覆盖本地媒体库、网盘媒体库、TMDB 服务和播放请求。交付前运行完整测试、静态分析、Windows Release、签名 MSIX 和桌面安装包验证。
