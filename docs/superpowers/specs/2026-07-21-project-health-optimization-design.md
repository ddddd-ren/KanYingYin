# 看影音第一阶段项目健康优化设计

## 背景

全项目检查确认当前工程可以通过格式检查、静态分析、892 项测试、Windows Release 构建和 MSIX 封装，但仍存在自动化门禁无法发现的可靠性与安全问题。本阶段优先处理边界明确、能够通过行为测试验证的问题，不进行播放器大型控制器重构，也不改变现有本地与个人网盘产品能力。

## 目标

本阶段建立三个稳定边界：

1. 日志展示、清理、导出和控制台输出使用统一日志来源与脱敏规则。
2. TMDB 凭据只通过安全存储持久化，请求只发送到官方 TMDB 服务。
3. Windows 外部播放器和快捷方式逻辑正确处理中文路径、临时数据与安装入口状态。

同时修复“关于”页面清理缓存时异步异常无法捕获的问题。

## 非目标

- 不删除或下线 OpenList、夸克、百度网盘功能。
- 不在本阶段裁决 `AGENTS.md` 与 `README.md` 的产品定位差异。
- 不拆分 `PlayerController`、`LocalController` 等大型控制器。
- 不改变播放器控件层级、动画时长、动画曲线或既有交互。
- 不修改用户原始视频、字幕、封面或远端文件。
- 不处理 `LocalCoverFinder` 的旧封面静默迁移；该问题进入后续独立的数据安全批次。

## 总体方案

采用分批闭环优化，每批都执行“失败测试、最小实现、定向回归、完整门禁、独立提交”。

### 第一批：日志与缓存可靠性

日志页面不再自行拼接日志目录或文件名，而是通过共享的 `RotatingLogWriter` 获取日志目录和文件列表。页面默认展示活动日志，并允许读取当前可用的轮转日志内容；清理操作必须等待日志写入队列刷新后再删除，避免边写边删。

`AppLogOutput` 对每一行先移除 ANSI 控制码并调用 `LogSanitizer`，再将相同的脱敏文本发送到控制台和日志文件。控制台不得输出包含原始远程 URL 路径、查询参数、Cookie、Authorization、API Key、Token 或密码的内容。

“关于”页面清理缓存前先检查目录是否存在，完整等待删除和容量重新统计。失败时显示不包含本地绝对路径的提示，并保持页面可继续操作。

### 第二批：TMDB 网络与凭据安全

TMDB 元数据请求只访问以下官方端点：

- `https://api.themoviedb.org/3`
- `https://image.tmdb.org`

删除 `PosterService` 中 `tmdb.lsmcloud.cc` 和 `api.tmdb.org` 回退。官方 API 不可用时返回明确失败或空结果，本地扫描、索引、封面回退和播放继续可用。

新增 TMDB 凭据存储边界，复用 `flutter_secure_storage`，但使用独立于云来源凭据的键名。读取顺序如下：

1. 优先读取用户安全存储中的 TMDB 凭据。
2. 安全存储没有值时读取 Hive 中的旧 `tmdbApiKey`。
3. 发现旧值后先写入安全存储；写入成功后删除 Hive 旧值。
4. 安全存储迁移失败时继续使用 Hive 旧值并记录脱敏警告，不能阻断应用启动、本地扫描或播放。
5. 用户清空 TMDB 凭据时同时删除安全存储值和 Hive 遗留值。

构建时 `KANYINGYIN_TMDB_API_KEY` 保持最低优先级，只在没有用户凭据时使用。设置页不得回显构建时内置值。

### 第三批：Windows 集成

外部播放器原生通道接收 Dart UTF-8 字符串后使用 Windows UTF-8 到 UTF-16 转换 API，确保中文、日文和其他非 ASCII 本地路径可以写入播放列表。临时播放列表创建在系统临时目录，文件名使用足够随机的不可预测值。

外部程序启动失败时立即删除临时播放列表；启动成功后由受控延迟清理机制删除。播放列表内容不得写入应用日志。Windows 对需要 Referer 或额外请求头的远程地址继续明确返回“不支持”，不能静默无响应，也不能把凭据写入临时文件。

快捷方式能力拆分为桌面入口和开始菜单入口状态。启动时：

- 已有桌面快捷方式时继续修复其目标、参数、应用标识和图标。
- 桌面与开始菜单入口都不存在时询问用户是否创建桌面快捷方式。
- 任一入口存在时不重复询问。
- 检测失败视为状态未知，显示可理解的错误或保留后续重试机会，不静默创建。
- 用户明确拒绝后保留当前“本次安装不再询问”的语义；重新安装后的全新设置仍会再次询问。

## 错误处理

- 所有清理、迁移和原生调用都必须等待异步结果，不使用无法捕获异常的未等待 Future。
- 安全存储失败不得导致应用初始化失败；仅 TMDB 功能进入不可用或使用兼容旧值状态。
- TMDB 断网、超时或凭据无效时不得清除已有本地索引和元数据。
- 日志错误提示不得包含凭据、完整远程 URL 或用户本地绝对路径。
- 外部播放器临时文件清理失败只记录脱敏警告，不中断应用内播放。

## 测试策略

新增测试必须验证真实行为，不新增仅检查源码字符串是否存在的测试。

### 日志与缓存

- 日志页面读取 `RotatingLogWriter.activeFileName` 对应的真实活动日志。
- 控制台接收的内容与文件内容都已经脱敏。
- 不存在缓存目录时清理成功且容量归零。
- 删除失败时异步异常被捕获并反馈，测试进程不出现未处理异常。

### TMDB

- 用户安全存储值优先于 Hive 旧值和构建时值。
- Hive 旧值迁移成功后被删除。
- 安全存储写入失败时继续使用旧值且不删除旧值。
- 清空凭据同时清理新旧存储。
- `PosterService` 不向非官方域名发送 API Key。
- 无 Key、断网和官方 API 失败时本地媒体扫描及播放请求构建测试继续通过。

### Windows 集成

- UTF-8 到 UTF-16 转换覆盖中文路径。
- 原生临时播放列表在失败时立即清理，在成功时进入延迟清理。
- Windows 遇到 Referer 播放请求时返回明确失败状态。
- 桌面和开始菜单入口组合覆盖“存在、均不存在、检测失败”三类行为。

## 验收与交付

每批完成后执行定向测试，并在第一阶段全部完成后执行：

```powershell
D:\flutter\bin\dart.bat format --output=none --set-exit-if-changed .
D:\flutter\bin\flutter.bat test
D:\flutter\bin\flutter.bat analyze
D:\flutter\bin\flutter.bat build windows --release
```

Windows 原生行为需要补充实机检查：

1. 中文路径本地视频可唤起外部播放器。
2. 日志页面可查看、复制和清理当前日志。
3. 旧 TMDB Key 首次启动后完成迁移，断网时本地扫描和播放可用。
4. 快捷方式缺失场景按设计询问，已有快捷方式能自动修复。

全部验收完成后升级补丁版本，并同步：

- `pubspec.yaml` 中的应用版本与 `msix_version`
- `RELEASE_NOTES.md`
- `lib/utils/version_history.dart`

随后生成 MSIX，验证包身份、发布者、版本和 x64 架构。若本机签名配置可用则验证数字签名；否则明确记录未签名状态。最终文件以 `看影音-版本号.msix` 复制到当前用户桌面。

每批使用独立中文提交信息。提交前检查 `git status --short` 和关键 diff，只暂存本阶段相关文件，不包含现有 `.learnings/ERRORS.md`、`.learnings/LEARNINGS.md` 或其他用户改动。
