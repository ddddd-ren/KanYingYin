# 如何让夸克原文件稳定播放

本文定义看影音的夸克原文件分段预读中转方案。目标是让 MPV 通过本机稳定字节流播放夸克视频，同时保留地址刷新、季度选集、字幕、硬件解码和 Anime4K。

## 文档计划

- **内容类型**：Conceptual
- **目标**：定义夸克分段中转的职责、数据流、安全边界、恢复策略和验收标准
- **读者**：看影音开发与测试人员
- **范围**：夸克原文件播放，不覆盖 OpenList、本地视频或 TMDB
- **待确认项**：无

## 已确认的问题

看影音 2.1.26 已能获取夸克原文件地址、刷新 Cookie，并向官方内容分发网络（CDN）发送最小必要请求头。日志确认 MPV 成功打开 5.95 GiB 的 4K 高效视频编码（HEVC）文件，识别音视频轨道并输出首帧。

持续监控发现 MPV 每播放约 1 秒便缓冲 6 至 50 秒。播放期间没有 HTTP、TLS、鉴权或文件打开错误，TCP 连接也保持建立。播放器进程的平均读取速度约为 0.22 MB/s，而相同 Cookie、文件和 CDN 的 16 MiB Range 请求平均达到 34.21 MiB/s，即 273.65 Mbps。

问题位于 MPV 与夸克 CDN 之间的读取方式。继续增大 demuxer 缓存只会延长等待，不能修复底层吞吐。

## 目标与非目标

本轮实现以下目标：

- 仅对夸克原文件启用本机分段中转
- 使用 16 MiB 远程 Range 请求获取稳定吞吐
- 支持顺序播放、文件头尾解析和任意位置跳转
- 自动刷新过期地址与 Cookie，并从原字节位置继续
- 显示预缓冲、实时速度、已缓存时长和重连状态
- 将单个播放会话的临时缓存限制在 256 MiB
- 保持 Cookie、完整远程 URL 和本机令牌不进入日志

本轮不实现以下内容：

- 不修改、移动、重命名或删除夸克文件
- 不完整下载远程视频
- 不为 OpenList 或本地视频启用中转
- 不改变 TMDB、媒体识别、季度选集和字幕匹配
- 不改变现有播放器控件层级、动画时长、动画曲线或交互行为

## 总体架构

`QuarkDriveClient` 继续解析原文件地址并保存刷新后的 Cookie。新增的 `QuarkRangeRelayService` 将远程资源转换为一次性本机播放地址，MPV 不再直接请求夸克 CDN。

数据按以下顺序流动：

1. `CloudPlaybackResolver` 从 `QuarkDriveClient` 获取远程 URI、必要请求头和远程文件信息
2. `QuarkRangeRelayService` 创建独立的 `QuarkRangeRelaySession`
3. 会话绑定 `127.0.0.1` 随机端口，并生成一次性随机令牌
4. 会话按 16 MiB 对齐请求夸克 CDN，并将分段写入临时缓存
5. `CloudResolvedPlayback` 将本机 URI、空远程请求头和会话租约交给播放器
6. MPV 使用普通 HTTP Range 读取本机地址
7. 切集、替换媒体或关闭播放器时，播放器释放会话租约

## 组件职责

### `QuarkRangeRelayService`

该服务创建和跟踪中转会话。每个播放媒体只允许一个活动会话，服务不复用其他视频的 Cookie、地址或缓存。

服务只接收已通过 `QuarkRequestPolicy` 验证的 HTTPS 夸克原文件资源。OpenList 和本地资源绕过该服务。

### `QuarkRangeRelaySession`

会话拥有本机服务器、远程资源快照、刷新回调、分段缓存和状态流。它只监听 `127.0.0.1`，不监听局域网或公网地址。

本机路径包含至少 128 位随机令牌。请求缺少令牌、令牌不匹配或使用非本机 Host 时，会话返回 404，不暴露资源状态。

### `QuarkRangeChunkCache`

缓存将远程文件按 16 MiB 对齐分段。每个会话最多保留 16 个分段，总上限为 256 MiB。

缓存使用看影音专属临时目录和随机会话名。缓存按最近最少使用策略淘汰，当前输出分段、文件头分段和文件尾分段在使用期间不会被淘汰。

### 播放会话租约

`CloudResolvedPlayback` 和 `PlaybackInitParams` 增加可选的强类型播放租约。`PlayerController` 持有当前租约，并在媒体替换、初始化失败或控制器销毁时幂等释放。

刷新播放资源时，播放器先建立新租约。新媒体完成接管后，播放器再释放旧租约，避免刷新间隙关闭仍在使用的本机地址。

### 中转状态

会话通过只读状态流提供以下状态：

- `connecting`：正在建立远程请求
- `prefetching`：正在准备首个可播放分段
- `ready`：读取速度和缓存满足播放需求
- `reconnecting`：正在重试网络请求或刷新地址
- `degraded`：读取速度低于当前媒体消耗速度
- `failed`：中转和允许的恢复操作均失败

状态包含最近 5 秒平均速度、累计接收字节、缓存字节和估算缓存时长。状态不得包含 Cookie、完整 URL、令牌或远程路径。

## 本机 HTTP 行为

本机超文本传输协议（HTTP）服务支持 `HEAD`、无 Range 的 `GET` 和单区间 Range `GET`。多区间请求返回 416，因为 MPV 播放不需要 multipart Range。

有效 Range 请求返回 206，并包含正确的 `Accept-Ranges`、`Content-Range`、`Content-Length` 和远程媒体类型。无 Range 请求返回 200，并从文件开头按顺序输出至文件末尾。

当请求跨越多个分段时，会话先输出已缓存字节，再等待当前分段下载。远程读取与本机输出使用背压，不能将完整分段复制到额外内存。

## 远程读取与预取

前台读取始终优先于预取。会话最多同时运行两个远程请求：一个满足 MPV 当前读取，另一个预取后续分段。

会话在返回本机 URI 前验证首个远程响应、文件总长度和 Range 能力。首次播放 `.mkv` 或 `.mp4` 时，会话同时启动文件头和文件尾预取，MPV 可以复用进行中的分段请求，不需要等待完整分段写入。

顺序播放时，会话预取当前分段之后的两个分段。MPV 发起不连续 Range 请求时，会话取消尚未使用的预取任务，并从新位置开始，不取消仍被本机响应消费的任务。

## 地址刷新与错误恢复

会话按错误类型执行有限恢复：

1. 远程响应为 401、403、412 或明确签名过期时，会话调用一次刷新回调
2. 刷新回调重新创建夸克客户端，获取新地址与新 Cookie，并验证官方主机边界
3. 会话用新资源重试当前字节区间，不改变本机 URI 和 MPV 播放位置
4. 传输层安全协议（TLS）、连接重置或超时按 500 ms、1 秒、2 秒退避，最多重试三次
5. 同一媒体会话只允许一次鉴权刷新，避免循环请求

远程 Range 响应必须匹配请求起点、终点和文件总长度。非零起点请求收到 200、`Content-Range` 错误、内容长度矛盾或不可信重定向时，会话停止该请求并报告协议错误。

中转启动前失败时，解析器可以回退现有夸克直连。中转已经向 MPV 提供数据后，不在同一播放器实例中静默切换直连，以免产生字节流错位。此时会话完成有限恢复后显示明确错误。

## 播放器状态展示

播放器复用现有加载与错误展示区域，不新增永久控制栏或交互按钮。

- 首帧前显示“夸克预缓冲中”，并附加实时速度
- 重连时显示“夸克正在重新连接”
- 最近 5 秒平均速度低于估算媒体消耗速度时显示“当前网盘读取速度不足”
- 连续稳定播放 5 秒后弱化并隐藏状态
- 播放器信息面板继续提供实时速度和已缓存时长，便于诊断

所有新提示沿用现有动画参数和交互行为。

## MPV 配合参数

本机中转地址使用独立的 MPV 网络缓存配置，不改变本地视频和 OpenList：

- `stream-buffer-size=4MiB`
- `cache-pause-initial=yes`
- `cache-pause-wait=5`
- `cache-secs=30`
- `demuxer-max-bytes=256MiB`
- `demuxer-max-back-bytes=32MiB`

该配置要求 MPV 首次播放和跳转后至少准备 5 秒媒体内容再恢复。中转吞吐正常时，预缓冲应在 3 至 5 秒内完成。配置值集中在强类型策略对象中，测试不得依赖散落的字符串常量。

最近 5 秒平均速度与媒体消耗速度的比较只在播放器已取得有效总时长后启用。媒体消耗速度按远程文件总字节数除以媒体总时长计算；总时长不可用时只显示读取速度，不报告 `degraded`。

## 生命周期与清理

会话关闭时按以下顺序清理：

1. 停止接受新的本机请求
2. 取消远程请求和预取任务
3. 关闭本机响应与 HTTP 服务器
4. 关闭远程 HTTP 客户端
5. 删除该会话的临时分段目录
6. 关闭状态流

所有清理操作必须幂等。应用崩溃后，下次启动只清理超过 24 小时且目录名符合中转缓存格式的孤立目录，不扫描或删除其他应用数据。

## 安全边界

- 远程请求只允许 HTTPS `drive.quark.cn`、其合法子域、`pds.quark.cn` 及其合法子域
- 跨主机重定向重新执行主机验证，不向不可信主机发送 Cookie
- 本机服务器只监听 IPv4 loopback，并拒绝外部 Host
- 本机 URI 使用随机端口和一次性令牌
- Cookie 只保存在内存和现有安全凭据存储中
- 日志只记录主机、分段编号、字节范围、耗时、速度和脱敏错误
- 临时文件名不包含剧名、路径、远程 ID、Cookie 或 URL

## 测试策略

单元测试使用本机伪远程服务器，不依赖真实夸克账号。测试覆盖以下行为：

- `HEAD`、完整 `GET`、单 Range 和非法多 Range
- 16 MiB 分段对齐、跨段输出、缓存命中和最近最少使用淘汰
- 文件头与文件尾预取、顺序预取和跳转取消
- 并发请求上限、背压和客户端提前断开
- 401、403、412 单次刷新与当前位置续传
- TLS 等价传输错误、连接重置、超时退避和重试上限
- 错误 `Content-Range`、长度矛盾、非零 Range 返回 200
- 恶意相似域名、跨主机重定向、错误令牌和非本机 Host
- 媒体替换、初始化失败、切集和播放器销毁时的租约清理
- 状态速度计算、降级判断和日志脱敏

集成测试覆盖 `QuarkDriveClient`、`CloudPlaybackResolver`、中转会话、`LocalVideoController` 和 `PlayerController` 的完整数据流。OpenList 与本地播放测试必须断言不会创建中转会话。

## 实机验收

使用已复现问题的 5.95 GiB 4K HEVC 样本执行以下验收：

- 首帧在点击播放后 5 秒内出现
- 连续播放 5 分钟不发生重复缓冲
- 跳转至 10%、50% 和 90% 后均在 3 秒内恢复
- 远程读取速度达到媒体平均消耗速度的两倍以上
- 临时缓存不超过 256 MiB
- 切集或退出后 3 秒内停止远程流量并删除会话缓存
- Cookie、完整 URL、一次性令牌和远程 ID 不出现在日志中
- 内嵌字幕、外部字幕、音轨、硬件解码、Anime4K、全屏和季度选集正常

工程验收包括完整 `flutter test`、无错误的 `flutter analyze`、Windows Release 构建、签名 MSIX、清单版本和桌面包 SHA-256 验证。
