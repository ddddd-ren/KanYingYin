# 本地与网盘 TMDB 统一刮削设计

## 背景

看影音当前的本地媒体库和网盘媒体库已经共用 `TmdbScrapeOptions`、`ITmdbClient` 与 `TmdbMatcher`，但仍分别实现搜索草稿、媒体类型判断、候选回退、字段覆盖和海报处理。相同作品可能因此得到不同搜索词、不同候选顺序或不同自动匹配结果。

本地链路当前以 `seriesName` 为单一搜索标题，在自动模式下根据是否存在集号只选择电影或电视剧；网盘资源链路会清洗文件名或目录名，并在自动模式下同时搜索电影和电视剧；网盘作品链路还会依次尝试多个标题候选。本地链路遵守字段锁定与覆盖选项，网盘链路在重新选择后通常直接覆盖索引字段。三条链路的季度海报筛选和图片缓存位置也各自实现。

此外，2.1.31 公共 MSIX 使用 `sign_msix: false`，Windows 更新界面无法验证发布者证书。后续交付必须使用现有签名材料生成可验证的安装包，并提供必要的证书安装入口。

## 目标

1. 本地媒体和网盘媒体对相同作品使用同一套名称清洗、类型判断、候选评分、自动匹配和元数据合并规则。
2. 本地和网盘保留各自的扫描、索引、图片落盘与播放边界，不把远程来源模型引入本地媒体库。
3. 保留手动确认、用户锁定和自定义标题；旧自动结果只在安全条件满足时迁移。
4. TMDB 不可用时不影响本地扫描、媒体库展示和播放，也不清除已有元数据。
5. 下一可交付版本生成经过验证的签名 MSIX 或带证书安装入口的签名异机安装包。

## 非目标

- 不合并本地索引仓库与网盘索引仓库。
- 不修改播放器控件、动画、硬件解码或 Anime4K 行为。
- 不删除、移动或重命名本地视频、字幕和封面。
- 不修改网盘文件、目录、远程 ID 或服务端元数据。
- 不在本阶段重写本地分集识别器或网盘作品树构建器。
- 不自动覆盖无法判断来源的旧手动匹配结果。

## 方案选择

### 采用：共享规则内核与来源适配器

新增与来源无关的刮削主题、搜索规划、元数据合并和海报选择边界。本地与网盘把已有识别结果转换为共享输入，执行同一个规则内核，再通过各自适配器持久化。

该方案能用契约测试证明规则一致，同时保留本地路径、网盘来源、图片缓存和索引事务的现有边界。

### 不采用：本地直接复用网盘服务

网盘服务依赖来源 ID、远程路径、作品树和云图片缓存。让本地直接调用会把远程概念泄漏到本地媒体库，并增加索引误更新风险。

### 不采用：继续保留两套服务并复制规则

改动较小，但规则会再次漂移，无法长期保证用户要求的“一致”。

## 共享领域模型

### `TmdbScrapeSubject`

统一描述一次作品级刮削所需的信息：

- 稳定媒体键，仅用于调用方关联结果，不参与评分。
- 按优先级排列的标题候选。
- 可选年份。
- 已识别的季度号与集号集合。
- 媒体类型证据：电影、电视剧或未知。
- 现有 TMDB 元数据。
- 标题、简介和海报锁定状态。
- 现有匹配来源：自动、手动或旧版未知。
- 已应用的统一规则版本。

标题候选由来源适配器提供原始信息，但必须经过同一个规范化器：手动刮削名优先，其次是已识别作品名、作品树候选、目录名或文件名；清除季集标识、年份括号、分辨率、编码、字幕组和发布规格后去重。年份作为独立字段参与评分，不重复留在搜索标题中。

### `TmdbMatchOrigin`

新增强类型来源：

- `automatic`：共享引擎达到阈值后自动选择。
- `manual`：用户在候选列表中明确选择。
- `legacyUnknown`：旧记录缺少可靠来源信息。

本地索引项、网盘资源记录和网盘作品记录都持久化来源及统一规则版本。读取旧 JSON 时回退 `legacyUnknown` 和版本 `0`，保持向后兼容。

## 统一搜索与匹配规则

### 搜索标题

共享引擎按候选优先级逐个搜索。空标题和规范化后重复标题会被忽略。某个标题返回候选后仍必须完成统一评分；低分或接近的候选不能因为“有结果”而自动选择。

### 媒体类型

- 用户明确选择电影或电视剧时严格遵守该选择。
- 自动模式下，只要存在有效季度或集号证据，就只搜索电视剧。
- 没有季集证据但来源明确为电影时只搜索电影。
- 无法判断时同时搜索电影和电视剧，并对合并候选统一去重、排序和评分。

电影和电视剧候选以 `mediaType + TMDB ID` 去重。同分时依次按标题完全匹配、年份匹配、媒体类型证据和 TMDB 返回顺序稳定排序，不能因本地或网盘调用顺序不同而改变结论。

### 自动匹配

两端统一使用 `TmdbScrapeOptions` 的语言、最低分数和候选领先幅度。最佳候选必须满足媒体类型证据、最低分数及相对第二候选的领先幅度，否则返回相同顺序的候选并进入“需要确认”。

自动选择后获取详情，并把实际评分写入 `matchConfidence`。手动选择获取相同详情，但来源记录为 `manual`，不能被后续自动任务替换。

## 统一元数据合并

新增纯函数型合并策略，输入现有元数据、TMDB 详情、刮削选项和字段锁定状态，输出新的 `TmdbMetadata`。

- 标题：标题锁定或 `overwriteTitle == false` 且已有标题时保留，否则使用 TMDB 中文标题和原始标题。
- 简介：简介锁定或 `overwriteOverview == false` 且已有简介时保留，否则使用新简介。
- 海报：海报锁定或 `overwritePoster == false` 且已有海报时保留；`fetchPoster == false` 时不引入新海报。
- 背景图：`fetchBackdrop == false` 时保留已有值，不引入新背景图。
- 评分、上映日期、语言、匹配时间和置信度按本次有效详情更新。
- 季度列表只保留媒体库实际存在的季度；无法确定实际季度时保留 TMDB 详情，但不预下载无关季度海报。

本地和网盘必须调用同一合并策略。来源适配器不能再次自行覆盖字段。

## 统一海报选择

新增纯函数型海报选择策略：

1. 电视剧且存在有效季度号时优先选择该季度海报。
2. 季度海报不存在时回退作品总海报。
3. 电影始终使用作品总海报。
4. `fetchPoster == false` 时不产生新下载任务。
5. 海报锁定时保留现有图片，不产生覆盖任务。

图片存储保持来源隔离：本地适配器继续保存到本地媒体目录，网盘适配器继续保存到看影音专属缓存目录。下载失败只返回图片失败状态，不撤销已成功的元数据更新。

## 来源适配器

### 本地媒体适配器

从同一作品的本地索引项构造主题，使用已识别作品名、季度、集号和年份。持久化时按作品范围原子更新本地索引，并保持字段锁定、手动分集修正和用户原始文件不变。

### 网盘作品适配器

从 `CloudWorkIdentity`、标题候选、实际季度和已有作品记录构造主题。匹配成功后更新作品记录和同作品索引项，继续使用应用缓存保存作品与季度海报。

### 网盘资源兼容适配器

现有资源级入口暂时保留，但搜索、匹配、合并和海报策略全部委托共享内核。作品树能够确定归属时优先使用作品级主题；只有独立视频或尚未进入作品树的资源使用资源级主题。

## 旧数据迁移

定义统一规则版本 `1`。扫描或加载时按以下规则处理版本 `0` 的匹配记录：

1. `manual`、字段锁定、自定义刮削名或已保存系列规则的记录直接保留，只补写来源和规则版本。
2. `legacyUnknown` 先用新规则重新评估，但不立即覆盖。
3. 新规则仍选中相同 `mediaType + TMDB ID` 时允许刷新详情、合并字段并写入版本 `1`。
4. 新规则选中不同 ID、候选不明确或请求失败时保留旧元数据，状态改为“需要确认”，不能清空标题、简介、评分或海报。
5. 迁移完成或进入待确认后记录规则版本，避免每次启动重复请求；用户手动重新刮削时仍可主动刷新。

新产生的自动与手动结果从一开始写入准确来源和规则版本，后续不再依赖旧数据启发式判断。

## 数据流

1. 来源扫描器完成本地或网盘媒体识别。
2. 来源适配器把作品转换成 `TmdbScrapeSubject`。
3. 共享引擎生成规范化标题候选与媒体类型集合。
4. `ITmdbClient` 只请求官方 TMDB API，返回的候选按类型和 ID 去重。
5. 共享匹配器产生稳定排序、自动匹配结论或待确认候选。
6. 自动匹配或用户手动选择后获取详情。
7. 共享合并策略与海报选择策略生成持久化结果和图片任务。
8. 来源适配器提交索引事务并执行来源专属图片缓存。
9. UI 从统一结果映射成功、待确认、失败和图片部分失败提示。

## 错误处理与安全边界

- 没有 API Key 时跳过 TMDB 请求，不阻断本地和网盘扫描。
- 断网、超时、凭据无效或 TMDB 服务异常时保留已有元数据和索引。
- 搜索失败不能写成“未匹配”；只有成功请求且候选为空才记录未匹配。
- 图片下载失败不回滚元数据，也不把完整远程 URL、Key 或本地绝对路径写入日志。
- 迁移失败保留旧规则版本以便重试，但使用退避或单次扫描限制避免请求风暴。
- 所有索引更新只作用于看影音索引和缓存，不删除用户原始视频、字幕、封面或网盘文件。

## 测试策略

### 共享规则单元测试

- 相同标题候选得到相同规范化搜索标题和年份。
- 季集证据、电影证据和未知证据得到确定的媒体类型集合。
- 电影与电视剧候选合并后稳定去重、排序和评分。
- 标准、严格和宽松置信度使用同一阈值逻辑。
- 字段锁定与覆盖选项产生相同合并结果。
- 季度海报缺失时统一回退作品海报。

### 本地与网盘一致性契约测试

用同一批电影、单季剧、多季剧、纯集号文件、带年份目录和发布规格文件名分别构造本地与网盘主题，断言两端得到相同的：

- 标题候选和搜索顺序。
- 年份和媒体类型集合。
- 候选排序、置信度及自动匹配结论。
- 元数据字段合并结果。
- 当前季度海报选择。

### 迁移测试

- 相同 TMDB ID 的旧自动结果安全刷新并写入版本 `1`。
- 不同 ID、低置信度和并列候选保留旧数据并进入待确认。
- 手动、锁定、自定义标题和系列规则记录不被自动替换。
- 断网、无 Key 和图片失败不清空已有媒体库数据。

### 回归测试

- 本地扫描、网盘扫描和播放请求构建在无 TMDB 时继续通过。
- 本地分季海报、网盘分季海报与手动候选流程继续通过。
- OpenList、夸克和百度来源使用同一网盘规则，不修改远程文件。

## 签名与交付

下一版本同步更新 `pubspec.yaml`、`RELEASE_NOTES.md`、`UPDATE_DIALOG_COPY.md`、`lib/core/app_version.dart` 和 `lib/utils/version_history.dart`。

交付流程必须：

1. 运行格式检查、完整测试、静态分析和 Windows Release 构建。
2. 使用当前用户签名目录中的 `certificate.pfx` 与受保护密码生成签名 MSIX。
3. 验证清单身份为 `com.kanyingyin.player`、发布者为 `CN=KanYingYin`、版本一致且架构为 x64。
4. 验证 `Get-AuthenticodeSignature` 返回 `Valid`。
5. 若目标电脑尚未信任证书，同时生成固定清单异机安装包，包含公钥证书、安装脚本、说明和 SHA-256，不包含 PFX、密码、TMDB Key 或临时构建参数。
6. 把最终 MSIX 以 `看影音-版本号.msix` 复制到当前用户桌面，并校验与构建产物哈希一致。

## 验收标准

- 相同媒体输入在本地与网盘得到相同 TMDB 搜索、候选和自动匹配结果。
- 本地与网盘统一遵守同一刮削设置和字段保护规则。
- 旧手动或锁定数据不被自动覆盖，旧自动结果只在同 ID 时刷新。
- TMDB 断网或无 Key 时扫描、媒体库和播放继续可用。
- 所有自动化测试和 Windows Release 构建通过。
- 桌面交付包清单、数字签名、版本、架构和哈希验证通过。
