# 把看影音优化为独立的 Windows 媒体库界面

本文定义第一阶段用户界面优化。目标是让看影音形成独立的 Windows 桌面媒体库体验，同时保持播放器行为、媒体数据和网盘能力不变。

## 目标与受众

本轮面向使用 Windows 管理本地视频和个人网盘媒体的用户。完成后，用户应能从应用框架、导航名称、媒体卡片和操作布局识别看影音自己的产品语言，而不是把它视为 Kazumi 的换色版本。

## 方案选择

本轮采用“先重塑应用框架，再优化内容浏览”的方案。它同时修改主题令牌、桌面侧栏、导航语义、媒体卡片和工具栏，但不重做播放器。

另外两个方案不采用：只更换颜色和图标无法消除原有信息架构；一次性重做所有页面和播放器会扩大回归范围，并违反保持播放器交互的项目约束。

## 隔离与回滚

所有修改位于 `D:\KanYingYin\.worktrees\ui-refresh-v1`，分支为 `codex/ui-refresh-v1`。主工作区 `D:\KanYingYin` 保持不变。

用户可以并行保留两个目录。若放弃本轮版本，可删除 worktree 和对应分支；若接受本轮版本，可将提交合并到 `main`。本轮不推送远端分支。

## 视觉系统

默认主题使用固定的“银幕档案馆”配色，不再只依赖 `colorSchemeSeed` 生成完整外观：

- 夜幕背景：`#0D1117`
- 主表面：`#151B24`
- 抬升表面：`#1D2632`
- 银幕白：`#EDF2F7`
- 雾蓝主色：`#78A9D4`
- 放映琥珀：`#D5A354`

浅色主题使用同一雾蓝主色和中性灰表面。动态配色继续作为用户选项，但默认品牌主题保持稳定。

现有 MiSans 继续承担中文正文。控件圆角收敛到 8 px 至 12 px，只有对话框和大型面板使用更大圆角。悬停反馈控制在 120 ms 至 180 ms，不增加持续装饰动画。

本轮不生成新的应用图标。现有 Kazumi 角色资产不再作为 Windows 界面元素使用，正式替换全平台图标另开独立设计任务，避免未经确认的生成图进入安装包。

## 应用框架与导航

宽窗口使用展开式桌面侧栏。侧栏顶部显示看影音名称，主要入口依次为“本地媒体库”和“网盘媒体库”，底部固定“设置”。“我的”改为“设置”，因为应用没有个人账户中心。

窗口宽度小于桌面断点时，侧栏收窄为图标导航；更窄时使用底部导航。布局根据宽度判断，不再根据横竖屏判断。

内容区域使用中性表面和 12 px 圆角，不再使用大面积主题主色容器。自定义标题栏、拖动区域和 Windows 窗口按钮行为保持不变。

## 媒体浏览

本地与网盘媒体继续使用同一张基础卡片。卡片保持 2:3 海报比例，标题和一行关键元数据常驻海报下方。悬停后显示播放入口与次要操作，不再把识别作品所需的全部文字隐藏在遮罩中。

卡片常驻状态不超过两个：字幕状态和刮削状态。分辨率、编码、文件时间等信息合并为一行次要文本。右键菜单和更多按钮继续承载编辑、重新匹配与详情操作。

海报网格使用桌面密度。单卡目标宽度为 168 px 至 220 px，并随窗口宽度调整列数。本地与网盘封面采用一致的裁切策略。

## 工具栏

本地媒体库首行保留来源选择、扫描、媒体库入口、刷新和面包屑。读取媒体信息、生成缩略图、获取海报和 TMDB 刮削收进“更多操作”菜单。

第二行承载状态、搜索和排序。扫描、刮削与缩略图任务继续显示现有进度，不改变任务执行逻辑。

网盘页保留来源选择、刷新和来源管理。移除来源、批量刮削与自动整理归入更多操作，危险操作继续要求确认，并明确不会删除网盘原始文件。

## 设置与遗留文案

首阶段只调整设置入口和用户可见语义，不重写全部设置控件。启动页选择的未知值兜底改为“本地媒体库”，清除“推荐”遗留文案。

关于页面继续保留 Kazumi 来源说明，但将它归入“开源许可与致谢”。本轮不删除许可证或归属信息。

## 数据、安全与错误状态

本轮不修改媒体索引格式、凭据、播放记录、网盘路径或 TMDB 数据。任何扫描、刮削、删除来源和播放错误继续沿用现有处理逻辑。

空状态使用具体动作，例如“选择本地文件夹”或“添加网盘来源”。加载、错误和无结果状态必须保留键盘可访问的恢复入口。

## 测试与验收

先为以下行为增加失败测试，再写实现：

1. 桌面宽窗口显示带文字的侧栏，窄窗口切换为紧凑导航。
2. 顶级设置入口显示“设置”，不再显示“我的”。
3. 内容外壳使用中性表面，不使用主色容器作为整页背景。
4. 媒体卡片在未悬停时仍显示标题和关键元数据。
5. 本地工具栏把次要媒体处理操作收进更多菜单。
6. 启动页兜底文案不再出现“推荐”。

完成定向测试后，执行格式检查、`flutter test`、`flutter analyze` 和 Windows Release 构建。随后更新补丁版本、发布说明和应用内版本历史，生成 Microsoft 应用包（MSIX），验证清单版本，并复制到当前用户桌面。

## 非目标

- 不改变播放器控件层级、动画时长、动画曲线、全屏、字幕、选集、硬件解码或 Anime4K 行为
- 不合并本地与网盘的数据仓库
- 不删除或迁移 `BangumiItem` 等内部兼容模型
- 不删除任何本地或网盘原始视频
- 不自动安装新 MSIX，不推送远端分支
