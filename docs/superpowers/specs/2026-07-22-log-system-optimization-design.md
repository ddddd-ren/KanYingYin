# 日志系统优化设计

## 目标

让看影音的诊断日志页面正确显示真实日志、在大日志文件下保持流畅、清晰区分严重程度，并确保控制台、文件与导出包均不会泄露凭据。

## 背景

当前文件日志由 `RotatingLogWriter` 写入 `kanyingyin.log`，但 `LogsPage` 仍读取旧文件名 `kanyingyin_logs.log`，导致“错误日志”页面通常没有数据，清空按钮也不会清除真实日志。

日志页会一次读取整个活动文件并同时保留完整文本、全部行和展示行。活动日志最大可达 10 MB，播放器日志频繁时会造成不必要的内存占用与界面等待。现有页面也缺少日志文件身份、记录状态和严重程度层级，呈现为密集的原始文字。

文件写入已进行脱敏，但控制台在脱敏前输出原始内容。`LogSanitizer` 也尚未覆盖带引号的 JSON 凭据键值。

## 设计决定

### 1. 单一日志数据源

`LogsPage` 不再自行调用 `getApplicationSupportDirectory` 或拼接文件名。它改为通过 `logger.dart` 中的日志 API 获取活动文件、日志目录和清空结果：

- 读取当前文件前等待写入队列完成；
- 清空时删除活动日志和所有由 `RotatingLogWriter` 管理的历史日志；
- 历史日志只显示名称和修改时间，并通过同一写入器提供的列表读取；
- 不处理或删除目录中不属于 `kanyingyin-*.log` 与 `kanyingyin.log` 的遗留文件，例如 `kazumi_logs.log`。

### 2. 尾部分页读取

新增一个只负责读取的日志尾部读取器，使用 `RandomAccessFile` 从文件末尾按块反向查找换行符：

- 初次加载返回最新 300 条完整文本行；
- 每次向上加载再读取 300 条更早记录；
- 读取器以字节偏移量作为分页游标，避免重复读取和一次性加载整个文件；
- 在 UTF-8 多字节字符边界处安全解码；遇到截断或读取错误时展示明确的错误状态，而不影响播放器。

日志页默认展示最新记录，用户向上滚动查看历史；复制操作针对当前选中的日志文件全文，读取前先刷新写入队列。

### 3. 诊断工作台界面

日志页保留 `SysAppBar` 与现有导航方式，但调整内容层级：

- 标题为“诊断日志”，顶部显示当前文件名、文件大小、最新修改时间与“正在记录”状态；
- 顶部工具区提供带文字的“复制日志”“清空日志”操作，清空前显示确认对话框；
- 提供当前日志与历史轮转文件的选择入口；
- 日志主体使用固定的深色等宽诊断面板，方便长文本和路径阅读；
- `TRACE`、`DEBUG`、`INFO`、`WARNING`、`ERROR`、`FATAL`、`PLAYER` 按级别使用不同色条和标签，内容仍然可选择与复制；
- 空状态、目录不可用、文件读取失败均给出具体说明与恢复操作；
- 现有“播放器表现层”的控件层级、动画时长、曲线和交互行为不在此改动范围内。

### 4. 统一脱敏

`AppLogOutput` 对每一条打印行先移除 ANSI 控制码并调用 `LogSanitizer`，随后将同一份安全文本输出至控制台和异步文件写入器。播放器日志继续通过同一脱敏器写入文件。

`LogSanitizer` 扩展处理带引号的 JSON 键值，例如 `"token":"value"`、`"Authorization":"Bearer value"`、`"cookie":"value"`，并保留原有 URL、HTTP 请求头和常见键值对脱敏规则。脱敏应保留协议、主机、端口、错误类型和堆栈信息，以保证诊断价值。

### 5. 范围边界

本迭代不改动日志轮转阈值、文件保留数量、诊断 ZIP 的保存位置或播放器底层日志采样频率。TMDB 网络异常的业务级恢复策略属于后续网络可靠性工作，不在此日志优化中更改。

## 测试与验收

新增或扩展测试覆盖：

- 日志页与写入器使用相同的活动文件路径；
- 清空操作删除活动日志和轮转日志，但不删除其他遗留文件；
- 尾部读取器能在包含中文的 UTF-8 文件中按页返回最新行并继续读取更早行；
- 日志页默认显示最新记录，能够切换历史日志，加载和失败状态可见；
- 控制台和文件使用同一脱敏文本；
- JSON 形式的 token、Authorization、Cookie 不出现在日志输出或诊断 ZIP；
- 原有日志轮转、导出、全局异常登记测试继续通过。

验收命令为 `flutter test`、`flutter analyze`、Windows Release 构建。作为可交付版本时，还需按项目规范更新版本、更新日志、生成并验证 MSIX、复制安装包到桌面，并只提交本次相关改动。
