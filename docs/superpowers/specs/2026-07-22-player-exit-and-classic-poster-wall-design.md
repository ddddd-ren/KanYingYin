# 修复播放器退出并恢复经典海报墙

本文定义看影音 2.1.37 的合并改动：修复从播放页返回时的生命周期竞态，并恢复提交 `1585d82` 之前的本地与网盘海报墙。两个改动共用一个签名安装包，但使用独立提交，支持分别回滚。

## 目标

离开播放页时，应用先停止界面计时器和用户输入，再释放底层播放器。播放器异步回调在资源释放后不再读取空对象。媒体库同时恢复改版前的整张海报覆盖式卡片和悬停动画。

新版主题、导航、工具栏和设置命名保持不变。

## 播放器退出门禁

新增单一职责的退出协调器。`VideoPage` 持有协调器，`PlayerItem` 订阅退出信号。协调器只记录是否已经开始退出，并以同步、幂等方式发出一次退出通知。

真正离开播放路由时，`VideoPage` 按以下顺序处理：

1. 检查并关闭对话框、桌面画中画或全屏状态
2. 拒绝重复的路由退出请求
3. 同步发出退出信号
4. `PlayerItem` 停止周期、隐藏、滚轮和音量提示计时器
5. `PlayerItem` 停止接收点击、双击、快捷键、滚轮和播放手势
6. 执行路由返回
7. 页面销毁时失效播放操作并释放播放器、订阅和播放租约

关闭对话框、退出桌面画中画或退出全屏只完成对应动作，不发出路由退出信号，也不释放播放器。

## 播放器资源可用性

`PlayerController` 提供明确的播放器可用状态。状态同时检查底层播放器是否存在，以及销毁流程是否已经开始。

周期刷新一次读取一份播放器状态快照。快照包含播放、缓冲、完成、音量、位置、缓冲进度和总时长。播放器不可用时不创建快照，周期计时器立即取消，不再分别访问多个带空值断言的属性。

播放、暂停、切换播放状态、跳转、音量和倍速操作先捕获当前播放器引用。播放器不存在或正在销毁时，操作直接结束。异步操作每次 `await` 后重新检查页面是否挂载、退出是否开始，以及播放器生命周期令牌是否仍有效。

播放器销毁保持幂等。重复调用共享同一退出状态，不会再次释放已释放的资源。

## 退出诊断日志

新增以下不含媒体地址和凭据的生命周期日志：

- 播放页开始路由退出
- 播放器界面已停止计时器和输入
- 播放器资源开始释放
- 播放器资源释放完成

播放器已经不可用时，控制操作静默结束，不为预期的退出竞态重复写入错误日志。非预期的释放失败仍记录错误与堆栈。

## 经典海报墙

仅反向恢复提交 `1585d82` 对以下生产文件的修改：

- `lib/features/library/presentation/immersive_media_card.dart`
- `lib/features/library/presentation/library_media_grid.dart`
- `lib/pages/cloud/resources/cloud_resource_poster_wall.dart`

卡片恢复为整张海报覆盖式布局，不保留海报下方的常驻信息区。标题、字幕、详情和全部徽章位于渐变信息层中。

普通媒体库卡片默认隐藏信息层。鼠标进入时，信息层使用 `AnimatedOpacity` 在 `160 ms` 内按 `Curves.easeOut` 显示；鼠标离开时按相同参数隐藏。悬停期间叠加透明度为 `0.04` 的白色蒙层。

卡片移除新版边框变色、阴影抬升和居中播放按钮。`overlayMode` 为 `always` 时信息层持续显示。加载状态使用透明度为 `0.34` 的主题遮罩和居中进度指示器，右上角操作菜单保持可见。

本地与网盘网格统一恢复以下参数：

- `maxCrossAxisExtent: 300`
- `childAspectRatio: 0.68`
- `crossAxisSpacing: 12`
- `mainAxisSpacing: 12`

点击、长按、右键、资源菜单、徽章操作、加载状态及播放入口保持现有行为。

## 测试策略

播放器修复先添加失败测试，再修改生产代码。测试覆盖：

1. 退出协调器只发出一次退出通知
2. 真正返回上一级前先停止计时器和输入
3. 连续返回只执行一次路由退出
4. 退出全屏、画中画或对话框时不释放播放器
5. 播放器不存在或正在销毁时，状态读取和控制操作不抛异常
6. 异步回调在退出后不再更新界面或播放器状态
7. 重复释放播放器只执行一次资源清理

海报墙测试先恢复以下预期，并确认它们在生产代码修改前失败：

1. 信息层初始透明度为 `0`，悬停后变为 `1`
2. 动画时长为 `160 ms`，曲线为 `Curves.easeOut`
3. 信息层包含标题、字幕、详情和全部徽章
4. 卡片不包含新版常驻信息区和居中播放按钮
5. 本地与网盘网格参数为 `300` 和 `0.68`
6. 加载遮罩、菜单、点击、长按和右键操作保持有效

相关测试通过后运行完整 `flutter test`、`flutter analyze` 和 Windows Release 构建。实机回归覆盖普通窗口返回、全屏返回、连续返回和重新进入播放页。
