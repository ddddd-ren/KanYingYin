# 看影音项目检查问题闭环优化设计

## 背景

2026 年 7 月 22 日的项目检查确认格式检查、静态分析、973 项测试与 Windows Release 构建均通过，同时发现项目约束文档、网络探测、发布供应链和依赖维护仍有可执行的改进项。本设计用于闭环这些检查结果，不改变播放器交互和用户媒体数据语义。

## 范围与成功标准

本轮完成以下事项：

1. 代理自动探测和直连探测恢复系统 TLS 证书校验，不再因无条件接受无效证书而产生可用性假阳性。
2. PR 与发布工作流使用第三方 Action 的完整提交 SHA，并保留可读版本注释。
3. `AGENTS.md` 与当前产品能力一致：应用面向本地媒体和用户自有个人网盘，但仍不包含公共在线影视搜索、插件规则、WebView 视频解析或在线评论。
4. 在不改变主版本架构的前提下更新可安全解析的依赖；主要版本升级逐个验证，若上游 API 迁移会扩大到产品行为变更，则保留当前约束并记录原因。
5. 复核现有结构优化成果。大型控制器只在存在可独立测试的剩余职责时继续提取，不为了减少行数进行机械拆分。
6. 完成补丁版本交付：同步版本、发布说明和版本历史，通过全部门禁，生成并验证 MSIX，复制到桌面并提交本轮文件。

## 方案比较

### 方案 A：只修安全问题

修改 TLS 探测和 CI Action，改动最小，但无法闭环文档漂移与依赖维护问题。

### 方案 B：一次性升级全部主要依赖并拆分大型控制器

表面上完成度最高，但会同时引入框架迁移、UI 回归和播放器生命周期风险，问题归因困难。

### 方案 C：分阶段闭环（采用）

先修复具有明确行为边界的安全与可靠性问题，再处理文档和依赖，最后基于已有架构测试判断是否需要额外职责提取。每个阶段都先建立失败测试，再做最小实现和回归验证。

## 详细设计

### TLS 探测

`ProxyManager` 的探测请求继续使用独立 `HttpClient`、既有超时和代理选择逻辑，但不设置 `badCertificateCallback`。无效、过期或主机名不匹配的证书必须由系统 TLS 栈拒绝，并按现有失败路径参与后续代理探测或离线判定。

OpenList 的“允许自签名证书”是用户对特定来源的显式设置，保持不变；通用 `NetworkConfig` 默认不放宽证书校验。

测试以真实行为为目标：使用本机临时 HTTPS 服务和测试证书，证明系统默认客户端拒绝无效证书。若私有探测方法无法在不暴露测试专用 API 的情况下直接调用，则增加最小的可注入客户端构造边界；不使用仅搜索源码字符串的测试代替行为测试。

### CI 供应链

从各 Action 官方仓库解析当前使用主版本标签对应的提交，工作流 `uses:` 固定为 40 位 SHA，并在同行注释原主版本。现有 Windows 格式、分析、测试、Release、MSIX、SignPath 与 GitHub Release 步骤保持不变。

更新工作流契约测试，要求所有非本地 Action 都使用完整 SHA，防止以后退回可变标签。

### 产品约束文档

以当前 README、路由、发布历史和测试覆盖证明的产品能力为准，只修正 `AGENTS.md` 的项目定位描述。个人网盘被定义为用户自有媒体入口，不等同于公共在线搜索。原始文件保护、断网可用和 Windows/MSIX 约束继续保留。

### 依赖维护

先执行普通 `flutter pub upgrade`，吸收兼容约束内的补丁版本。随后逐个评估 `flutter pub upgrade --major-versions --dry-run` 暴露的直接依赖：

- 能在不改变业务语义且通过完整门禁时升级。
- 需要 Flutter Modular 路由架构、文件选择交互或播放器控制重写的升级不与本轮混合，保留约束并在交付说明中记录。
- Git 固定提交的 media-kit 与 audio-service-mpris 不自动漂移。

### 结构复核

现有项目结构优化已经提取本地媒体库偏好、元数据编排、字幕偏好、TrueHD 策略、播放器快捷键和浮层协调，并建立架构依赖测试。本轮通过依赖图和剩余职责复核确认边界；只有发现新的、行为稳定且可独立验证的职责时才提取。

## 错误处理与回滚边界

- TLS 探测失败沿用现有离线/重新探测逻辑，不清除用户媒体或凭据。
- 依赖升级若导致 API 不兼容，回退该单项约束，不回退已经验证的其他改进。
- MSIX 生成或签名失败时保留已通过的代码验证，但不得宣称交付完成。
- 本轮不包含推送远端；现有 `.learnings` 用户改动不得暂存或提交。

## 验证与交付

定向测试通过后执行：

```powershell
D:\flutter\bin\dart.bat format --output=none --set-exit-if-changed .
D:\flutter\bin\flutter.bat test
D:\flutter\bin\flutter.bat analyze
D:\flutter\bin\flutter.bat build windows --release
```

随后更新补丁版本、`RELEASE_NOTES.md` 和 `lib/utils/version_history.dart`，生成 MSIX，验证清单 Identity、Publisher、Version 和 x64 架构，并将 `看影音-版本号.msix` 复制到当前用户桌面。

## 非目标

- 不改变播放器控件层级、动画、手势、全屏、字幕、硬件解码或 Anime4K 行为。
- 不删除 OpenList、夸克或百度网盘能力。
- 不删除、移动或重命名用户本地或远端原始媒体。
- 不创建正式 GitHub Release，不推送远端分支。
