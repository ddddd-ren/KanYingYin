# 设置中心 UI 重构设计

## 背景

看影音当前设置主页及部分设置子页依赖 `card_settings_ui`，其列表、分组卡片、留白和动效与新版桌面导航及品牌主题不统一。不同页面还混用 `AppBar`、`SysAppBar`、普通表单和多套内容宽度，导致进入子页时视觉层级和转场不连续。

本次将整个设置区域重构为看影音自己的“档案馆控制中心”。用户已选择 A 版响应式控制中心布局，以及 B 版“影院氛围”动效。所有设置含义、入口、路由、保存方式和业务结果保持不变。

## 目标

- 设置主页与全部设置子页面使用统一的看影音桌面视觉语言。
- 移除设置区域对旧 `card_settings_ui` 组件的依赖；迁移完成且项目无其他引用后删除该依赖。
- 保留现有 Flutter Modular 路由、MobX/Provider 状态、Hive/安全存储键和业务服务调用。
- 让鼠标、键盘和触控操作获得一致、连续且可关闭的动效反馈。
- 同时适配浅色、深色、OLED、动态配色和窄窗口。

## 非目标

- 不新增、删除或重命名设置项。
- 不改变设置默认值、持久化格式、保存时机或错误语义。
- 不修改播放器控件层级、播放器动画、解码、字幕、选集、硬件解码或 Anime4K 业务行为。
- 不改变本地扫描、网盘来源、TMDB 刮削或用户文件安全策略。
- 不重构设置以外的媒体库、海报墙、侧栏或工具栏。

## 视觉架构

### 设置主页

设置主页使用响应式控制中心网格，而不是连续分组列表。

- 主内容最大宽度为 1180px，居中显示。
- 宽度不小于 1180px 时使用三列；760–1179px 使用两列；小于 760px 使用单列。
- 页面顶部展示“设置”和简短说明，不重复应用侧栏已有导航信息。
- 功能卡按“媒体资料”“媒体来源”“识别”“播放器”“应用”“其他”组织，卡片内显示标题、说明和必要的当前状态摘要。
- TMDB 卡作为跨列重点卡；其他卡保持相同交互面积和层级。
- 卡片点击目标与当前路由一一对应，不在主页直接修改配置。

### 设置子页

所有子页使用统一的 `KSettingsScaffold`：

- 统一标题栏、返回按钮、说明区域、内容宽度和滚动行为。
- 子页内容最大宽度为 920px。
- 设置按语义分区，每个分区使用独立表面面板。
- 面板圆角 14px，使用主题的两级表面色、细边框和克制阴影。
- 图标使用主题色半透明底板；状态值使用胶囊或紧凑尾部文本。
- 长页面只让整页内容进入时淡入，不让每一行级联播放动画。

## 公共组件

新增设置表现层组件，放在 `lib/features/settings/presentation/`：

- `KSettingsScaffold`：标题、返回、内容宽度和滚动容器。
- `KSettingsSection`：分区标题、说明及面板边界。
- `KSettingsTile`：基础操作行，支持说明、尾部控件、启用状态和点击。
- `KSettingsNavigationTile`：带目标箭头及状态摘要的导航行。
- `KSettingsSwitchTile`：保持现有开关回调和保存时机。
- `KSettingsRadioTile<T>`：保持强类型单选值和回调。
- `KSettingsStatusBadge`：展示“自动”“已开启”“关闭”等只读状态。
- `SettingsHubCard`：设置主页的响应式功能卡。

组件只接收显示数据和回调，不读取 Hive、控制器或服务，不承载业务逻辑。页面继续负责取得状态并调用现有方法。

## 数据流与操作逻辑

- 当前页面中的 `onPressed`、`onChanged`、`setState`、存储写入、控制器调用和异步服务调用原样迁移到新组件回调。
- 开关继续按现有方式即时保存，不引入延迟提交或统一“保存”按钮。
- 单选项继续使用原有值类型，不把枚举或对象改为字符串。
- 打开对话框、目录选择器、凭据输入、来源编辑和扫描流程保持原路由及参数。
- 异步操作期间沿用现有禁用、进度和错误状态；新组件只负责统一视觉呈现。
- 返回导航继续由 Flutter Modular 管理，不缓存或复制业务页面状态。

## 动效与衔接

采用用户选择的“影院氛围”节奏：

- 功能卡悬停：280ms，`easeOutCubic`，上浮约 3px、缩放至 1.012，并增强边框及阴影。
- 功能卡按下：90ms，轻微回落至约 0.99，松开后执行现有导航回调。
- 设置页前进转场：260ms，水平方向共享轴效果；新页面从右侧约 16px 淡入，旧页面轻微淡出。
- 返回转场使用同一动画的反方向，不更改系统返回与页面返回按钮逻辑。
- 开关、状态和异步结果：180ms 交叉渐变。
- 子页内容：整块 220ms 淡入与轻微位移，不做逐行瀑布动画。
- 不使用实时模糊、大面积持续动画或高成本着色器。
- `MediaQuery.disableAnimations` 为真时取消位移与缩放，只保留不超过 80ms 的淡入或直接切换。

## 主题与响应式

- 所有颜色从 `ThemeData.colorScheme`、`CardTheme` 和文本主题派生，不写死浅色或深色专用前景色。
- 浅色、深色、动态配色和 OLED 使用相同组件结构。
- 窄窗口不产生水平滚动；尾部状态在空间不足时换行到说明下方。
- 文本缩放后不裁剪标题、说明、状态和开关。
- 鼠标悬停、键盘焦点、回车/空格激活和触控按下均有明确反馈。

## 错误与空状态

- 继续使用现有 SnackBar、Dialog 和页面错误文案，不吞掉异常或改变异常分类。
- 网盘来源为空、显示模式不可用、TMDB 凭据缺失等状态使用统一空状态容器，但原有操作入口和回调保持不变。
- 扫描、保存或连接失败后，当前配置和值不被新 UI 重置。
- 设置表现组件本身不得直接发起网络请求、扫描或文件操作。
