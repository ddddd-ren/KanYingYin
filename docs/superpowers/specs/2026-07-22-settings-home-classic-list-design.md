# 设置首页经典列表风格设计

## 背景

看影音 2.1.38 将设置首页改为“银幕档案馆”卡片墙。实际使用中，大尺寸卡片占用空间较多，信息密度低于原来的分组列表。关于页同时仍显示“界面与操作参考 Kazumi”，不符合当前产品展示需求。

## 目标

- 设置首页恢复接近旧版的紧凑单列分组列表。
- 保留看影音现有自有设置组件，不恢复 `card_settings_ui`。
- 删除关于页整条“界面与操作参考 Kazumi”设置项。
- 保持全部设置入口、路由、业务回调和操作动画不变。

## 非目标

- 不修改任何设置子页的布局和功能。
- 不修改播放器、媒体库、网盘或 TMDB 业务逻辑。
- 不调整悬停、按下、页面转场的时长与曲线。
- 不删除 README 中必要的开源来源说明或开源许可证入口。

## 设置首页布局

设置首页继续使用现有 `SysAppBar` 和 `SettingsHubContent`，内容区域改用 `KSettingsList`、`KSettingsSection` 与 `KSettingsTile.navigation`。

页面不再显示“银幕档案馆”标题区、TMDB 重点卡片和响应式卡片网格。内容最大宽度保持适合桌面阅读的 1000 像素，并按以下顺序显示：

1. 本地媒体库
   - TMDB 刮削
   - 网盘数据源
   - 媒体识别
2. 播放器设置
   - 播放设置
   - 操作设置
3. 应用与外观
   - 外观设置
   - 界面设置
4. 其他
   - 关于

每个入口保留当前图标、说明和路由。宽屏与窄屏均使用同一列分组列表，避免卡片高度和列数变化引起视觉跳动。

## 交互与动画

- 设置项继续使用 `KSettingsTile` 的悬停和按下反馈。
- 页面路由继续使用 `SettingsMotion.pageDuration` 和现有右滑淡入转场。
- Windows“减少动画”适配保持不变。
- `SettingsHubContent` 仍通过 `onOpenPath` 回调打开页面，不直接新增路由或状态依赖。

## 关于页

从“开源许可与致谢”分组中删除整条“界面与操作参考 Kazumi”设置项。

“开源许可证”入口、许可证页面和 README 中的来源说明继续保留，保证法律与项目溯源信息不受影响。

## 代码清理

设置首页迁移完成后，删除只服务于卡片墙的 `SettingsHubCard` 与 `SettingsHubLayout`，同步移除导出和对应卡片布局测试，避免无调用的旧表现代码残留。

## 测试

- 源码测试确认设置首页使用 `KSettingsList`、`KSettingsSection` 和 `KSettingsTile.navigation`。
- Widget 测试在 1280、900、640 像素宽度验证八个入口、四个分组和无布局异常。
- 路由测试确认八个路径及 `Modular.to.pushNamed(path)` 保持不变。
- 关于页测试确认 UI 源码不再包含 Kazumi，同时 README 仍保留来源说明。
- 完整执行格式检查、`flutter test`、`flutter analyze` 和 Windows Release 构建。

## 版本与交付

本次作为新的可交付迭代发布，版本从 2.1.38 更新至 2.1.39。同步更新 `pubspec.yaml`、MSIX 清单版本、发布说明和版本历史，生成签名 MSIX 与异机安装包并复制到桌面。
