# 路径导航与本地文件夹地址输入设计

## 目标

修正媒体库工具栏中空白过大、路径右对齐和外观类似输入框的问题，并让应用内本地文件夹选择器支持直接输入或粘贴 Windows 地址。现有目录选择结果、扫描入口、排序、搜索、导航顺序和业务回调保持不变。

## 范围

- 调整媒体库工具栏的路径导航表现。
- 调整本地文件夹选择页顶部的目录导航区域。
- 增加可编辑地址、键盘提交、跳转按钮、路径校验和错误反馈。
- 补充路径输入、跳转失败、键盘操作、多宽度布局和动作转发测试。
- 作为独立的 2.1.41 签名版本交付，继续保留当前隔离分支和 worktree。

不修改播放器、经典海报墙、本地扫描规则、TMDB、网盘目录选择器和用户原始媒体文件。

## 视觉设计

### 媒体库工具栏

- 保留原三行结构以及所有按钮顺序。
- 移除路径区域独立的大面积胶囊边框，路径直接融入第一行命令表面。
- 路径从左侧开始显示，前置轻量文件夹图标；盘符、分隔符和当前目录形成清晰层级。
- 普通路径段使用主题主色，当前目录使用正文色和中等字重。
- 短路径不再靠右；长路径溢出时仍优先显示当前目录。
- “上级目录”改用无直杆的圆角向上箭头，统一置于 32×32、8px 圆角的按钮区域，避免当前直杆箭头破坏视觉节奏。

### 本地文件夹选择页

- 顶部标题和右侧“选择当前目录”保持不变。
- 第二行改为“上级目录按钮 + 常驻地址输入框 + 跳转按钮”。
- 上级目录同样使用无直杆的圆角向上箭头。
- 地址框高度、圆角、边框和聚焦色复用现有 Material 3 主题，不增加阴影或装饰动画。
- 跳转按钮同时提供箭头和“跳转”文字，避免只有图标时语义不清。

## 交互设计

- 地址框始终可以输入，支持键盘选择、复制和粘贴。
- 按 Enter 或点击“跳转”执行同一动作。
- 输入内容会去除首尾空白；粘贴带成对双引号的地址时自动去除外层引号。
- 支持盘符目录和 Windows 网络共享路径，实际可访问性继续由注入的目录加载器判断。
- 地址为空时提交，返回“此电脑”的磁盘列表。
- 成功进入目录、点击子文件夹或返回上级后，地址框同步为实际当前路径。
- 加载期间禁用重复跳转，并在地址框尾部显示小型进度。
- 手动输入的地址不存在或不可访问时，保留当前有效路径和文件夹列表，在地址框下方显示“目录不存在或无法访问”；不切换到全页错误状态。
- 浏览过程中移动硬盘断开时，继续使用现有不可访问状态和“返回磁盘列表”入口。
- “选择当前目录”仍返回最后一次成功加载的路径，不能返回尚未验证的输入内容。

## 状态边界

- 地址控制器由页面持有并在页面销毁时释放。
- 手动输入文本与已验证的 `_currentPath` 分离，只有目录加载成功后才提交为当前路径。
- 快速连续提交只接受最后一次导航结果，较早的异步结果不能覆盖较新的地址。
- 页面退出后，任何目录加载回调都不得再更新状态。

## 测试与验收

- 输入盘符路径后按 Enter 可以进入目标目录。
- 点击跳转按钮与 Enter 使用完全相同的行为。
- 输入空地址返回磁盘列表。
- 输入带双引号的地址能够正确规范化。
- 无效地址保留原目录和列表，并显示行内错误。
- 较旧的异步加载结果不会覆盖较新的导航。
- 进入子目录和返回上级会同步地址框。
- “选择当前目录”只返回最后一次验证成功的路径。
- 媒体库工具栏和本地目录选择器在 1280、900、640px 下无溢出。
- 工具栏按钮顺序、搜索、排序、扫描和路径回调参数保持不变。
- 播放器及经典海报墙文件相对 2.1.40 不产生差异。
- `flutter test`、`flutter analyze`、Windows Release 和签名 MSIX 全部通过。
