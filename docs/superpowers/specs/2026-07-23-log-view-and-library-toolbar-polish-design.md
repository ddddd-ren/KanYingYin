# 日志展示与媒体库工具栏轻量优化设计

## 背景

当前日志页直接以横向滚动的等宽文本列表展示完整日志，加载、空数据和失败状态只有单行提示，复制与清空使用两个悬浮按钮。技术信息完整，但普通用户难以快速判断应用是否正常，也不便于定位提醒和错误。

媒体库顶部路径工具栏已包含目录选择、媒体来源、扫描、片库、最近目录、上级目录、刷新、更多操作、路径、状态、排序和搜索，但连续图标缺少视觉分组，按钮、路径和搜索区域的层级不够清晰。

## 目标

- 日志页默认简洁易读，需要时可以查看未经改写的完整脱敏原文。
- 支持按等级筛选和关键词搜索日志。
- 保留现有日志读取、轮转、脱敏、复制、清理和增量加载行为。
- 媒体库工具栏保持原有功能、顺序和三行结构，只做轻量视觉整理。
- 日志页与工具栏在常见桌面宽度下不溢出，并遵循 Windows“减少动画”。

## 非目标

- 不修改日志写入格式、日志等级、轮转数量和脱敏策略。
- 不生成可能误导用户的自然语言故障结论。
- 不修改媒体库搜索、排序、扫描、路径导航或更多菜单的业务逻辑。
- 不修改海报墙、播放器控件或播放器动画。
- 不恢复或引入第三方设置 UI 依赖。

## 日志事件模型

新增独立的日志表现模型和解析器。解析器只依赖 `LogArchiveReader.readAll()` 返回的脱敏文本，不访问日志目录或写入服务。

一条事件以现有格式的头部开始：

```text
[ISO-8601 时间] LEVEL
```

支持的等级为 `TRACE`、`DEBUG`、`INFO`、`WARNING`、`ERROR`、`FATAL` 和 `PLAYER`。从头部之后到下一条头部之前的所有行都属于同一事件。

表现模型包含：

- 时间；
- 原始等级；
- 展示分类；
- 摘要；
- 完整原文；
- 原始顺序。

摘要取事件内容的第一条有效文本，允许移除 PrettyPrinter 的边框字符和首尾空白，但不改写内容含义。无法识别头部的内容不会丢弃：前置孤立内容形成“其他记录”，后续孤立行归入上一条事件。

列表按有效时间倒序显示；时间无法解析时保持原始相对顺序。

## 等级与健康摘要

展示分类固定映射为：

- 普通记录：`TRACE`、`DEBUG`、`INFO`、`PLAYER`；
- 提醒：`WARNING`；
- 错误：`ERROR`、`FATAL`；
- 其他记录：无法识别等级的保留内容。

顶部健康摘要只依据事件等级计数：

- 有错误时显示“发现需要关注的问题”；
- 没有错误但有提醒时显示“有少量运行提醒”；
- 没有错误和提醒时显示“运行状态良好”；
- 没有日志时交给空状态组件展示，不生成健康结论。

健康摘要不读取媒体文件内容，不推断错误原因。

## 日志页布局

页面标题使用“运行记录”。内容最大宽度与现有设置页保持协调，使用以下层级：

1. 顶部工具栏：复制全部、清空；
2. 健康摘要：结论、全部记录数、提醒数、错误数；
3. 搜索与筛选：关键词输入框、全部、提醒、错误；
4. 单列事件列表；
5. 滚动到底部后按批次加载更多事件。

每条事件默认显示等级标识、摘要、时间和展开箭头。点击整行后在原位置展开完整等宽原文，再次点击收起。只允许多条同时展开，不引入全局互斥状态。

完整原文继续位于 `SelectionArea` 内，便于选择局部内容。

## 日志搜索与筛选

日志搜索是实际过滤能力，不是装饰输入框。

- 搜索范围为事件摘要和完整脱敏原文；
- 英文匹配不区分大小写，中文按原字符匹配；
- 首尾空白会被忽略；
- 搜索结果与等级筛选叠加生效；
- 输入框提供清空按钮；
- 页面显示过滤后的事件数量；
- 搜索不会修改 `_fullContent`，复制全部仍复制完整日志；
- 清空日志后同时清空搜索关键词、筛选结果和展开状态。

不实现正则表达式、模糊搜索、搜索历史和联网搜索。

## 加载与异常状态

- 加载中：统一状态面板和进度指示；
- 空日志：说明当前没有运行记录；
- 无搜索结果：保留搜索框和筛选器，提示调整条件；
- 读取失败：显示加载失败状态，不展示虚假的空日志；
- 复制失败和清理失败：继续使用现有 Toast 文案；
- 解析失败：保留原文为“其他记录”，不能因为表现层解析失败丢失日志。

复制全部和清空继续调用现有 `_copyLogs` 与 `_clearLogs`，不修改 `LogArchiveReader` 的职责。

## 动画与可访问性

- 事件展开和收起使用约 180ms 的 `easeOutCubic` 动画；
- 筛选切换使用轻量颜色过渡，不做列表整体位移动画；
- Windows“减少动画”启用时，动画时长降为不超过 80ms；
- 事件行提供按钮语义、展开状态和可读等级；
- 图标按钮保留 Tooltip；
- 颜色不是等级的唯一表达，等级同时显示文字或可访问性标签。

## 媒体库工具栏轻量优化

`LibraryPathBar` 保持现有三行结构和动作顺序：

1. 第一行：选择目录、媒体来源、扫描媒体库、片库、最近目录、上级目录、刷新、更多媒体操作、路径；
2. 第二行：目录状态、名称/大小/日期排序；
3. 第三行：搜索当前目录。

只调整表现：

- 图标按钮统一尺寸、圆角、悬停和按下底色；
- 在媒体操作与目录导航之间增加轻量分隔；
- “更多媒体操作”使用稳定的雾蓝强调底板；
- 路径使用轻量表面和边框，始终保持单行；
- 空间不足时优先省略祖先目录，当前目录仍可见；
- 排序选中态使用统一胶囊底板和方向图标；
- 搜索框使用与设置页一致的表面、边框和圆角；
- 状态文本保持单行省略，不挤压排序控件。

不移动、不合并、不隐藏任何现有动作，也不改变 `LibraryPathBarViewData` 和回调签名。

## 代码边界

- 日志解析与表现模型放在 `lib/features/logs/presentation/`，不放入页面 State。
- 日志事件卡、健康摘要、筛选栏和状态面板作为小型表现组件，与日志读取器解耦。
- `LogsPage` 负责异步加载、分页、搜索筛选状态和现有复制/清理动作编排。
- `LibraryPathBar` 只修改表现代码，不改调用方和控制器。
