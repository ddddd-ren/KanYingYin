# 工具栏、路径输入与轨道语言确认设计

## 背景

本地媒体库第一行目前把“获取海报、读取媒体信息、生成缩略图、批量刮削 TMDB 信息”收在“更多媒体操作”菜单中。“获取海报”与完整 TMDB 匹配能力重叠，剩余常用动作又多了一层点击。路径区域仍占据整行剩余空间，不能直接输入地址。

播放器的内嵌字幕与音轨依赖容器提供的语言代码和标题。部分文件没有写入这些元数据，当前会在菜单中显示“未知语种”。用户要求界面不再出现该文案，并要求每条轨道最终具有明确语言。

## 目标

1. 从媒体库顶部入口移除“获取海报”，但保留单部影片获取/更换封面以及底层海报能力。
2. 将读取媒体信息、生成缩略图、匹配影片信息三个动作直接显示为独立图标按钮。
3. 将路径区域改为紧凑、常驻、可编辑的本地文件夹地址框。
4. 自动识别内嵌字幕和音轨语言；自动识别不可靠时先保证正常播放与选轨，再让用户在有判断依据后确认并在本地记住。
5. 所有用户可见界面均不显示“未知语种”。
6. 保持媒体操作语义、播放器切轨、退出流程、控件层级和动画行为不变。

## 非目标

- 不删除 `fetchPosters` 控制器、服务、单部影片封面操作或已有海报缓存。
- 不让地址框接受单个视频文件；本轮仅接受本地磁盘、移动磁盘和 UNC 文件夹路径。
- 不新增在线语言识别服务，不上传媒体路径、轨道标题或字幕内容。
- 不加入 PGS OCR。PGS 是图形字幕，OCR 仍可能误判；无法从可靠元数据确认时使用一次人工确认来保证正确性。
- 不修改经典海报墙、本地与网盘卡片、播放器动画时长、曲线、字幕渲染和解码行为。

## 方案选择

### 工具栏与地址栏

采用已确认的 A 方案：三个媒体操作使用现有图标按钮样式直接展开；路径改为常驻地址框。未采用带文字按钮方案，因为会显著压缩路径空间；未采用独立地址行，因为会增加工具栏高度。

### 轨道语言

采用“可靠自动识别 + 播放不中断 + 按需人工确认 + 本地持久化”的组合：

- 只依赖元数据会遗漏未标注轨道。
- 强行根据图像或少量字符猜测不能保证正确。
- 未能高置信识别的轨道先沿用原轨道名称和既有选轨规则，不阻塞播放器；用户看到字幕或听到音轨后可确认，因此最终显示结果明确且可纠正。

## 媒体库工具栏设计

### 操作布局

删除 `PopupMenuButton<_LibrarySecondaryAction>` 及顶部的“获取海报”菜单项。按以下顺序放置三个独立按钮：

1. 读取媒体信息：`Icons.info_outline`
2. 生成缩略图：`Icons.photo_camera_outlined`
3. 匹配影片信息：`Icons.cloud_sync_outlined`

按钮继续使用 `LibraryPathBar` 的统一按钮样式、禁用规则和 Tooltip。执行中用原有忙碌状态替换对应图标为小型进度环；不改变控制器调用：

- `onFetchMediaInfo`
- `onGenerateThumbnails`
- `onMatchMetadata`

用户可见名称由“批量刮削 TMDB 信息”简化为“匹配影片信息”。该名称仅调整表达，不改变处理范围和 TMDB 设置。

`onFetchPosters` 不再由顶部组件调用。为降低回滚成本，第一阶段允许保留构造参数和底层方法；测试必须保证顶部不再渲染该入口，单部影片封面入口继续存在。

### 响应式约束

三个动作、上级目录和刷新按钮始终直接可见。地址框使用弹性宽度：

- 首选宽度：250px
- 最小宽度：145px
- 不占满整行剩余空间
- 1280、900、640px 测试宽度不得产生横向溢出

窗口变窄时优先缩短地址框，不把三个媒体操作重新收进更多菜单。

## 文件夹地址输入设计

### 组件边界

在 `LibraryPathBar` 内新增独立的有状态地址输入组件。它只负责文本编辑、提交、同步和行内错误，不直接访问文件系统。`LocalPage` 继续负责调用 `LocalController.navigateTo` 并返回成功或面向用户的错误结果。

输入组件接收：

- 当前已验证目录
- 是否正在加载
- 异步提交回调

这样可以保持地址输入状态与媒体库导航逻辑隔离，也便于单独测试。

### 输入规则

- 点击地址框即可输入或粘贴。
- 按 Enter 提交；不增加常驻文字按钮，尾部使用回车/跳转图标提示。
- 去除首尾空白和成对双引号，与本地文件夹选择器保持一致。
- 支持盘符路径和 UNC 文件夹路径。
- 空输入不改变当前目录，显示“请输入文件夹地址”。
- 输入单个文件路径时拒绝跳转，显示“请输入文件夹地址”。
- 地址不存在或不可访问时，保留当前目录、文件列表和输入文本，显示行内错误。
- 成功后同步为控制器确认的规范路径，并清除错误。
- 通过面包屑、历史记录、上级目录或目录卡片导航成功时，地址框同步新路径。

### 显示规则

地址框左侧保留文件夹图标。未编辑时让长路径优先露出末尾目录；获得焦点后光标位于文本末尾，用户仍可正常选择与编辑完整路径。地址框不使用大面积胶囊背景，不恢复旧的整行路径面板。

## 轨道语言识别设计

### 数据模型

`EmbeddedTrackInfo` 继续保留用于自动选择优先级的 `TrackLanguageKind`，同时增加独立的语言描述：

- 标准语言代码或用户自定义标识
- 用户可见语言名称
- 来源：元数据、标题别名或用户确认
- 是否已经确认

`unknown` 可以作为内部未解析状态存在，但不得通过 `primaryLabel`、`detailLabel`、对话框、日志摘要或菜单文案暴露给用户。

### 自动识别顺序

1. 规范化容器语言代码，兼容 ISO 639-1、ISO 639-2 和常见 BCP-47 写法。
2. 解析轨道标题中的中文名称、英文名称、常见缩写和地区标记。
3. 保留现有国语、粤语、台配、简体、繁体、简繁双语和英语识别规则，并补充日语、韩语及常见语种映射。
4. 只有命中明确代码或别名才视为自动确认；不根据文件名、作品国家或编码格式猜测语言。

识别结果与自动选轨分离。普通话、粤语、台配和中文字幕仍按现有优先级选择；其他语言继续尊重文件默认轨和用户手动选择。

### 首次确认流程

轨道列表首次稳定后，如果存在未解析轨道：

1. 不暂缓默认选轨，也不在播放开始时自动弹出模态窗口。字幕与音轨继续按文件默认轨和既有优先级正常选择。
2. 未解析轨道在菜单中沿用“字幕轨道 N”或“音轨 N”作为主标题，并显示编码和原始标题；任何位置都不显示“未知语种”。
3. 用户看到字幕内容或听到音轨后，可从对应的字幕、音轨菜单进入“确认语言”或“修改语言”。
4. 语言确认窗口允许单独确认当前轨道，不要求一次处理媒体中的全部未解析轨道。
5. 确认窗口提供“稍后确认”和关闭按钮，支持 `Esc` 与点击遮罩关闭；关闭不切轨、不暂停，也不影响播放。
6. 语言选择器提供可搜索的常见语言列表，并允许填写自定义语言名称，保证不受固定枚举限制。
7. 保存成功后立即刷新音轨与字幕菜单，但不重新执行默认选轨，以免打断用户当前选择。

播放器左上角的退出/返回按钮不属于语言确认窗口。语言确认不得在播放初始化阶段以模态遮罩覆盖播放器，因此该按钮在播放初始化及常规播放期间始终可操作。用户主动打开确认窗口时，可先通过窗口自身的关闭入口退出确认，再使用播放器退出按钮。

如果播放器正在退出、媒体已切换或组件已销毁，对话框结果必须丢弃，不能访问已释放播放器。确认期间切换媒体时，旧媒体的异步结果也不能污染新媒体。

### 持久化

新增独立的轨道语言偏好服务，不把职责塞进字幕样式设置。记录保存在看影音现有 Hive 设置存储中。

记录键由以下信息形成稳定指纹后哈希：

- 本地文件的规范路径或网盘播放目标稳定标识
- 轨道类型
- 轨道 ID
- 编码
- 原始标题

存储键不直接包含完整媒体路径。记录值保存规范语言代码、显示名称和确认时间。轨道标题或编码变化导致指纹变化时重新确认，避免把旧选择套到另一条轨道。

读取损坏或不兼容记录时忽略该条并重新确认；保存失败时当前播放会话仍使用用户选择，同时提示“语言设置未能保存，下次可能需要重新确认”。

### 菜单文案

确认后的轨道主标题始终为明确语言，例如：

- `简体中文`
- `日语`
- `英语 / English`
- 用户输入的自定义语言名称

轨道编号、原始标题、`PGS / ASS / SRT / TrueHD` 和声道信息放在详情行。任何组合都不得生成“未知语种”。

尚未确认的轨道使用原有中性名称“字幕轨道 N”或“音轨 N”，并在当前轨道的菜单操作中提供“确认语言”；它不是语言名称，也不使用“未知”“其他语种”等猜测性文案。

## 状态与错误处理

- 工具栏动作沿用现有 MobX 忙碌状态，重复点击被禁用。
- 地址提交期间禁用重复提交，但不清空当前列表。
- 地址导航采用最新请求获胜，旧请求完成后不得覆盖新路径。
- 未解析语言不得阻止默认选轨，播放初始化不得因待确认列表提前返回。
- 语言确认使用媒体 generation；退出、切集或重建控制器都会使旧结果失效。
- 关闭或稍后处理确认窗口只丢弃本次输入，不清空轨道列表、不修改当前选轨，也不触碰播放器生命周期。
- 语言确认不调用播放、暂停或切轨之外的新原生接口，不改变退出时停止计时器、停止输入、再释放播放器的既有顺序。

## 测试策略

### 媒体库工具栏

- 顶部不再出现“获取海报”和更多媒体操作按钮。
- 三个媒体操作图标始终可见，Tooltip、禁用和回调转发正确。
- 忙碌状态只影响对应按钮。
- 1280、900、640px 无溢出。
- 单部影片获取/更换封面入口仍存在。

### 地址输入

- 当前路径与地址文本双向同步。
- Enter 提交盘符与 UNC 文件夹路径。
- 成对引号与首尾空白被规范化。
- 空地址、文件地址、不存在和不可访问目录给出准确行内错误。
- 失败保留当前路径和文件列表；成功后清除错误并滚动到列表顶部。
- 快速连续导航只应用最后一次结果。

### 轨道语言

- 覆盖 ISO 代码、BCP-47、标题别名和常见语言。
- 任意未解析轨道的所有用户可见标签都不含“未知语种”。
- 未解析轨道不会阻断默认字幕挂载，也不会在播放开始时自动弹窗。
- 从字幕或音轨菜单可对当前未解析轨道确认语言；已识别和已保存轨道不重复询问。
- “稍后确认”、关闭按钮、`Esc` 和点击遮罩均能退出确认窗口，播放与左上角退出键不受影响。
- 多条未解析轨道可分别确认，不要求一次全部处理。
- 自定义语言可以保存、加载和显示。
- 指纹稳定且存储键不泄露原始路径。
- 损坏记录安全回退到确认流程。
- 媒体切换和播放器退出后，旧对话框结果不再写状态或访问播放器。
- 既有音轨、字幕优先级和手动选择保护测试继续通过。
